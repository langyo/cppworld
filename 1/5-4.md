# 分身术 · 函数重载

在日常生活中，"打开"这个词可以用在很多场景：打开门、打开书、打开电脑、打开网页……虽然都叫"打开"，但具体的操作却完全不同。

C++ 中也有类似的机制，允许我们用同一个名字定义多个函数，只要它们的参数不同。这就是**函数重载**（Function Overloading）。

------

假设我们要编写一个函数来计算两个数的最大值。对于整数和浮点数，我们需要分别处理：

不使用重载的写法：

```cpp
int maxInt(int a, int b)
{
    return (a > b) ? a : b;
}

double maxDouble(double a, double b)
{
    return (a > b) ? a : b;
}
```

使用函数重载：

```cpp
int max(int a, int b)
{
    return (a > b) ? a : b;
}

double max(double a, double b)
{
    return (a > b) ? a : b;
}
```

现在，我们可以用同一个函数名 `max` 来处理不同类型的参数：

```cpp
int main()
{
    cout << max(10, 20) << endl;        // 调用 int 版本，输出 20
    cout << max(3.14, 2.71) << endl;    // 调用 double 版本，输出 3.14
    return 0;
}
```

编译器会根据传入参数的类型，自动选择调用哪个版本的函数。这个过程称为**重载决议**（Overload Resolution）。

------

**重载的规则**

函数重载的依据是**函数签名**的不同。函数签名包括函数名和参数列表（参数的类型、个数和顺序），但**不包括返回类型**。

以下情况可以构成重载：

```cpp
// 参数类型不同
void print(int n);
void print(double n);
void print(const string& s);

// 参数个数不同
void log(const string& msg);
void log(const string& msg, int level);

// 参数顺序不同
void display(int x, double y);
void display(double x, int y);
```

以下情况**不能**构成重载：

```cpp
// 错误！仅返回类型不同，不能重载
int getValue();
double getValue();  // 编译错误

// 错误！参数名不同不算重载
void func(int a);
void func(int b);  // 编译错误，与上面的声明相同
```

------

**重载与默认参数**

当函数重载与默认参数结合使用时，需要注意避免二义性：

```cpp
void func(int a, int b = 10);
void func(int a);  // 这会导致二义性

int main()
{
    func(5);  // 编译错误！编译器不知道该调用哪个版本
    return 0;
}
```

当调用 `func(5)` 时，它既可以匹配 `func(int a)`，也可以匹配 `func(int a, int b = 10)`（使用默认参数）。编译器无法判断应该调用哪一个，因此会报错。

------

**重载决议的过程**

当编译器遇到函数调用时，它会按照以下步骤选择最佳匹配：

1. **精确匹配**：参数类型完全相同。
2. **类型提升**：如 `char` 到 `int`，`float` 到 `double`。
3. **标准转换**：如 `int` 到 `double`，指针到 `void*`。
4. **用户定义的转换**：使用转换函数或转换构造函数。

```cpp
void func(int n);      // #1
void func(double n);   // #2
void func(long n);     // #3

int main()
{
    func(10);      // 精确匹配 #1
    func(3.14);    // 精确匹配 #2
    func(10L);     // 精确匹配 #3
    func('A');     // char 提升为 int，匹配 #1
    func(10.0f);   // float 提升为 double，匹配 #2
    return 0;
}
```

如果编译器找不到唯一的最佳匹配，就会报错，提示调用有二义性。

------

**const 重载**

对于引用和指针参数，`const` 和非 `const` 版本可以构成重载：

```cpp
void process(string& s)
{
    cout << "非 const 版本" << endl;
    s += "!";  // 可以修改
}

void process(const string& s)
{
    cout << "const 版本" << endl;
    // 不能修改 s
}

int main()
{
    string str = "Hello";
    const string constStr = "World";
    
    process(str);       // 调用非 const 版本
    process(constStr);  // 调用 const 版本
    process("Test");    // 调用 const 版本（字符串字面量是 const 的）
    
    return 0;
}
```

这种技术常用于实现容器的元素访问：对于非 const 对象返回可修改的引用，对于 const 对象返回只读引用。

------

**成员函数的重载**

类的成员函数也可以重载，而且 `const` 成员函数和非 `const` 成员函数可以构成重载：

```cpp
class MyClass
{
public:
    void show()
    {
        cout << "非 const 成员函数" << endl;
    }
    
    void show() const
    {
        cout << "const 成员函数" << endl;
    }
};

int main()
{
    MyClass obj;
    const MyClass constObj;
    
    obj.show();       // 调用非 const 版本
    constObj.show();  // 调用 const 版本
    
    return 0;
}
```

------

**运算符重载**

函数重载的一个重要应用是**运算符重载**。我们可以为自定义类型重新定义运算符的行为：

```cpp
struct Point
{
    int x, y;
};

// 重载 + 运算符
Point operator+(const Point& a, const Point& b)
{
    return {a.x + b.x, a.y + b.y};
}

// 重载 << 运算符用于输出
ostream& operator<<(ostream& os, const Point& p)
{
    os << "(" << p.x << ", " << p.y << ")";
    return os;
}

int main()
{
    Point p1 = {1, 2};
    Point p2 = {3, 4};
    Point p3 = p1 + p2;  // 使用重载的 + 运算符
    
    cout << p3 << endl;  // 使用重载的 << 运算符，输出 (4, 6)
    
    return 0;
}
```

运算符重载让我们能够以自然的方式操作自定义类型。我们将在后续章节中详细学习运算符重载。

------

函数重载是 C++ 的强大特性之一，它让我们能够用统一的接口处理不同类型的数据，使代码更加简洁易读。
