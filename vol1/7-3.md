# 询问自己 · this 指针

来看一段成员函数的代码：

```cpp
class Point
{
private:
    int x, y;

public:
    void set(int x, int y)
    {
        x = x;      // 嗯？哪个 x 是哪个 x？
        y = y;
    }
};
```

这段代码写完了，编译器可能会给个警告——参数名叫 `x`，成员也叫 `x`，写在函数里面到底怎么分？按语法，这里的 `x = x` 会找离它最近的那个 `x`——也就是参数 `x`，结果就是参数自己赋给了自己，成员变量纹丝未动。显然这不是我们想要的。

解决办法当然可以给参数换个名字（比如叫 `nx`、`ny`），但很丑。C++ 提供了一种更优雅的手段来区分"参数"和"成员"——`this` 指针。

------

**this 是什么**

在每一个非静态成员函数的体内，C++ 会隐式地塞给你一个指针，名叫 `this`。它指向**调用这个成员函数的那个对象**。

也就是说，如果你写：

```cpp
Point p;
p.set(10, 20);
```

那么在 `set` 函数内部，`this` 就是指向外面那个 `p` 的指针。

有了 `this`，我们就可以把刚才那段写正确：

```cpp
void set(int x, int y)
{
    this->x = x;   // 左边的 x 是成员，通过 this 访问；右边的是参数
    this->y = y;
}
```

`this->x` 一眼就能让人看明白——"我这个函数所属的那个对象的成员 x"。`=` 右边那个光秃秃的 `x` 自然是参数——离着近的那个。

------

**this 的本质**

`this` 是一个**指针**，不是引用。所以访问成员必须用 `->` 而不是 `.`。取当前对象本身要写 `*this`：

```cpp
class Entity
{
public:
    Entity* getPointer() { return this; }         // 返回指向自己的指针
    Entity& getReference() { return *this; }       // 返回自己的引用
};
```

这两行在以后写链式调用时会很常用：

```cpp
class Counter
{
private:
    int value = 0;

public:
    Counter& increment()    // 返回 *this 的引用
    {
        ++value;
        return *this;
    }

    Counter& decrement()
    {
        --value;
        return *this;
    }
};

int main()
{
    Counter c;
    c.increment().increment().decrement();  // 链式调用——爽
    return 0;
}
```

`increment()` 做完自增后还了一个自己的引用，外面接着 `increment()`，再接着 `decrement()`，一行搞定三件事。这就是 `*this` 最常见的使用场景。

------

**const 成员函数中的 this**

如果成员函数加了 `const`，那函数内部的 `this` 就变成了**指向 const 对象的指针**。你不能通过它修改任何成员——这跟函数外约好的一样：

```cpp
class Point
{
    int x, y;
public:
    void set(int x) { this->x = x; }        // 可以改
    int getX() const { return x; } // const 成员函数里 this 是 const 的，改不了
};
```

------

**什么时候要用 this**

大多数时候你不需要写 `this`。调用同一个类的其它成员，直接写名字就行——编译器默认就知道你指的是哪个对象的成员，它帮你把 `this->` 给隐式加上了。

真正需要显式写 `this` 的场景：
1. **参数与成员同名**——`this->x = x`，区分身份。
2. **返回对象自身**——`return *this;`，链式调用。
3. **比较是不是同一个对象**——有时会看到 `if (this == &other)`，判断传入的对象是不是自己。

除此之外，没有必要处处写 `this->`。有些编程风格喜欢都加上前缀，更显眼，但 C++ 社区的一般默契是只在必须的时候加。

------

**this 不存在的场合**

`this` 只在**非静态成员函数**里才存在。静态成员函数（下一节要讲，前缀 `static`）里面没有 `this`，因为它不属于任何具体对象，而属于类本身。

也就是说：
- 普通成员函数 → 有 `this`。
- 静态成员函数 → 没有 `this`。
