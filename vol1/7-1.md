# 系统的构造 · 类的定义与封装

到目前为止，我们用的都是 C++ 内置的基本类型——`int`、`double`、`char`，再加上标准库送的 `string`、`vector`。这些在简单场景下够用，但现实中的"数据"往往是一捆一捆挤在一起的。

比如，你在写一个图书管理系统。一本书有书名、作者、ISBN、价格、库存量……如果全部拆成独立的变量：

```cpp
string book1Title;
string book1Author;
string book1ISBN;
double book1Price;
int    book1Stock;

string book2Title;
string book2Author;
// 一千本书——你要写到什么时候？
```

这样又乱又容易出错。我们需要把同一本书的相关属性"打包"在一起，捏成一个新的类型。这个新类型，在 C++ 里就叫**类**。

------

**一个最简单的类**

让我们从一个超级简单的类开始：

```cpp
class Book
{
public:
    string title;
    string author;
    double price;
    int    stock;
};
```

这就定义了一个叫 `Book` 的新类型。把它想象成一张蓝图形状——它描述了"书"这种数据的样子。蓝图本身不是书，你得用蓝图去造出具体的书实体来：

```cpp
int main()
{
    Book b1;           // 用 Book 这个蓝图创建了一个具体实例 b1
    b1.title  = "C++ Primer";
    b1.author = "Stanley Lippman";
    b1.price  = 128.0;
    b1.stock  = 50;

    cout << "《" << b1.title << "》作者是 " << b1.author << endl;

    return 0;
}
```

`b1` 就是类 Book 的一个**对象**（object），或者叫**实例**（instance）。类和对象的关系，就像饼干的模具和饼干——模具有一个，饼干可以想压多少压多少。

------

**封装：把数据和操作绑在一起**

上面的 `Book` 只是把数据捆在了一起，还算不上真正的面向对象。面向对象编程的核心思想之一是**封装**——把数据和操作这个数据的方法放在一起，对外隐藏内部的实现细节。

比如，书的库存不应该让外部随便改。万一谁手一滑把 `stock` 设成 -100，那可就闹大笑话了。我们加上访问控制：

```cpp
class Book
{
private:            // 私有：只有类内部能碰
    string title;
    string author;
    double price;
    int    stock;

public:             // 公有：外面也能用
    void setStock(int s)       // 通过函数来修改库存，可以在函数里加检查
    {
        if (s >= 0)
            stock = s;
    }

    int getStock() const       // 通过函数来读取库存
    {
        return stock;
    }

    void print() const
    {
        cout << "《" << title << "》作者 " << author
            << "，价格 " << price << "，库存 " << stock << endl;
    }
};
```

`private` 标明的部分只能从类内部（即类的成员函数中）去访问，外部代码是无法碰到的。`public` 则是对外开放的界面——`setStock`、`getStock`、`print` 这些成员函数就是外部和 `Book` 对象打交道的通道。

这就是**封装**的意义：把细节藏起来，只给外界一个干净的、经过约束的接口。外人不会（也不能）直接偷瞄或乱改你的数据。

------

**成员函数**

成员函数就是"属于类"的函数。它们写在类里面，能直接访问这个类的 `private` 成员——这点普通函数做不到。

成员函数的声明写法和普通函数基本一样：

```cpp
class Rect
{
private:
    int width, height;

public:
    int area() const       // 声明并定义了一个常量成员函数
    {
        return width * height;
    }
};

int main()
{
    Rect r;
    // r.width = 10;   // ❌ 碰不到，width 是私有的！
    cout << r.area() << endl;   // ✅ 通过公有函数间接使用 width 和 height
}
```

`area()` 后面那个 `const` 是什么意思？它说明"这个成员函数不会修改对象的任何成员变量"。加了 `const` 的成员函数既能被普通对象调用，也能被 `const` 对象调用。不加的话，`const` 对象就不乐意让你用。

这是个好习惯：凡是不会修改内部数据的成员函数，都应该加上 `const`。它既是对阅读代码的人的一个承诺，也是防手滑的一道屏障。

------

**类定义分写：.h + .cpp**

实际项目中，类的定义通常分开放进**头文件**和**源文件**：

```cpp
// Book.h —— 头文件，放类的声明
class Book
{
private:
    string title;
    double price;
    int stock;

public:
    void setStock(int s);
    int  getStock() const;
    void print() const;
};
```

```cpp
// Book.cpp —— 源文件，放成员函数的实现
#include "Book.h"
#include <iostream>
using namespace std;

void Book::setStock(int s)    // 注意 "Book::"，说明这个函数属于 Book 类
{
    if (s >= 0) stock = s;
}

int Book::getStock() const
{
    return stock;
}

void Book::print() const
{
    cout << "《" << title << "》库存 " << stock << endl;
}
```

`Book::` 这个前缀告诉编译器："这个叫 `setStock` 的函数是 Book 类的成员，不是外面的野函数。" 这样即便别的类也有一个叫 `setStock` 的成员函数，也不会跟 Bad 这个搞混。

分离声明和实现的另一个好处是编译速度——改了实现部分，只重新编译对应的 `.cpp` 就行；如果全写在头文件里，随便改一行，整个项目都得跟着重新编。

------

**小结**

我们刚才学到了：
- **类**是用户自定义的类型，用来把相关的数据和函数打包在一起。
- **对象**是类的实例，是按照类的蓝图造出来的实体。
- **封装**使用 `private`/`public` 来控制访问权限，隐藏实现细节。
- **成员函数**属于类，能直接访问私有成员。
- 加 `const` 的成员函数承诺不修改对象状态。
- 大型项目会把类声明和实现分开放，既清晰又省编译时间。
