# 瓶子与漏水防护 · 数组库与再探原生数组

前面学了原生数组 `int arr[10]`，也学了 `vector`。但还有一种更贴近传统数组的选择——`std::array`。它把数组的"零开销"性能保留下来，同时堵上了原生数组最痛的几个漏洞。

------

## 原生数组的三个漏水点

```cpp
// 问题 1：退化为指针
void printArray(int arr[])          // arr 实际上是个 int*，不知道大小
{
    // sizeof(arr) → 4 或 8（指针大小），不是数组大小！
    // 你只能额外再传一个 size 进来
}

// 问题 2：没有越界检查
int data[10];
data[15] = 42;                      // 编译通过，运行时可能炸、可能不炸但破坏了别的变量

// 问题 3：不能直接拷贝
int src[5] = {1, 2, 3, 4, 5};
int dst[5];
// dst = src;                        //   编译错误——数组不能整体赋值
```

------

## std::array：带着容器接口的数组

`std::array<T, N>` 在 `<array>` 中。它在栈上分配连续内存（和原生数组一模一样），却提供了 `size()`、迭代器、`at()` 等标准容器接口。

```cpp
#include <array>
#include <iostream>

int main()
{
    array<int, 5> arr = {1, 2, 3, 4, 5};

    cout << "大小: " << arr.size() << "\n";      // 5 —— 不会退化！
    cout << "arr[2] = " << arr[2] << "\n";       // 3
    cout << "arr.at(3) = " << arr.at(3) << "\n"; // 4 —— 带越界检查

    // 不会退化的函数传递
    for (int x : arr)                                  // 范围 for 可用
        cout << x << " ";
    cout << "\n";

    // 整体拷贝——原生数组做不到的事
    auto copy = arr;                                   // 底层所有元素自动复制
    copy[0] = 999;
    cout << "arr[0] = " << arr[0] << " (不受影响)\n";   // 1

    try
    {
        cout << arr.at(10) << "\n";    // 越界——抛 out_of_range
    }
    catch (const out_of_range& e)
    {
        cerr << "异常: " << e.what() << "\n";
    }
}
```

`array<int, 5>` 和 `array<int, 6>` 是两种不同的类型——大小是类型的一部分。所以 `array` 不能像 `vector` 那样在运行期改变大小，但换来的是和原生数组完全相同的布局和性能。

------

## 一个应用：固定月份天数查询

用 `array` 管理 12 个月的天数，干净利落：

```cpp
#include <array>
#include <iostream>

int main()
{
    const array<int, 13> daysInMonth = { 0, 31, 28, 31, 30, 31, 30,
                                                31, 31, 30, 31, 30, 31 };
    // 下标 0 占位，让 1 月对下标 1

    int month;
    cout << "输入月份 (1-12): ";
    cin >> month;

    if (month < 1 || month > 12)
        cerr << "无效的月份\n";
    else
        cout << month << " 月有 " << daysInMonth.at(month) << " 天\n";
}
```

`at()` 保证即使用户输入了 13，程序不会默默去读不存在的位置，而是干净地抛出异常。

------

| 特性 | 原生数组 | `std::array` | `std::vector` |
| ---- | -------- | ------------ | ------------- |
| 大小可变 | 否 | 否 | **是** |
| 传给函数退化 | 退化 | **不退化** | 不退化 |
| 越界检查 | 无 | `at()` 有 | `at()` 有 |
| 内存分布 | 栈 | 栈 | 堆 |
| 典型场景 | 对接 C API、嵌入式 | 固定大小 + 容器接口 | 几乎所有日常场景 |

规律：**能用 `vector` 的地方别用原生数组；如果大小是编译期常量且想要容器接口，用 `array`；剩下只有和 C 库交互、嵌入式极端环境才回到原生数组。**
