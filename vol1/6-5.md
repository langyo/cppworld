# 瓶子与漏水防护 · 数组库与再探原生数组

之前我们学过 C 风格的原生数组——`int arr[10]` 这种。到了这一节，我们再重新审视一下它，同时也认识认识标准库给的另一个选择：`std::array`。

------

**原生数组：好用但不安全**

原生数组又好又坏。好处是它最简单、最直接、没有任何开销——你要的，编译器就给你。坏处是：

1. **大小写死**：声明时多大就是多大，不能改。
2. **没有越界检查**：`arr[50]` 在 `arr` 只有 10 个元素的情况下也能通过编译（运气好时会有警告），运行时可能默默读了块不该读的内存。
3. **传给函数时会"掉帽子"**：数组作为参数传进函数时，会退化成一个指针，`sizeof` 啥的全丢了。

关于退化，这里要重点展开一下：

```cpp
void printArray(int arr[])  // 看似传了个数组？
{
    cout << sizeof(arr) << endl;  // 打印出来是 8 或 4 —— 指针的大小，不是数组的大小！
}

int main()
{
    int data[100];
    cout << sizeof(data) << endl; // 打印出来是 400（100 个 int = 400 字节）
    printArray(data);
}
```

在 `main` 里 `data` 是个数组，`sizeof` 自然是 400。但一传到 `printArray` 里，`arr` 就已经是一个普通的 `int*` 指针了，它只知道"我指着某个地址"，而不知道"这个地址上原本放了 100 个 int"。这就是"掉帽子"——函数看不到数组原来的大小。

这就是为什么我们之前传数组给函数时，总要顺带多传一个 `size` 参数——不传就真不知道多大了。

------

**std::array：标准库给你的"改良版数组"**

标准库提供了 `std::array`，位于 `<array>` 头文件中。它跟原生数组非常相似——用栈内存、大小编译期固定、元素连续排布——但它包装成了一个标准容器，不会退化：

```cpp
#include <array>
using namespace std;

array<int, 5> arr = {1, 2, 3, 4, 5};

cout << arr.size() << endl;   // 5——size() 随时都能查到
cout << arr[2] << endl;       // 3——下标也能用
cout << arr.at(3) << endl;    // 4——带越界检查的访问方式
```

注意写法是 `array<类型, 大小>`。大小是**整型常量**，也是类型的一部分——`array<int, 5>` 和 `array<int, 6>` 是不同的类型。所以你不能在运行时才决定一个 `array` 有多大。

传给函数时，它不会退化：

```cpp
void printArray(array<int, 5>& arr)   // 传引用进来，保持完整的 array 身份
{
    cout << arr.size() << endl;        // 不会退化，依然能查到大小
    for (int x : arr) cout << x << " ";
}
```

想拿 `array` 当函数参数时，一般用**引用**（或常引用）来传——别让它被复制一整份。

------

**原生数组 vs vector vs array：一张表说清**

| | 原生数组 `int[10]` | `std::array<int, 5>` | `std::vector<int>` |
| ---- | ---- | ---- | ---- |
| 大小可变？ | 否 | 否 | 是 |
| 越界检查？ | 无 | `at()` 有 | `at()` 有 |
| 传入函数退化？ | 会退化 | 不退化 | 不退化 |
| 内存位置 | 栈 | 栈 | 堆 |
| 适用场景 | 极简场景，或对接 C 接口 | 固定大小但想要容器接口 | 绝大多数日常场景 |

简而言之：**大多数情况用 `vector`，大小绝对不会变且需要容器接口的用 `array`，剩下的时候（对接 C 代码、嵌入式等受限环境）用原生数组。**

`array` 同时也支持迭代器，可以让标准库的算法直接在它身上干活，这一点是原生数组做不到"开箱即用"的。

------

**字符串又分 native 和 std**

字符串也有两套——C 风格的 `char[]` 和标准库的 `string`（即 `std::string`）。它们的关系跟原生数组与 `vector`/`array` 之间的关系非常相似：

```cpp
// C 风格
char cstr[] = "Hello";

// 标准库
string str = "Hello";
```

你能用 `vector` 就别用原生数组，同样的，能用 `string` 就别用 C 风格字符串。原因不用再重复了吧？

唯一要注意的是——对接大量旧系统或操作系统 API 时，对方可能只接受 `const char*` 或 `char*`。这种情况下，调 `string` 的 `.c_str()` 把标准字符串拿到临时 C 风格字符串去应对外部接口就行。

------

**不要越界、不要越界、不要越界**

重要的事情说三遍。不论你是用原生数组、`array` 还是 `vector`，访问元素前，想清楚你要的下标是不是在 `size()`（或对应的长度）以内。

养成习惯：**调试的时候用 `at()`，稳定发布后用 `[]` 提速。** 这就是两种访问方式最合理的分工。
