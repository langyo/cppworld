# 亮相前的准备 · 类的构造函数

回忆一下我们之前创建对象的方式：

```cpp
Book b1;
b1.title  = "C++ Primer";
b1.author = "Stanley Lippman";
b1.price  = 128.0;
b1.stock  = 50;
```

先造个空壳子，再一项一项往里填数据。好笨。

我们想要的是：**创建对象的同时，一步到位把初始值全给它。** 就像变量可以在声明时顺便初始化一样——`int a = 0;`，凭什么对象就不行？

当然可以。C++ 提供了一种特殊的成员函数，会在对象诞生时自动执行，专用来初始化对象——它就叫**构造函数**。

------

**Hello, 构造函数！**

构造函数的名字**必须和类名相同**，并且**没有返回值**（连 `void` 都不要！）：

```cpp
class Book
{
private:
    string title;
    string author;
    double price;
    int    stock;

public:
    // 构造函数
    Book(const string& t, const string& a, double p, int s)
    {
        title  = t;
        author = a;
        price  = p;
        stock  = s;
        cout << "《" << title << "》已加入库存" << endl;
    }

    void print() const
    {
        cout << "《" << title << "》作者 " << author
             << "，价格 " << price << "，库存 " << stock << endl;
    }
};
```

创建对象：

```cpp
Book b1("C++ Primer", "Stanley Lippman", 128.0, 50);
b1.print();   // 一步到位，爽
```

构造函数的执行时机：在对象诞生时自动调用。你看到 `Book b1(...)` 时，`b1` 被创建，构造函数立刻跑起来，把 `t`、`a`、`p`、`s` 依次填进成员里。

------

**默认构造函数**

如果你一个构造函数都不写，编译器会偷偷给你塞一个**默认构造函数**——它什么也不干，只是让对象能够被创建出来而已。

这个编译器给的默认构造函数会尝试调用每个成员的默认构造方式：
- 原生类型（int、double 等）不会被初始化——它们是什么值是随机的（栈上的垃圾值）。
- 类类型成员（如 string、vector）会被调用它们自己的默认构造函数。

但如果你一旦**自己写了任何一个构造函数**，编译器就不会再替你生成默认构造函数了。这时你就不能用 `Book b;` 这种"不传参数"的方式创建对象——除非你也手写了一个不需要参数的构造函数。

```cpp
class Book
{
public:
    Book() = default;     // 告诉编译器：把默认的那个补回来
    Book(const string& t, const string& a, double p, int s)
    {
        // 带参数的初始化...
    }
};
```

`= default` 是一种明确告诉编译器"虽然我写了别的构造函数，但我还是要你帮我生成默认的那个"的手段。只有默认构造、拷贝构造、移动构造、拷贝赋值、移动赋值、析构函数这六种特殊函数可以这样"点名让编译器生成"。

------

**初始化列表：真正的"初始化"**

前面构造函数里我们用的是赋值：

```cpp
Book(const string& t, const string& a, double p, int s)
{
    title  = t;   // 这不是初始化，这是赋值
    author = a;
    price  = p;
    stock  = s;
}
```

真正的初始化是在函数体之前完成的，用**初始化列表**：

```cpp
Book(const string& t, const string& a, double p, int s)
    : title(t), author(a), price(p), stock(s)   // 初始化列表
{
    // 构造函数体（如果需要额外的操作就写这里；不需要的话留空也行）
}
```

冒号后面的 `title(t)` 等价于"在 title 这个成员出生时，直接用 t 去初始化它"——即一步到位把 title 构造成 t 的副本，没有先默认构造再赋值这种绕一整个圈子的低效操作。

对于 `int`、`double` 这种内置类型，赋初值和在函数体里赋值的差距几乎可以忽略不计；但对于 `string`、`vector` 这些类类型成员来说，初始化列表能省下一次先构造出空壳再覆盖的浪费。为了统一养成好习惯，请**优先使用初始化列表**。

另外，初始化列表里成员的顺序应该跟它们在类中声明的顺序一致——即使你写反了，C++ 也会按照声明顺序来初始化，而不是你在列表里的顺序。顺序不一致虽然不会报错，但足以让依赖初始化顺序的代码埋下暗雷。

------

**成员默认值（C++11）**

C++11 起，你可以直接在声明成员变量时给定默认值：

```cpp
class Book
{
private:
    string title  = "未知书名";
    string author = "佚名";
    double price  = 0.0;
    int    stock  = 0;

public:
    Book() = default;   // 使用上面的默认值

    Book(const string& t, const string& a)
        : title(t), author(a)  // price 和 stock 用默认值
    {}
};
```

如果构造函数没有为某个成员提供初始值，就会采用声明处的那个默认值。这样写出来的代码干净不少。

------

**析构函数：谢幕**

有出场就有退场。构造函数在对象诞生时自动执行，而**析构函数**在对象消亡时自动执行。它的名字是类名前加波浪号 `~`，也没有返回值，也不接受参数：

```cpp
class Soldier
{
private:
    static int count;
    int id;
public:
    Soldier() : id(++count)
    {
        cout << "士兵 " << id << " 报到" << endl;
    }

    ~Soldier()     // 析构函数！
    {
        cout << "士兵 " << id << " 退役" << endl;
        --count;
    }
};
```

当对象离开作用域、被 delete（如果是动态创建的）、或者程序结束时，析构函数就会自动执行。这个机制是 C++ 内存管理体系（简称 RAII——资源获取即初始化）的基石——析构函数负责回收构造函数申请的资源。

对于学到这里的新手来说，暂时不需要自己写析构函数——编译器会帮你生成一个，它会自动销毁对象的各个成员。等你学到动态内存那一章，析构函数的价值就会充分展现出来了。
