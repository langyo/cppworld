# 总指挥中心 · 主函数

我们现在知道，任何一个 C++ 程序都是由许许多多的函数拼成的。可放眼整个程序，谁说了算？谁会第一个跑到起跑线上？

答案是 `main` 函数。它是整场演出的导演、总指挥中心、顶头上司。操作系统启动一个程序时，做的第一件事就是来找 `main`，对着它喊一声："开工！"

------

每个 C++ 程序都必须有且只有一个 `main` 函数。没有它，程序根本编不过去——编译器会摊摊手说："我给你翻译了一大堆代码，但我找不到入口在哪啊？"

`main` 函数长得是这个样子的：

```cpp
int main()
{
    return 0;
}
```

我们拆开来看：

- **`int`**：主函数的返回类型。它告诉操作系统"等程序跑完了，我会还你一个整数当汇报。"
- **`main`**：函数名，必须叫这个，改不了。
- **`()`**：参数列表，目前是空的。等下我们会看到带参数的情况。
- **`return 0;`**：程序正常结束，给操作系统报告"一切顺利，没有异常"。

------

**返回值到底给谁看了**

你可能从来没见过 `main` 的返回值长啥样，因为它在图形程序里几乎是隐形的。但在命令行环境下，这个值就很实用了。

Linux/Mac 用户可以试试这个：

```bash
./my_program && echo "成功" || echo "失败"
```

如果 `my_program` 返回 0，屏幕上会显示"成功"；返回非 0，就显示"失败"。

编译工具的自动化脚本（比如构建脚本）就靠这个机理判断程序是不是正常跑完了。返回 0 代表成功，非 0 的一般含义是"出了这样那样的问题"。因此，千万记得：程序正常结束时一定要 `return 0;`。

事实上，在 `main` 函数里，如果你不写 `return` 语句，编译器也会自动给你补一个 `return 0;`。但这不代表你就能心安理得地偷懒——只有 `main` 有这个特权，别的函数谁敢不 `return`，编译器就骂谁。

------

**main 也能收"快递"：命令行参数**

有些程序不是孤零零跑的。你打开终端敲命令的时候，往往会在程序名后面跟上一些附加信息：

```bash
g++ hello.cpp -o hello
```

这里 `hello.cpp` 和 `-o`、`hello` 都是传给 g++ 的"补充说明"，我们管它叫**命令行参数**。那么问题来了——程序怎么读到这些参数呢？

答案是，`main` 函数也能接收参数进来！标准规定 `main` 可以有两种形式：

```cpp
int main()                       // 不需要参数时
int main(int argc, char* argv[]) // 需要命令行参数时
```

第二种形式接收两个参数：

- **`argc`**（argument count）：参数的个数，包含程序名本身在内。
- **`argv`**（argument vector）：一个字符串数组，每个字符串是一个参数。

还是用刚才的编译命令举例：

```bash
g++ hello.cpp -o hello
```

对 `g++` 程序来说：

| `argc` | `argv[0]` | `argv[1]` | `argv[2]` | `argv[3]` |
| ------ | --------- | --------- | --------- | --------- |
| 4 | `g++` | `hello.cpp` | `-o` | `hello` |

接着看一个简单用法：

```cpp
#include <iostream>
using namespace std;

int main(int argc, char* argv[])
{
    cout << "参数个数: " << argc << endl;
    for (int i = 0; i < argc; ++i)
        cout << "argv[" << i << "]: " << argv[i] << endl;
    return 0;
}
```

编译后执行：

```bash
./my_program apple banana orange
```

输出：

```
参数个数: 4
argv[0]: ./my_program
argv[1]: apple
argv[2]: banana
argv[3]: orange
```

看，我们自己写的程序也能接收参数了。以后写工具程序的时候，这招用得非常频繁。

> 顺便一提，`char* argv[]` 这种写法其实是在说"`argv` 是一个数组，里面每个元素都是 `char*`（C 风格字符串的指针）"。指针的事我们后面会细讲，现在只要先知道 argv 存的就是一个个参数字符串就行。

------

**main 是特殊的，但你不需要特殊对待它**

`main` 虽然地位崇高——它是程序的起点，是操作系统直接调用的函数——但从写法上看，它跟普通函数没有本质区别。它也能调用别的函数，也能写循环、写判断、使用标准库；它唯一的"特权"大概也就是可以不写 `return` 而编译器不生气了。

请把 `main` 好好利用起来：把它当成调度台，让它协调调用各个功能函数，而不是把所有业务逻辑一团脑全塞进去。一个写得干净的 `main` 应该像这样：

```cpp
int main()
{
    init();        // 初始化
    run();         // 主逻辑
    cleanup();     // 收尾
    return 0;
}
```

简简单单，清清楚楚。复杂的活，让小弟函数去干。
