# 可供理解的数据 · 字符串库

在第二章我们见过 C 风格字符串——就是那种用双引号括起来、末尾藏一个 `\0` 的 `char` 数组。它能用，但用到后面你会越来越烦：

- 拼接？没有 `+`，得调 C 库函数 `strcat`，还得时刻注意目标数组够不够大。
- 比较大小？`==` 比的是地址指针不是内容，得用 `strcmp`。
- 查长度？`strlen` 需要从开头一直扫描到 `\0` 为止，性能随时间线性增长。
- 不小心末尾少了个 `\0`？这就是越界读写，轻则乱码，重则崩溃。

标准库的 `std::string` 把上面这些烦心事全替你解决了。藏在 `<string>` 头文件里。

------

## 创建字符串

```cpp
#include <iostream>
#include <string>

int main()
{
    string s1;                           // 空字符串
    string s2 = "Hello, C++ World!";      // 从字面量初始化
    string s3(s2);                        // 从另一个 string 拷贝
    string s4(10, '=');                   // 十个 '=' → "=========="

    cout << s4 << "\n";
    return 0;
}
```

直接把 `string` 对象丢给 `cout` 就能输出，不用管底层 `\0`，不用担心越界。

------

## 拼接

和 C 风格字符串最大的体验差异就是拼接：

```cpp
string first = "Hello";
string second = "World";

string full = first + " " + second + "!";
cout << full << "\n";    // Hello World!

// 还可以 +=
full += "\n---- 再加一行 ----";
```

`+` 运算符两边只要有一个是 `string`，另一边可以是 `string`、`const char*` 或 `char`，编译器自己搞定转换。

但是——**不能把两个纯 C 风格字符串相加**：

```cpp
// string s = "Hello" + "World";    // ❌ 两个都是 const char*，没有重载的 +
string s = "Hello" + string("World");  // ✅ 至少有一个是 string
```

------

## 查长度、判空、访问字符

```cpp
string s = "C++ World";

cout << s.size() << "\n";     // 9
cout << s.length() << "\n";   // 9，跟 size() 是一回事

if (!s.empty())
    cout << "字符串非空\n";

cout << s[0] << "\n";         // 'C'
cout << s.front() << "\n";    // 'C'
cout << s.back() << "\n";     // 'd'
cout << s.at(3) << "\n";      // ' '（带越界检查）

s[0] = 'J';                       // 修改字符
cout << s << "\n";           // J++ World
```

`[]` 和 `at()` 的区别只在一个地方——越界时 `[]` 不报错（未定义行为），`at()` 会抛出 `std::out_of_range` 异常。调试阶段优先用 `at()`，确保没问题了再换 `[]` 跑速度。

------

## 增删改

```cpp
string s = "World";

s.insert(0, "Hello ");          // 在第 0 位前插入 → "Hello World"
s.append("!!!");                // 尾部追加 → "Hello World!!!"
s += "?";                       // 和 append 一样 → "Hello World!!!?"

s.erase(5, 6);                  // 从第 5 位开始删 6 个字符 → "Hello!!?"
s.replace(5, 3, " C++");       // 把第 5 位起的 3 个字符替换为 " C++" → "Hello C++"

s.clear();                      // 清空
```

注意 `insert`、`erase`、`replace` 操作的第一个参数都是**索引位置**（从 0 开始），第二个参数是**要操作的字符个数**。

------

## 查找：从里面找到目标

```cpp
string s = "Hello, C++ World!";

auto pos = s.find("C++");         // 返回第一次出现的位置，这里是 7
if (pos != string::npos)     // npos = "not found"  
    cout << "在位置 " << pos << " 找到了\n";

auto pos2 = s.rfind("l");         // 从右往左找 → 14（最后一个 l）
auto pos3 = s.find_first_of("aeiou");   // 找到第一个元音字母 → 1（e）
auto pos4 = s.find_last_of("aeiou");    // 找到最后一个元音字母 → 12（o）
```

**注意**：`find` 找不到时返回 `std::string::npos`（一个非常大的无符号整数），不是 -1。初学者常写 `if (pos < 0)` 来判空——在无符号类型下这永远是 false。正确的写法只有一个：`if (pos != std::string::npos)`。

------

## 提取子串

```cpp
string s = "Hello, C++ World!";
string sub = s.substr(7, 3);     // 从位置 7 开始取 3 个字符 → "C++"
string rest = s.substr(7);       // 从位置 7 取到末尾 → "C++ World!"
```

## 数字和字符串互转

C++11 引入了一套简短直观的转换函数，就在 `<string>` 里：

```cpp
string s1 = to_string(42);          // "42"
string s2 = to_string(3.14159);     // "3.141590"

int a       = stoi("42");                // string → int      → 42
double d    = stod("3.14159");           // string → double   → 3.14159
long double ld = stold("1.23e10");       // string → long double
```

`to_string` 对浮点数默认输出 6 位小数。如果你需要更精确的格式化控制（比如只保留 2 位），用 `ostringstream` 配合 `<iomanip>`（后面第 13 章讲）。

## C 风格字符串 ↔ string 互转

经常需要在老 API（只认 `const char*`）和新代码之间架桥：

```cpp
// string → C 风格
string s = "Data";
const char* p = s.c_str();             // 拿到只读的 C 风格字符串
// 注意：p 在 s 被修改或销毁后就失效了

// C 风格 → string
const char* text = "Hello from C!";
string s2(text);                  // 直接构造
string s3 = text;                 // 直接赋值也行
```

`c_str()` 和 `data()` 返回的都是指向 `string` 内部缓冲区的指针，不能通过它们修改内容，也不能在 `string` 对象被销毁后还拿着用。

------

`string` 就是把 C 语言里需要十行代码的字符串操作变成一行、而且永远不会越界的现代工具。凡是你需要跟文本打交道的地方——日志、配置文件、用户输入、序列化——都应该首选 `string` 而不是 `char[]`。两者的速度在绝大多数场景下已经没有可感知的差距，而 `string` 能替你消灭掉的最大隐患——缓冲区溢出——却是 `char[]` 永远无法磨灭的伤疤。
