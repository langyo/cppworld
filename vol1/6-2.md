# 可供理解的数据 · 字符串库

在前几章里，我们用过 C 风格字符串——就是那种用双引号括起来、末尾带个 `\0` 的 `char` 数组。它能用，但用得越深入越心累：

- 拼接？没有 `+`，得用 C 库函数 `strcat`。
- 比大小？`==` 不管用，得用 `strcmp`。
- 查长度？得用 `strlen`，但这东西要遍历到 `\0` 为止，慢。
- 一不小心忘了 `\0`？内存就溢出了。

标准库给了我们一个新的选择：`std::string`。它是为处理字符串而生的类，把上面这些烦心事都给包圆了。

------

**Hello, string!**

用 `string` 之前要先引头文件 `<string>`：

```cpp
#include <iostream>
#include <string>
using namespace std;

int main()
{
    string str = "Hello, C++ World!";
    cout << str << endl;
    return 0;
}
```

直接输出，不用操心 `\0`，不用担心数组越界。这才是现代 C++ 处理字符串该有的样子。

------

**创建字符串**

```cpp
string s1;                    // 空字符串
string s2 = "Hello";          // 从 C 风格字符串初始化
string s3("Hello");           // 另一种等价的写法
string s4 = s3;               // 从另一个 string 拷贝
string s5(10, '=');           // 十个 '=' → "=========="
```

常用的大概就这几种。

------

**拼接字符串**

在 C++ 中，`string` 可以用 `+` 号直接拼接，这是 C 风格字符串想都不敢想的：

```cpp
string first = "Hello";
string second = "World";
string full = first + " " + second + "!";
cout << full << endl;    // Hello World!
```

`+` 运算符的两边，只要有一个是 `string`，另一个就可以是 `string`、字符、C 风格字符串中的任何一种。

但注意：不能把两个纯粹的 C 风格字符串直接相加：

```cpp
string s = "Hello" + "World";       // ❌ 两个都是 C 风格字符串
string s = "Hello" + string("World"); // ✅ 有一个是 string 就行
```

------

**常用操作一览**

下面这些操作可以说是日常必须：

```cpp
string s = "C++ World";

s.size();        // 返回字符串长度 => 9
s.length();      // 同 size()，完全一样的东西
s.empty();       // 空字符串返回 true
s[0];            // 访问第 1 个字符 => 'C'
s.front();       // 访问首字符 => 'C'
s.back();        // 访问末字符 => 'd'
s.at(3);         // 访问第 4 个字符 => ' '（带越界检查）

s.clear();       // 清空字符串

s += "!!!";      // 尾部追加（等价于 append）
s.insert(0, "Hi, "); // 在位置 0 处插入
s.erase(0, 4);   // 从位置 0 开始，删除 4 个字符
s.replace(0, 3, "Java"); // 把前 3 个字符替换成 "Java"
```

`[]` 和 `at()` 的区别：`[]` 越界了不检查，属于未定义行为；`at()` 越界了会抛异常，更安全但稍慢。调试期间优先用 `at()`，确认无误后再用 `[]` 追求效率。

------

**字符串比较**

`string` 直接用 `==`、`!=`、`<`、`>` 等比较运算符，比较的是字典序：

```cpp
string a = "apple";
string b = "banana";

if (a < b) cout << "apple 的字典序在 banana 前面" << endl;  // 成立
```

内部逐字符比 ASCII 码，跟字典的排列逻辑一致。

另外还有一个 `compare()` 函数，返回 0（相等）、正数（大于）、负数（小于）。不过实际写代码时，`==` 和 `<` 就够用了，`compare()` 更常用于很精细的局部比对：

```cpp
string s = "abcdef";
s.compare(0, 3, "abc");  // 把 s 的前 3 个字符跟 "abc" 比较 => 0（相等）
```

------

**查找和提取**

```cpp
string s = "Hello, C++ World!";

s.find("C++");      // 返回第一个匹配的位置 => 7
s.rfind("l");       // 从右往左找 => 13（最后一个 l）
s.find_first_of("aeiou"); // 找到第一个元音字母的位置 => 1
s.find_last_of("aeiou");  // 找到最后一个元音字母的位置 => 13

s.substr(7, 3);     // 从位置 7 开始取 3 个字符 => "C++"
s.substr(7);        // 从位置 7 一直取到末尾 => "C++ World!"
```

如果 `find` 没找到，它不会返回 0 或 -1，而是返回一个特殊常量 `string::npos`。因此查找的时候要这样写：

```cpp
if (s.find("C++") != string::npos)
    cout << "找到了！" << endl;
```

这是一个比较容易掉的坑——初学者常以为找不到就返回 -1，实际上在 `size_t` 无符号类型下，-1 会变成一个天文数字，跟 `npos` 直接比才是正确姿势。

------

**转换：数字 ↔ 字符串**

C++11 起，标准库提供了一套简洁的转换函数：

```cpp
string s1 = to_string(42);         // "42"
string s2 = to_string(3.14159);    // "3.141590"
string s3 = to_string(0xABCD);     // 直接转十六进制的十进制值

int a        = stoi("42");         // string to int    → 42
double d     = stod("3.14159");    // string to double → 3.14159
long double ld = stold("1.23e10"); // string to long double
```

这套函数藏在 `<string>` 里，不需要别的头文件。

------

**C 风格字符串 ↔ string 互转**

有时候你免不了要跟老代码打交道——那些还在用 `char*` 的 API：

```cpp
// string → C 风格字符串
string s = "Data";
const char* p = s.c_str();   // 拿到一个只读的 C 风格字符串

// C 风格字符串 → string
const char* text = "Hello from C!";
string s2 = text;              // 直接赋值就行
string s3(text);               // 构造函数也行
```

注意 `c_str()` 返回的是 `const char*`（只读），你不能拿它去修改原字符串的内容。
