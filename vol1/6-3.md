# 无限车厢 · 向量库

前面我们学过了数组——一排带编号的小格子，每个格子装一个元素。老实是老实，但有个死穴：**大小在出生那一刻就定死了，一辈子都改不了**。

你声明了 `int arr[50]`，那它这辈子就只能装 50 个元素。你想装 51 个？加不进去，换个更大的数组重新搬。这在实际编程中简直是灾难——我怎么知道用户会输多少行数据？我怎么知道文件有多大？

标准库有解决方案：`std::vector`。它依然是"一列带编号的小格子"，但你随时可以在末尾往上添格子，或者从末尾撤掉格子。它的英文原意是"向量"，但你可以把它理解成**一列无限长的火车车厢——想挂多少节、拆多少节，随你。**

------

**Hello, vector!**

用 `vector` 要引头文件 `<vector>`：

```cpp
#include <iostream>
#include <vector>
using namespace std;

int main()
{
    vector<int> v = {1, 2, 3, 4, 5};

    cout << v[2] << endl;   // 输出 3（第三个元素，下标是 2）

    return 0;
}
```

`vector<int>` 表示"这列火车只能拉整数"。`int` 在这里是**模版参数**（后面会学），可以换成任何类型——`double`、`string`、甚至你自己定义的类型。

------

**创建 vector**

```cpp
vector<int> v1;                     // 空 vector，一个元素都没有
vector<int> v2(10);                 // 10 个元素的 vector，全部被默认初始化为 0
vector<int> v3(10, 42);             // 10 个元素，全部都是 42
vector<int> v4 = {1, 2, 3, 4, 5};   // 用初始化列表塞 5 个元素
vector<int> v5(v4);                 // 从 v4 拷贝一份
vector<int> v6 = v4;                // 同上，另一种等价写法
```

------

**添、删、查、改**

这才是 `vector` 真正的威力所在：

```cpp
vector<int> v;
v.push_back(10);          // 在末尾追加 10
v.push_back(20);          // 在末尾再追加 20
v.push_back(30);          // 在末尾再追加 30
// 现在 v = {10, 20, 30}

v.pop_back();             // 把末尾的 30 摘掉
// 现在 v = {10, 20}

v[0] = 100;               // 把第一个元素改成 100
// 现在 v = {100, 20}

cout << v.size() << endl; // 当前有几个元素 => 2
cout << v[0] << endl;     // 看第一个元素 => 100
```

注意 `size()` 告诉你的是**当前实际有几个元素**，而不是"能装几个"（那是 `capacity()` 的事，以后再说）。

------

**遍历 vector**

方法有好几种，从原始到优雅：

```cpp
vector<int> v = {10, 20, 30, 40, 50};

// 方法一：传统 for 循环
for (int i = 0; i < v.size(); ++i)
    cout << v[i] << " ";

// 方法二：范围 for（C++11 起）
for (int x : v)
    cout << x << " ";

// 如果不想拷贝里面的东西（元素大的时候），用引用
for (const int& x : v)    // 常引用：速度又快，又不会意外修改
    cout << x << " ";
```

推荐在不需要修改元素的情况下用方法二的常引用版本。

------

**vector vs 数组：什么时候用什么**

`vector` 这么好用，是不是数组就该退役了？并不是。

- **数组适合**：数量绝对确定不变的小数据（比如棋盘格数、月份数 12 个月）。
- **vector 适合**：数量不确定、要动态增删的场景（大多数实际情况）。

此外，数组的效率在极端场景下要略高于 `vector`（因为没有动态开销），但这种差异在 99% 的场景下根本感觉不到。**缺省情况下直接用 `vector`，除非你有非常充足的理由用数组。**

------

**越界访问：一个不会放过你的坑**

```cpp
vector<int> v = {1, 2, 3};
cout << v[5] << endl;         // 越界！未定义行为——程序可能崩，可能不崩但读到了垃圾数据
cout << v.at(5) << endl;      // 越界！但 at() 会抛出 out_of_range 异常，程序能体面处理
```

数组的 `[]` 和 `vector` 的 `[]` 都不做越界检查。如果你不确定下标是否合法，就用 `at()` ——它会帮你兜底。

------

**一些常用的小操作**

```cpp
v.empty();           // 是不是空的？
v.front();           // 第一个元素
v.back();            // 最后一个元素
v.clear();           // 清空所有元素
v.resize(5);         // 把大小改成 5（多的丢掉，少的补默认值）
v.reserve(100);      // 预留能装 100 个元素的空间（不改变 size，只是提前申请内存）
```

`reserve` 是常见的优化手段：如果你已经知道未来要往 vector 里塞大约 1000 个元素，提前调用 `reserve(1000)`，就可以避免在塞的过程中反复重新申请内存（后者会导致性能抖动）。

------

`vector` 是我们学到的第一个标准库容器，但不是唯一一个。在第八章还会遇到 `list`、`deque`、`map`、`set` 等更多容器。但可以这么说——在所有容器中，`vector` 的出镜率至少占了七八成。把它用熟，你就算踏入标准库的大门了。
