# 无限车厢 · 向量库

我们很早就学会了数组——一排带编号的小格子，每个格子装一个同类型的元素。可靠是可靠，但有一个死穴：**大小在生下来的那一刻就定死了，终身不能改变。**

`int arr[50]` 这辈子就只能装 50 个元素。用户要输入 51 个数据怎么办？文件比预想的大怎么办？你只能宣告旧数组已废、重新分配、全部搬过去——三个步骤全是手工活且随时出人为失误。

标准库的 `std::vector` 完美解决了这个痛点。它依然是一排带编号的小格子，但你随时可以在一端接上新车厢、或者拆掉末尾的旧车厢。它的英文原意是"向量"，但你可以把它比作一列**无限可伸缩的火车**——需要多长就挂多少节。

------

## 创建与初始化

```cpp
#include <iostream>
#include <vector>

int main()
{
    vector<int> v1;                        // 空 vector，一个元素都没有
    vector<int> v2(10);                    // 10 个元素，全初始化为 0
    vector<int> v3(10, 42);                // 10 个元素，全是 42
    vector<int> v4 = {1, 2, 3, 4, 5};      // 初始化列表
    vector<int> v5(v4);                    // 从另一个 vector 拷贝
    vector<int> v6 = v4;                   // 同上，写法不同
}
```

`vector<int>` 的 `<int>` 是模板参数——表示"这列火车只能拉整数"。换成 `double`、`string`、甚至你自己定义的类都可以，语法一样的。

------

## 核心操作：增删查改

```cpp
#include <iostream>
#include <vector>

int main()
{
    vector<int> v;

    v.push_back(10);            // 尾部追加 10  → {10}
    v.push_back(20);            // 尾部追加 20  → {10, 20}
    v.push_back(30);            // 尾部追加 30  → {10, 20, 30}
    v.pop_back();               // 尾部摘掉 30  → {10, 20}

    v[0] = 100;                 // 修改第 0 个 → {100, 20}

    cout << "当前有 " << v.size() << " 个元素\n";     // 2
    cout << "第一个: " << v[0] << ", 第二个: " << v[1] << "\n";
}
```

注意 `size()` 返回的是**现在实际有多少个元素**，不是"最多能装几个"（那是 `capacity()` 的事，一般不用管）。

------

## 遍历：几种姿势

```cpp
vector<int> v = {10, 20, 30, 40, 50};

// 经典下标 for
for (size_t i = 0; i < v.size(); ++i)
    cout << v[i] << " ";

cout << "\n";

// 范围 for（C++11）——最简洁
for (int x : v)
    cout << x << " ";

cout << "\n";

// 常引用范围 for——不拷贝，不改内容，最快
for (const int& x : v)
    cout << x << " ";
```

如果不需要修改元素，按照"常引用范围 for"写最稳妥。如果是大对象（如 `vector<string>`），不用引用会造成每个元素都拷贝一遍，又慢又浪费。

------

## 常用小工具

```cpp
vector<int> v = {1, 2, 3};

bool empty = v.empty();         // 是不是空的？
int first = v.front();          // 第一个元素
int last = v.back();            // 最后一个元素
v.clear();                      // 全部清空

v.resize(5);                    // 把大小改成 5（多的扔掉，少的用默认值补）
v.reserve(100);                 // 提前预定 100 个位置的空间（不改变 size，只分配容量）
```

`reserve` 是一个优化技巧：如果你提前知道这个 vector 以后要塞大约 10000 个元素，在一开始就 `reserve(10000)`。否则每次 `push_back` 到容量不够时，vector 都要在背后悄悄重新分配一块更大的新内存、把旧数据全部搬过去——这个反复搬家会浪费不少时间。先 reserve 了，后面的 push_back 就是纯内存写入，速度拉到极致。

------

## 做一个实际工具：从用户输入中收集、排序、输出

把上面学的串起来——写一个程序，让用户不断输入整数存进 vector，直到输入 0 为止，排序后倒序输出：

```cpp
#include <iostream>
#include <vector>
#include <algorithm>

int main()
{
    vector<int> data;
    int value;

    cout << "输入整数（输入 0 结束）:\n";
    while (cin >> value && value != 0)
        data.push_back(value);

    sort(data.begin(), data.end());          // 排序

    cout << "排好序的结果（从小到大）:\n";
    for (int x : data)
        cout << x << " ";

    cout << "\n\n从大到小:\n";
    for (auto it = data.rbegin(); it != data.rend(); ++it)   // 反向遍历
        cout << *it << " ";

    cout << "\n";
    return 0;
}
```

这就是 `vector` 在实战中的日常用法——读入未知数量的一组东西、放进去、做一顿批量操作。它替你承担了扩容、释放、越界检查的脏累活，你只需要关心干什么。

------

## vector vs 数组：选哪个

对于刚学 C++ 的人，一条简单的规则就足够：**能用 `vector` 就不用原生数组。** 两者的速度在绝大多数现实场景下没有可感知的差异，而 `vector` 能消灭掉你写出越界访问、忘记释放、传参时"掉帽子"退化这些 bug 的机会。如果大小在编译期就能确定且你想要容器接口（`size()`、迭代器、`at()`），就用 `std::array`；否则一律用 `vector`。
