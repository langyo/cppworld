# 剥皮的技巧 · 数组与成员访问

先来设想一个场景：

> 你在《Minecraft》里有一个大箱子，里头乱七八糟塞了几十样物品。想从中翻出一个钻石镐？晕了吧。

我们需要的不是一个大箱子，而是一排带编号的小格子——每个格子里放一样东西，想拿哪一个，直接按编号找就是了。

在 C++ 中，我们管这样的"一排小格子"叫**数组**。

------

> 0

没错，又是老朋友数字零。

还记得上一章我们学了什么？类型、变量、运算符。假如我们要存 5 个学生的成绩，最笨的办法是这样：

```cpp
int s0 = 85;
int s1 = 92;
int s2 = 78;
int s3 = 88;
int s4 = 95;
```

五个变量，还算凑合。那如果是 50 个学生呢？500 个呢？一个一个声明，手都写断了。

数组就是为了解决这个问题而生的。它允许我们一次声明一组类型相同的数据，并且用**编号**（专业术语叫**下标**或**索引**）来访问其中的每一个：

```cpp
int scores[5] = {85, 92, 78, 88, 95};
```

这条声明做了什么？它告诉编译器："给我在内存中划一块能装 5 个 `int` 的空间，初始化成这五个值。"

现在，如果你想拿到第二个学生的成绩，这样写就行了：

```cpp
cout << scores[1] << endl;  // 输出 92
```

等等，第二个学生……编号是 `1`？为什么不是 `2`？这里有一个非常非常非常重要的坑，我们待会细讲。

------

**从零开始的编号**

绝大多数时候，我们数数都是从 1 开始的：第 1 个、第 2 个、第 3 个……

但在 C++ 的世界里，数组的下标是从 `0` 开始的。

也就是说：

| 学生 | 在数学中是 | 在 C++ 中下标是 |
| ---- | ---------- | --------------- |
| 第一个 | 第 1 个 | `0` |
| 第二个 | 第 2 个 | `1` |
| 第三个 | 第 3 个 | `2` |
| 第四个 | 第 4 个 | `3` |
| 第五个 | 第 5 个 | `4` |

所以 `scores[0]` 是第一个学生的成绩 `85`，`scores[1]` 是第二个学生的成绩 `92`，`scores[4]` 是最后一个学生的成绩 `95`。

为什么会这样？因为计算机把数组当成了**从某个地址开始的连续内存**。`scores[0]` 的意思是"从起点位置往后 0 个位置"，也就是起点本身；`scores[1]` 是"从起点往后 1 个位置"，以此类推。

刚开始学确实容易搞混，但习惯之后你会觉得从零开始也挺自然的——就像地上一层楼在有些国家叫"1 楼"，在另一些国家叫"0 楼"（地面层）一样。

------

**声明数组的几种姿势**

声明的数组并不一定要马上填好内容。你也可以先开好空间，后续再往里填：

```cpp
int arr[10];           // 声明了 10 个 int 的空间，但里面的值是不确定的（垃圾值）
int arr2[10] = {};     // 声明并全部初始化为 0
int arr3[10] = {1, 2}; // 前两个是 1 和 2，其余全部自动填 0
int arr4[] = {1, 2, 3, 4, 5};  // 编译器自动推断数组大小为 5
```

按需选择姿势即可。最常用的是第四种——懒得数有几个东西，让编译器自己算去。

------

数组里装什么东西都行，只要是同一类型：

```cpp
char   name[10] = {'H', 'e', 'l', 'l', 'o'};
double prices[50];
bool   flags[100] = {};
```

不过呢，数组有个不灵活的地方——它的大小在声明时就定死了，后面不能改。比如你声明了一个 `int scores[50]`，那这个数组一辈子最多只能存 50 个元素，多一个都塞不下。超出范围去访问（比如 `scores[50]`）是**未定义行为**——程序可能会崩溃，也可能不崩溃但默默搞坏了别的数据，总之别这么干。

想用能随时加长缩短的"动态数组"？别急，等我们学到标准库的 `vector` 再说。

**访问数组成员：`.` 运算符**

数组的下标用中括号 `[]` 来写。但 C++ 中还有另一种访问成员的方式——使用点号 `.`。

什么时候用点号？当你手中的数据是"一捆属性"而不是"一串相同的东西"时。

比如我们之前碰到过的结构体：

```cpp
struct Point
{
    int x;
    int y;
};

Point p;
p.x = 3;    // 用 . 运算符访问 p 的成员 x
p.y = 5;    // 用 . 运算符访问 p 的成员 y
```

`p.x` 翻译成人话就是："p 这个东西里头的 x 那个部分"。

**总结：数组用 `[]`，结构体/类用 `.`。**

那数组里的每个元素本身也是某种类型，如果数组存的是结构体呢？两种运算符就可以组合使用了：

```cpp
Point points[10];          // 10 个 Point 的数组
points[0].x = 100;         // 第 1 个 Point 的 x 成员设为 100
points[0].y = 200;         // 第 1 个 Point 的 y 成员设为 200
```

从外到内看：`points` 是个数组，`points[0]` 是它的第一个元素（一个 `Point`），`.x` 是这个 `Point` 里的成员 `x`。一层一层剥开，像剥洋葱。

**C 风格字符串：字符数组的特殊形态**

下面这个东西看着眼熟吗？

```cpp
char str[] = "Hello";
```

答案是：它就是字符数组的简写形式，等价于这样写：

```cpp
char str[] = {'H', 'e', 'l', 'l', 'o', '\0'};
```

注意最后面多了一个 `'\0'`。这个看不见的小尾巴叫作**空字符**，它是 C 风格字符串的"结束标志"——计算机靠它来知道字符串读到哪里该停。没有它，程序会一直读下去，读到不该读的内存为止。

> 这种以 `\0` 结尾的字符数组，正式名称叫**空终止字节字符串**（Null-terminated byte string），但我们一般就管它叫 **C 风格字符串**。

C 风格字符串一直可以用 `cout` 直接输出：

```cpp
char message[] = "C++ 是世界的主宰！";
cout << message << endl;
```

但它毕竟是 C 语言时代的遗产，容易出错，也没有现成的长度查询、拼接等功能。到了第六章，我们会介绍 C++ 标准库的 `string` 类——那才是现代 C++ 处理字符串的正道。
