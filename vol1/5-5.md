# 自助服务 · 默认实参与函数指针

上一节我们见识了函数的"分身术"——重载。今天我们来学另外两项绝活：一是给参数设默认值（省得调用方每次都要填），二是把函数本身当数据传来传去（把行为当作参数）。

------

**默认实参：不爱填？我帮你填**

有时候，某个参数在绝大多数情况下都取同一个值。比如：

```cpp
void drawCircle(int x, int y, int radius, int color);
```

画圆的时候，颜色九成都是黑色——但要你每次都写 `drawCircle(100, 200, 50, BLACK)`，写多了也烦。我们直接给 `color` 设个默认值：

```cpp
void drawCircle(int x, int y, int radius, int color = BLACK);

// 调用的时候，可以给颜色，也可以不给：
drawCircle(100, 200, 50, RED);   // 用红色画
drawCircle(100, 200, 50);        // 没填颜色，那就用默认的 BLACK
```

这个 `BLACK` 就叫**默认实参**。调用方可以不填，编译器会自动拿默认值补上。

几个要点：

1. **默认实参只能往右靠**：一旦某个参数设了默认值，它右边的参数**也必须**有默认值。

```cpp
void f(int a, int b = 1, int c = 2);   // ✅ 可以
void f(int a = 1, int b, int c = 2);   // ❌ b 夹在中间没默认值，不行
```

想想也合理——调用的时候总不能跳过中间的参数去填最后一个吧？

2. **默认实参写在声明中，不是定义中**。如果你分开声明和定义，默认值只放在声明里，定义里不要重复。

```cpp
void drawCircle(int x, int y, int radius, int color = BLACK); // 声明中写

void drawCircle(int x, int y, int radius, int color)           // 定义中不写
{
    // 实际的画圆代码……
}
```

3. **同一个函数的不同声明中，同一个参数只能赋一次默认值**。不能在一个声明里给 `a` 赋 `0`，在另一个声明里又赋 `1`。

------

**默认实参与重载的互动**

默认实参和重载是可以一起用的，但要防二义。

```cpp
void print(int value, int base = 10);   // 版本 1
void print(int value);                   // 版本 2
```

当你写 `print(42)` 时，编译器陷入了两难：既可以调版本 1（用默认的 base=10），也可以调版本 2。两个都说得通，于是编译器罢工——报二义性错误。

解决方法就是：别让重载和默认实参同时覆盖同一种调用形式。要么去掉其中一个版本，要么调整参数让它们各自独一无二。

------

**函数指针：把函数当数据用**

函数本身也能被视为一种"数据"——可以被存进变量，可以被当成参数传给另一个函数，甚至可以被当成返回值拿到手。

怎么把一个函数的"地址"取出来？用**函数指针**。

```cpp
int add(int a, int b) { return a + b; }

// 声明一个指向"接收两个 int、返回 int"的函数的指针
int (*p)(int, int);

p = add;               // 把 add 函数的"住址"交给 p
int result = p(3, 5);  // 通过指针调用函数 => 8
int result2 = (*p)(3, 5); // 另一种等价写法
```

`int (*p)(int, int)` 从左往右读：`*p` 说明 p 是个指针；`(int, int)` 说明它指向的东西是函数，且接收两个 int；`int` 说明这个函数返回 int。最外侧那对括号千万不能省——如果写成 `int* p(int, int)` 就变成"声明一个返回 `int*` 的函数"了，完全不是一个意思。

------

**函数指针做参数：把行为当参数传**

函数指针最常见也最有用的用法是**作为另一个函数的参数**。它让你可以把"一段行为"传进函数里。

举个例子——假设你要给一大堆数字做点"什么事"，但不清楚到底要做什么：

```cpp
// 对数组中的每个元素做点什么
void forEach(int* arr, int size, void (*action)(int&))
{
    for (int i = 0; i < size; ++i)
        action(arr[i]);
}

// 具体的行为由调用者决定
void doubleIt(int& x) { x *= 2; }
void squareIt(int& x) { x *= x; }
void printIt(int& x)  { cout << x << " "; }

int main()
{
    int data[] = {1, 2, 3, 4, 5};

    forEach(data, 5, doubleIt);  // 每个数翻倍
    forEach(data, 5, printIt);   // 打印每个数
    forEach(data, 5, squareIt);  // 每个数平方

    return 0;
}
```

多妙——`forEach` 本身跟具体的行为完全无关，它只负责"遍历"。至于遍历过程中要干什么，由传入的函数指针说了算。这就是所谓的**策略模式**在 C 语言时代的朴素实现。

------

**函数指针看起来有点丑，怎么办**

不得不承认，函数指针的写法确实不太好看。`int (*p)(int, int)` 这种造型让很多人看了头大。别担心，C++ 提供了两种更清爽的办法：

**办法一：`typedef` 给函数指针起别名**

```cpp
typedef int (*TwoIntFunc)(int, int);   // 起个名字
TwoIntFunc p = add;                     // 用起来和普通变量差不多
p(3, 5);
```

**办法二：`using` 起别名（C++11 起）**

```cpp
using TwoIntFunc = int(*)(int, int);   // 更接近变量的写法
TwoIntFunc p = add;
```

如果还是觉得别扭，别急——后面我们会学到 `std::function`，它会用一种更接近普通变量的写法来封装所有"能被调用的东西"。等我们再往后学 Lambda 表达式时，函数式编程的味道就更浓了。

------

**返回值也可以是函数指针**

这写法看着有点惊悚，但不难理解：

```cpp
// getOperation 返回一个指向"两个 int 入、一个 int 出"的函数的指针
int (*getOperation(char op))(int, int)
{
    switch (op)
    {
        case '+': return add;
        case '-': return subtract;
        default:  return nullptr;
    }
}

int main()
{
    auto op = getOperation('+');
    cout << op(100, 200) << endl; // 300
}
```

当然这写法也太丑了，通常我们会用 `typedef` 或 `using` 润色一下：

```cpp
using FPtr = int(*)(int, int);

FPtr getOperation(char op)
{
    if (op == '+') return add;
    return nullptr;
}
```

顺眼多了吧。善用别名，是驯服复杂类型的好习惯。
