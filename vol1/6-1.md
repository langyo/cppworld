# 军械库 · 初识标准库

到此为止，我们用的东西大都靠自己手写。但 C++ 之所以强大，并不仅仅因为它是个语言——而是因为它带了一个超大规模、功能齐全的**标准库**。

如果把 C++ 语言本身比作一套五金工具，那标准库就是一个塞满了各种预制零件的大库房。螺丝、齿轮、导轨、传送带……人家都造好了，你拿起来就能用，完全不用自己从头鼓捣。

标准库是 C++ 不可分割的一部分。不会用标准库的 C++ 程序员，等于白学了一半。

------

**标准库里都有些什么**

我们来快速扫一眼：

| 分类 | 包含内容 |
| ---- | -------- |
| 输入输出 | `cin`、`cout`、文件读写等 |
| 字符串 | `string` 类——比 `char[]` 好用一万倍 |
| 容器 | `vector`、`map`、`set` 等——装东西的篮子 |
| 算法 | 排序、查找、统计——人家都写好了 |
| 数学计算 | 绝对值、三角函数、对数、平方根…… |
| 日期时间 | 计时的、计日的、计秒的 |
| 内存管理 | 智能指针——忘了 `delete` 也不怕内存泄漏 |
| 线程 | 并发编程——让程序"一心多用" |

是不是有点眼花缭乱？别慌，标准库虽大，但我们在前几章就不知不觉地用过其中不少了。

------

**你已经用过的东西**

从第一章第一天开始，你就一直在用标准库。回忆一下：

```cpp
#include <iostream>        // 输入输出库
std::cout << "Hello!" << std::endl;  // cout 和 endl 都是标准库的对象
std::cin >> variable;                  // cin 也是
```

`iostream` 是标准库中负责控制台输入输出的头文件。它给了我们四个标准流对象：

| 对象 | 全称 | 用途 |
| ---- | ---- | ---- |
| `cin` | Console In | 标准输入 |
| `cout` | Console Out | 标准输出 |
| `cerr` | Console Error | 标准错误（输出警告和错误消息） |
| `clog` | Console Log | 标准日志（输出一般性日志信息） |

你可能一直只用过 `cin` 和 `cout`，而 `cerr` 和 `clog` 也一直在那里等着——当你需要把"正常输出"和"报错消息"分开的时候，用 `cerr` 而不是 `cout` 是个好习惯。

------

**头文件：你要什么就得引什么**

标准库的各个功能分散在不同的**头文件**里。你想用什么，就要先 `#include` 对应的头文件。

一个常见的误解是胡乱写 `#include <std_all.h>` 之类的（根本没有这个文件）。C++ 的哲学是"用啥引啥"——少引一个不会让你省内存（跟最终程序大小没关系），多引一个也顶多拉长编译时间。

下面是一些最常用的头文件和它们提供的东西：

| 头文件 | 提供的内容 |
| ------ | ---------- |
| `<iostream>` | `cin`, `cout`, `cerr`, `clog` |
| `<string>` | `std::string` 类 |
| `<vector>` | `std::vector` 容器 |
| `<algorithm>` | 排序、查找、最大最小值等通用算法 |
| `<cmath>` | 数学函数——`sqrt`, `abs`, `sin`, `log`…… |
| `<cstdlib>` | 通用工具——随机数（旧版）、系统命令等 |
| `<ctime>` | 时间相关函数（C 语言风格） |
| `<fstream>` | 文件读写 |
| `<sstream>` | 字符串流 |

还有一些我们目前阶段不太可能用到的，以后随用随讲。

------

**命名空间 std**

你可能注意到了，前面章节的正文代码里都把 `std::` 给省了（在第一章时我们已经事先打过招呼）。但随例源代码中它并没有被省略。这是因为我们在文件开头写了这么一句：

```cpp
using namespace std;
```

这句的意思是：把 `std` 命名空间里的所有名字都"解封"出来，这样写 `cout` 就不用再写 `std::cout` 了。

方便归方便，**但有一个坑**：如果你随随便便写 `using namespace std;`，将来你定义的东西可能会跟标准库里的名字撞车，导致莫名其妙的编译错误。在头文件中尤其不要写这行——它会污染每一个包含了这份头文件的源文件。

比较好的做法是：

- **小项目、练习**：一上来写条 `using namespace std;`，无伤大雅。
- **大型项目**：宁可多打几个 `std::`，也不要搞出命名冲突的破事。
- **折中方案**：只释放你确实要频繁用的那几个名字——

```cpp
using std::cout;
using std::cin;
using std::endl;
using std::string;
```

这样你拿到了 `cout`、`cin`、`endl` 和 `string` 的免写前缀权，同时不把标准库全家老小都拉出来。

------

**标准库版本在演进**

标准库不是一成不变的。从 C++98 到 C++20，标准库一直在长胖。作为初学者，你不用操心版本差异——挑一个现代的编译器（GCC 10+、Clang 12+、MSVC 2019+），它们都会带着 C++17 起步的标准库。

我们会从本章接下来的各节开始，深入认识标准库的几个核心设施：字符串、容器、迭代器和算法。它们是你今后写 C++ 代码的日常伙伴，就像厨房里的刀、砧板和锅——不是每次都会全用，但缺了哪样都难受。
