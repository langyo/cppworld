# 军械库 · 初识标准库

到现在为止，你写的 C++ 程序里几乎每一行都在用标准库——`cout`、`cin`、`string`、`vector`——只是它们来得太自然，你也许没意识到它们已经打了好几回助攻了。

C++ 标准库不是语言语法的附属品，而是跟语法地位同等的半边天。不会用它，等于只会用半门语言。

------

## 标准库里有些啥

| 大类 | 包含的内容 |
| ---- | ---------- |
| 输入输出 | `cin`、`cout`、`cerr`、文件流、字符串流 |
| 字符串 | `std::string`——比 `char[]` 安全十倍 |
| 容器 | `vector`、`map`、`set`、`list`——装东西的篮子 |
| 算法 | 排序、查找、计数、最值——都是人家给你写好的 |
| 数学 | 绝对值、三角函数、对数、平方根 |
| 日期时间 | `std::chrono`——精度到纳秒的计时工具 |
| 内存管理 | `unique_ptr`、`shared_ptr`——再也不要手写 delete |
| 线程 | `std::thread`——让程序一心多用 |
| 文件系统 | `std::filesystem`（C++17）——跨平台文件操作 |
| 正则表达式 | `std::regex`——字符串模式匹配 |

初次看到这张表可能有点眼花，但别被唬住。每一样都是独立的，用到时进相应的头文件引进来就行。

------

## 你已经不自觉地用过的东西

```cpp
#include <iostream>       // 输入输出库
#include <string>         // 字符串库

int main()
{
    cout << "输入你的名字: ";
    string name;
    cin >> name;
    cout << "你好，" << name << "！\n";
}
```

三行代码里用了两个标准库——`<iostream>` 提供了 `cin`/`cout`，`<string>` 提供了 `std::string`。实际上标准库大约 80% 的内容都是在类似层级上提供成熟、安全、零开销的常用能力——你只管引对应的头文件，拿起来就用。

------

## 头文件：用啥引啥

C++ 没有"一口吞全库"的头文件。你要用 `vector`，就引 `<vector>`；要用 `sort`，就引 `<algorithm>`。这种模块化设计的代价是你需要记住常用的头文件名字，好处是不引入你用不到的功能，编译更快。

常用头文件速查：

| 头文件 | 提供的核心内容 |
| ------ | ------------- |
| `<iostream>` | `cin`, `cout`, `cerr` |
| `<string>` | `std::string` |
| `<vector>` | `std::vector` |
| `<map>` | `std::map` |
| `<set>` | `std::set` |
| `<algorithm>` | `sort`, `find`, `count`, `max`, `min` |
| `<cmath>` | `sqrt`, `abs`, `sin`, `log` |
| `<memory>` | `unique_ptr`, `shared_ptr`, `make_shared` |
| `<fstream>` | `ifstream`, `ofstream`——文件读写 |
| `<sstream>` | `istringstream`, `ostringstream`——字符串当流操作 |
| `<random>` | `mt19937` 引擎, `uniform_int_distribution` 等 |
| `<chrono>` | 时间、计时 |
| `<regex>` | 正则表达式 |
| `<thread>` | `std::thread`——多线程 |
| `<filesystem>` | C++17——目录遍历、文件复制等 |

## 命名空间 std

标准库的一切都塞在 `std` 命名空间里。写 `std::cout` 是因为 `cout` 不是全局名字，需要走进 `std` 领地才能找到。

一行 `using namespace std;` 可以把 `std` 的全部门对当前作用域敞开。在小练习、随手代码中用无妨——但大型项目和头文件中**千万别写**，会把整个全局命名区搞污染，导致你定义的变量跟数百个标准库名字撞车。

折中方案是用 `using` 声明只把你真正频繁使用的那几个名字释放出来：

```cpp
using std::cout;
using std::cin;
using std::string;
using std::vector;
```

这样你能写短名字的地方只有这四个，别的一律用 `std::`——既省了前缀，又不会无意命名冲突。
