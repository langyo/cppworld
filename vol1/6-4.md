# 游走于每一处 · 迭代器

你有过这样的经历吗？拿 `vector` 遍历元素，写 `for (int i = 0; i < v.size(); ++i)`。这在 `vector` 身上没问题——用下标访问很快。但如果你换了一个容器呢？是不是还得用下标？

答案是：不是所有容器都支持下标。比如我们以后要学的 `list`（链表）和 `set`（集合），就没法用 `v[i]` 直接跳过去拿——人家的结构决定了不能这么干。那怎么遍历？答案是**迭代器**。

------

**什么是迭代器**

迭代器（iterator）是一个统一的概念，它像一根"导航棒"，指向容器中的某个位置。你可以：

- 获取容器开头的迭代器：`begin()`
- 获取容器末尾之后那个位置的迭代器：`end()`（注意，这不是最后一个元素，是**最后一个元素的后面**）
- 把迭代器往后挪一位：`++`
- 读当前迭代器指向的元素：`*`

就好比你在玻璃陈列柜前边走边看——迭代器就是你指向某一件展品的手，你顺着走，就能依次看到每一件。

------

**最简单用法**

```cpp
#include <iostream>
#include <vector>
using namespace std;

int main()
{
    vector<int> v = {10, 20, 30, 40, 50};

    // 遍历
    for (auto it = v.begin(); it != v.end(); ++it)
        cout << *it << " ";    // *it = 拿当前指向的元素

    return 0;
}
```

图示一下：

```
  begin()                          end() (这里不是元素，是"已经出界"的位置)
     ↓                               ↓
   [10] [20] [30] [40] [50]
    ↑
  it = begin() → *it 就是 10
   之后 ++it → *it 就是 20
   ...
   当 it == end() 时就停下来
```

`end()` 不是最后一个元素 `50`，而是 `50` 的**下一位**。这看起来有点浪费，但它带来的好处是：`begin()` 到 `end()` 刚好就形成了**一个左闭右开区间**，逻辑跟很多人习惯的范围判断（`for (int i = 0; i < size; ++i)`）对得很整齐。

------

**迭代器的类型**

刚才我们写的 `auto it = v.begin()` 使用了 `auto` 关键字。那么如果不写 `auto`，迭代器到底是什么类型？

```cpp
vector<int>::iterator it = v.begin();          // 可读可写
vector<int>::const_iterator cit = v.begin();   // 只读
```

两种迭代器的区别：
- `iterator`：通过它既能读也能改元素。
- `const_iterator`：通过它只能读不能改。

如果你只是遍历看一眼元素而不修改，应该用 `const_iterator`；修改了则用 `iterator`。当然用 `auto` 的话就不用纠结了——如果你的 `vector` 是 `const` 的，`begin()` 会自动返回 `const_iterator`。

------

**迭代器不只是给 vector 用的**

迭代器的最大价值在于它的**统一性**。无论底层容器是什么——`vector`、`list`、`set`、`map`——遍历的写法都一样：

```cpp
vector<int> v = {1, 2, 3};
for (auto it = v.begin(); it != v.end(); ++it)
    cout << *it << " ";

// 将来学了 list:
list<string> words = {"hello", "world"};
for (auto it = words.begin(); it != words.end(); ++it)
    cout << *it << " ";   // 写法一模一样！
```

正是靠这种统一性，标准库里的算法（比如 `sort`、`find`）才能做到"跟容器无关"——它们只跟迭代器打交道，容器只要提供迭代器就行。

------

**迭代器的运算**

最常用的是 `++`（往下一个）和 `--`（往上一个）。`vector` 和 `string` 的迭代器还支持其它几种运算：

```cpp
auto it = v.begin();

it + 3        // 往后跳 3 个位置
it - 1        // 往前跳 1 个位置
it2 - it1     // 两个迭代器之间差多少位
*it           // 解引用，拿到当前指向的元素
it->member    // 等价于 (*it).member，访问指向元素的成员
```

不过这些额外运算不是所有容器都支持的。比如 `list` 的迭代器就只能 `++` 和 `--`，不支持直接跳到 `+5` 的位置——还那句话，链表的结构决定了它不能一步到位地跳。

迭代器根据能力不同被分了好几个等级（术语叫"迭代器类别"），我们现在先不用管这个。知道 `begin()` 可取头、`end()` 可取尾后、`++` 可前移、`*` 可取元素就够用了。

------

**迭代器失效**

这是一个必须知道的概念。在 `vector` 上做某些操作后，你之前存下的迭代器可能会"变废"。

最常见的情况是：**你在 vector 上 push_back 之后，之前的迭代器就不能再用了**。为什么？因为 push_back 可能触发重新分配内存——vector 的内脏（存储数组）整个被移到另一块地方去了，旧地址上的迭代器就指着废弃的老地方看空气了。

```cpp
vector<int> v = {1, 2, 3};
auto it = v.begin();

v.push_back(4);   // 可能导致重新分配，it 失效！
cout << *it;       // 危险！可能读到垃圾甚至崩溃
```

解决的办法就是：push_back 之后如果想要继续迭代，重新获取迭代器：

```cpp
v.push_back(4);
it = v.begin();    // 重新获取，It's safe
```

------

**范围 for 背后就是迭代器**

C++11 引入的范围 `for` 循环：

```cpp
for (int x : v)
    cout << x << " ";
```

本质上就是被编译器翻译成了迭代器的写法。它的背后和你写 `for (auto it = begin(); it != end(); ++it)` 生成的基本是一样的代码。两者可以自由切换，选你觉得更清楚的那一种。
