# 公共场所 · 类作用域与静态成员

前面几节，我们关注的都是属于**某个具体对象**的成员。但有时候，我们需要一些属于**整个类本身**的东西——不属于张三，也不属于李四，而是这个类型共有的。

比如：统计某个类已经造了多少个对象，或者为某个类设立一个所有对象共享的配置项。

C++ 中，这种需求靠**静态成员**来解决。

------

**静态成员变量：类共有的数据**

普通的成员变量是"每个对象各存一份"的。而静态成员变量是"整个类只有一份，所有对象共享的"。

声明一个静态成员变量使用 `static` 关键字：

```cpp
class Soldier
{
private:
    static int count;    // 静态成员——只声明，不定义
    int id;
public:
    Soldier()
    {
        id = ++count;    // 每造一个士兵，编号 +1
        cout << "士兵 " << id << " 已成立" << endl;
    }

    static int getCount()    // 静态成员函数——同样属于类本身
    {
        return count;
    }
};

// 在外面定义静态成员变量——必须写！而且只能在全局作用域范围写一次
int Soldier::count = 0;

int main()
{
    cout << "目前士兵数: " << Soldier::getCount() << endl; // 0

    Soldier s1;    // 士兵 1 已成立
    Soldier s2;    // 士兵 2 已成立
    Soldier s3;    // 士兵 3 已成立

    cout << "目前士兵数: " << Soldier::getCount() << endl; // 3

    return 0;
}
```

几个要注意的地方：

- **只声明不定义**：在类里写 `static int count;` 是声明。你还必须在类外的全局范围一带写 `int Soldier::count = 0;` 来定义和初始化它。漏掉这一步会导致链接错误。
- **属于类，不属于对象**：`count` 是 `Soldier` 的，不是 `s1`、`s2` 的。你可以直接通过 `Soldier::getCount()` 来访问，而不需要有任何具体的 Soldier 对象。
- 当然你也能用 `s1.getCount()` 这种对象点的方式调用静态成员函数——但从字面看反而容易让人误会它是某个对象的专属成员，所以强烈建议用 `类名::` 的方式。

------

**静态成员函数：为类而生，不为对象而生**

静态成员函数使用 `static` 标记。它和普通成员函数最大的区别：

- 没有 `this` 指针——毕竟它不隶属于任何对象。
- 只能访问静态成员变量和静态成员函数，不能访问普通的（非 static 的）成员。

顺理成章——既然没有 `this`，它就没办法知道"你说的 `id` 到底是哪个对象的 `id`"。静态函数能看到的数据，必然是类级别的、全局一份的那个数据。

```cpp
class Math
{
public:
    static int add(int a, int b) { return a + b; }    // 不需要任何对象
    static int abs(int x) { return x < 0 ? -x : x; }
};

int main()
{
    cout << Math::add(3, 5) << endl;    // 8
    cout << Math::abs(-10) << endl;     // 10
    // Math 类不需要创建任何对象，直接用类名调函数就行
}
```

很多工具性质的函数——比如纯数学计算、字符串处理辅助——写成静态成员函数比写成普通独立函数更"有归属感"——一看就知道这函数跟哪个类有关。

------

**类作用域**

C++ 的作用域是按花括号来划的，而类的作用域是整个类的花括号内部。类里的成员变量和成员函数都属于这个作用域。

外面访问类成员的时候，要用 `类名::成员名` 或 `对象.成员名`。这个 `::` 叫**作用域运算符**，相当于在说"去这个作用域找到那个名字"。

```cpp
class Box
{
public:
    static const int MAX_WEIGHT = 1000;
    int weight;
};

int value = Box::MAX_WEIGHT;   // 通过作用域运算符访问静态常量

Box b;
b.weight = 500;                 // 通过对象访问普通成员
```

`static const` 成员可以在类内直接初始化（例子中的 `= 1000`），因为它是整数或枚举类型的编译期常量。普通的静态成员（非 const 或不是整数类型）则必须在类外单独定义，就像前面 Soldier 的例子那样。

------

**应用场景**

静态成员最常见的用途：

1. **计数器**：记录某类一共创了多少个对象（如前面的 `Soldier`）。
2. **常量**：与类相关的固定值，如 `Math::PI`。
3. **工具函数库**：纯计算性的辅助函数，组到一个类里封装起来，免得散落各处成为孤魂野鬼。
4. **单例模式**：确保某个类在整个程序中只存在一个实例——这件事靠静态的全局唯一的指针或引用实现，后面会学到。
