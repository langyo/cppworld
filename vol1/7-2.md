# 高级容器 · 类类型

在上一节，我们定义了第一个类。你可能也注意到了——类本身其实就是一个自定义的类型。这就意味着：**凡是被内置类型能做到的事情，你自定义的类类型也能做到。**

类类型的对象可以：

- 作为变量被声明和使用
- 作为另一个类的成员
- 作为数组或 vector 的元素
- 作为函数的参数和返回值

这也是 C++ 不同于很多语言的地方——用它定义出来的新类型，在语言里享受和内置类型几乎同等的待遇，没有因为你是"后来写的"就被当成二等公民。

------

**一个类作为另一个类的成员**

一个类可以包含另一个类的对象作为自己的成员——这就是**组合**（composition）。

```cpp
class Point
{
public:
    int x, y;
};

class Circle
{
private:
    Point center;      // 圆的圆心，center 是 Point 类型的对象
    int    radius;     // 圆的半径
public:
    void draw() const
    {
        cout << "绘制一个圆，圆心在 (" 
             << center.x << ", " << center.y << ")，半径 " 
             << radius << endl;
    }
};
```

上面 `Circle` 里有一个 `Point` 成员，这跟 `struct` 早就支持的嵌套没什么区别——但类可以做得更多：通过构造函数来控制这些成员怎么被初始化（下节就讲）。

------

**类在容器中**

类类型的对象可以像内置类型一样放进 `vector` 里：

```cpp
vector<Point> points;
points.push_back({10, 20});
points.push_back({30, 40});

for (const auto& p : points)
    cout << "(" << p.x << ", " << p.y << ") ";
```

`vector<Point>` 的用法和 `vector<int>` 几乎一模一样。标准库容器的模板参数不受限于内置类型——只要你的类支持必要的操作（如可复制、可比较等），就能往里塞。

------

**类作为函数参数和返回值**

类对象可以作为函数参数传递，也可以作为返回值返回：

```cpp
Point createPoint(int x, int y)
{
    Point p;
    p.x = x;
    p.y = y;
    return p;     // 返回一个 Point 对象
}

void printPoint(const Point& p)   // 常引用传入，避免拷贝开销
{
    cout << "(" << p.x << ", " << p.y << ")" << endl;
}
```

**按值 vs 按引用**：如果对象比较大（里面的数据很多），尽量用引用传递（尤其是常引用）来避免整个对象的拷贝开销。这个之前讲函数参数的时候已经说过，对类对象同样适用。

------

**类的拷贝**

默认情况下，C++ 会为你的类自动生成拷贝——把一个对象的每个成员逐个复制到另一个对象里：

```cpp
Point p1;
p1.x = 10;
p1.y = 20;

Point p2 = p1;    // 拷贝：p2.x = 10, p2.y = 20
p2.x = 999;       // 只改 p2，p1 不受影响
```

这种默认拷贝在实际开发中大多数时候够用了。但如果你的类持有"独占资源"（比如动态分配的内存块、文件句柄、网络连接），默认拷贝可能会引发灾难——两个对象持有同一份资源的所有权，一方归还资源，另一方就拿了空头支票。这种情况我们在第九章讲动态内存时会重点展开。

------

**const 对象和 const 成员函数**

如果你的对象是 `const` 的，那你只能调用它的 `const` 成员函数：

```cpp
class Counter
{
private:
    int count = 0;
public:
    void increment()         // 非 const —— 会修改 count
    {
        ++count;
    }

    int getCount() const     // const —— 不会修改任何成员
    {
        return count;
    }
};

int main()
{
    Counter c1;
    c1.increment();       // 普通对象可以调这两个函数
    cout << c1.getCount();

    const Counter c2;
    // c2.increment();    // ❌ const 对象不能调非 const 成员函数
    cout << c2.getCount(); // ✅ const 对象可以调 const 成员函数

    return 0;
}
```

记住一条原则就够：**是否 `const` 是对象和成员函数之间的约定。只读不写的函数要标上 `const`，`const` 对象只能找 `const` 成员函数。**

------

**mutable：const 函数中的"例外"**

有一种特殊场景：你希望在 `const` 成员函数里修改某个成员变量——不是为了改对象的核心数据，而是为了某种缓存、统计、或调试计数器。C++ 允许你给该成员打上 `mutable` 标记，逃过 `const` 封锁：

```cpp
class Cache
{
private:
    mutable int accessCount = 0;    // const 函数也能改
    int data;
public:
    int getData() const
    {
        ++accessCount;    // 虽然函数是 const，但 accessCount 是 mutable
        return data;
    }
};
```

`mutable` 在实际工程中使用频率不是很高，但知道它可以避免一些走投无路时的 const_cast 暴力手段（后者是非常不推荐的做法，等于自己拆掉编译器给你的安全护栏）。

------

类类型没有你想象中那么神秘——它就是你自己造的一种新类型，跟 `int`、`string`、`vector` 一样能用、能传、能装容器。区别只在于，内置类型是 C++ 自带第一套模具，类是你自己画的第二套模具。
