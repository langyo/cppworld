# 分身术 · 函数重载

同一个名字，能不能对应多件事？

当然可以。就像"开门"和"开口"，虽然都带个"开"字，但干的完全是两码事。C++ 的**函数重载**也允许你给不同的函数起同一个名字——只要它们的参数长得不一样。

------

**Why：为什么需要重载**

先看一个场景。你要写一个取较大值的函数：

```cpp
int maxInt(int a, int b)           { return a > b ? a : b; }
int maxDouble(double a, double b)  { return a > b ? a : b; }
```

因为函数名不能撞车，你只好一个叫 `maxInt`，一个叫 `maxDouble`，各叫各的。调用的时候得记清楚哪个名字对应哪种类型，烦不烦？

C++ 告诉你：不必这么麻烦。你可以让它们**共用同一个名字**：

```cpp
int max(int a, int b)          { return a > b ? a : b; }
double max(double a, double b) { return a > b ? a : b; }
```

调用的时候写 `max(3, 5)`，编译器一看参数是 `int`，就自动帮你调第一个版本；写 `max(3.14, 2.71)`，编译器一看是 `double`，就调第二个。不用你操心，干得利利索索。

------

**重载的规则**

不是随便乱写同名函数都叫重载。重载的关键在于：**参数列表必须不同**。要么参数个数不同，要么参数类型不同，要么个数和类型都不一样。

下面这些全是合法的重载：

```cpp
void print(int value);
void print(double value);       // 类型不同
void print(int value, int end); // 个数不同
void print(double value, int end); // 类型和个数都不同
```

而下面这些**不**算重载，编译器会直接报"重复定义"：

```cpp
int  add(int a, int b);
void add(int a, int b);   // ❌ 参数一样，光靠返回值不同不算重载！
```

**返回值类型不参与重载的判断**。只改个返回类型，编译器根本分不清你想调哪一个版本。

------

**编译器是怎么匹配的**

调函数时，编译器会拿实参的类型去逐一比对各个重载版本，挑出最匹配的那个。这个过程叫**重载决议**。

```cpp
void doSomething(int x)    { cout << "int 版本" << endl; }
void doSomething(double x) { cout << "double 版本" << endl; }

int main()
{
    doSomething(10);       // 输出 "int 版本"
    doSomething(3.14);     // 输出 "double 版本"
    doSomething(10.0f);    // float → 输出什么？
}
```

第三个调用 `doSomething(10.0f)` 传的是 `float`。编译器一看没有 `float` 版本，但 `float` 可以自动转成 `double`，于是调了 `double` 版本。这就是所谓的**隐式类型转换**在起作用。

但有时候编译器会犯难："两个版本都能接这个值，到底挑谁？"比如：

```cpp
void test(int x, double y)    { cout << "版本 1" << endl; }
void test(double x, int y)    { cout << "版本 2" << endl; }

int main()
{
    test(1, 2);   // 两个版本都能选，编译器懵逼，报 ambiguous 错误
}
```

这就叫**二义调用**，编译器直接放弃并报错。遇到这种情况，你得手动"打个小抄"：把实参强转成明确的类型。

```cpp
test(1, double(2));  // 明确告诉编译器选版本 1
```

------

**重载与默认实参的配合**

默认实参我们下一节会详讲，但这里先提一下：默认实参也能和重载配合，但也要小心二义性。

```cpp
void draw(int x, int y);          // 需要一个位置
void draw(int x, int y, int size = 10); // 需要一个位置和一个可选尺寸

draw(100, 200);  // ❌ 到底是调第一个还是第二个？编译器也茫然
```

如果你的重载版本里有一个带默认参数、另一个不带，调用时如果只传了不带默认参数所需数量的实参，编译器就会陷入选择困难——两难都符合。这时候，要么去掉重载合并成一个函数，要么老老实实让参数个数有明确的差异。

------

**重载不是什么**

函数重载只对**函数名**生效。它不作用于：
- **返回值类型**（之前已经说过）
- **顶层 const**——`int` vs `const int` 在参数层面不区分重载
- **引用是底层的一部分**：`int&` 和 `const int&` 是能区分重载的

除此之外，重载还有一个局限：**它只看参数表，不看代码里到底干了什么**。哪怕你写了三个人生理想完全不同的同名函数，只要参数表抄得一模一样，编译器就当它们是同一个。

------

**总结**

重载就像给函数装了一个分身术。同一段"起个名字"的功夫，能对应上不同类型、不同个数的参数，让调用方写代码更加顺手——想取最大值？`max` 一下就好了，管它 `int` 还是 `double`。
