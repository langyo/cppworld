# 欲工必先求利器 · 编程环境配置

古人云："工欲善其事，必先利其器。"在正式开始编写程序之前，我们得先把家伙事儿备齐了。

也许你听过"记事本也能编程"这种说法。理论上确实可以，但实际操作起来那可真叫一个折磨——没有语法高亮，没有自动补全，没有错误提示，简直就像在黑灯瞎火里摸鱼。咱们是学 C++ 的，不是来受刑的。

挑一件称手的武器，我们上路吧。

------

编写 C++ 程序，最少需要准备两样东西：

- **编译器**：负责把我们写的代码翻译成计算机能跑的程序。
- **文本编辑器**或**集成开发环境**（IDE）：写代码用的。

编译器是必需品，缺了它啥也干不成。我们写的 C++ 代码，计算机其实根本看不懂——就像我们不能直接读懂一堆 0 和 1 一样，计算机也没法理解我们用英语单词拼出来的代码。编译器就是这个"翻译官"，把人类的代码翻译成机器能执行的指令。

常见的 C++ 编译器有三种：

| 编译器 | 支持平台 | 一句话介绍 |
| ------ | -------- | ---------- |
| GCC | Windows, Linux, macOS | 开源免费的老大哥，用得最广 |
| Clang | Windows, Linux, macOS | 同样是开源免费，错误提示特别友好 |
| MSVC | Windows | 微软亲儿子，和 Windows 系统结合最紧密 |

对于初学者来说，挑哪个编译器其实都差不多。重要的是先把环境跑起来，让程序能动弹。

------

**Windows 平台**

在 Windows 上，推荐两种方式：

**方案一：Visual Studio**

Visual Studio 是微软出的集成开发环境，一站式解决所有问题。安装的时候，在安装程序中勾选"使用 C++ 的桌面开发"这个工作负载就行。安装完成后：

1. 打开 Visual Studio，点击"创建新项目"
2. 选"控制台应用"模板
3. 填个项目名，选个存放位置
4. 点击"创建"

完事。

**方案二：Visual Studio Code + MinGW**

VS Code 是轻量级编辑器，搭配 MinGW 编译器就能搞 C++ 开发：

1. 下载安装 VS Code
2. 下载 MinGW-w64，解压到一个合适的目录
3. 把 MinGW 的 `bin` 文件夹路径加到系统环境变量 `PATH` 里
4. 在 VS Code 中装 C/C++ 扩展
5. 配一下编译和调试任务

如果你觉得第二个方案太折腾，别纠结，直接用方案一。

------

**Linux 平台**

Linux 系统一般已经预装了 GCC。打开终端，敲一下这行命令检查：

```bash
g++ --version
```

如果提示找不到命令，就用包管理器装一下：

```bash
# Ubuntu / Debian
sudo apt install g++

# Fedora
sudo dnf install gcc-c++

# Arch Linux
sudo pacman -S gcc
```

编辑器随你喜欢——VS Code、Vim、或者 CLion、Qt Creator 这种重型 IDE 都行。

------

**macOS 平台**

macOS 下推荐装 Xcode 命令行工具：

```bash
xcode-select --install
```

这条命令会装上 Clang 编译器和其他开发工具。之后你就可以用 VS Code 或 Xcode 来开发了。

------

环境配好了没有？来验证一下。打开终端或命令提示符，输入：

```bash
g++ --version
```

（如果你装的是 Clang，那就打 `clang++ --version`）

如果能看到版本号蹦出来，恭喜，编译器装好了。

接下来我们真刀真枪跑一下第一节的程序。用文本编辑器新建一个叫 `hello.cpp` 的文件，敲入：

```cpp
#include <iostream>
int main()
{
    std::cout << "Hi, C++ World!" << std::endl;
    return 0;
}
```

保存好文件，终端里切到文件所在的目录，输入编译命令：

```bash
g++ hello.cpp -o hello
```

这行命令的意思是：拿 g++ 编译器，编译 `hello.cpp` 这个源文件，吐出一个叫 `hello` 的可执行文件。

编译完没报错的话，就运行它：

```bash
# Windows
hello.exe

# Linux / macOS
./hello
```

屏幕上要是出现了 `Hi, C++ World!`，环境就配好了！

------

在实战开发中还会用到不少辅助工具：

- **调试器**：能让你一行一行跑程序，观察变量的值，揪出藏在暗处的 bug。
- **版本控制**：比如 Git，帮你管理代码的每一个历史版本。
- **构建工具**：比如 CMake，帮你管理大项目的编译流程。

这些工具我们现在不用急着学，等用到的时候再逐步上手。

家伙已经备齐，我们这就出发吧。
