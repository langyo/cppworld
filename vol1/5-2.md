# 以不变应万变 · 深入参数与返回值

上一节我们认识了函数——它就像一条流水线，原料从入口进去，成品从出口出来。今天我们来看看：原料怎么进、成品怎么出。

------

**参数：函数的"入口"**

回顾一下，我们之前写的 `make` 函数：

```cpp
int make(int n)
{
    return n / 9;
}
```

`int n` 就是函数的**形参**。当你写下 `make(kelps)` 时，变量 `kelps` 里存的值就被复制给了 `n`，这个过程叫**传参**。

`kelps` 叫**实参**——你真正交出去的；`n` 叫**形参**——函数收到的副本。

把实参的值复制一份给形参，叫作**值传递**。这是 C++ 默认的传参方式。

值传递有什么特点呢？看这段代码：

```cpp
void addOne(int n)
{
    n = n + 1;
}

int main()
{
    int a = 5;
    addOne(a);
    cout << a << endl;   // 输出 5，不是 6
}
```

怎么还是 5？因为函数收到的是 `a` 的**副本**，不是 `a` 本身。函数里改了副本，原来的 `a` 纹丝不动。

------

**引用：让函数摸到"正本"**

如果你希望函数内部能修改外面的变量，怎么办？用**引用传递**。

在参数类型后面加个 `&`，就把值传递变成了引用传递：

```cpp
void addOne(int& n)     // 注意这个 & 号！
{
    n = n + 1;
}

int main()
{
    int a = 5;
    addOne(a);
    cout << a << endl;   // 输出 6！
}
```

这回函数收到的不是副本，而是 `a` 的"别名"。`n` 和外面的 `a` 指向的是同一块内存，改 `n` 就是改 `a`。

引用传递最常见的场景有两个：一是需要修改外部变量的值时，二是要传递大块数据（比如一个装着好几千个元素的对象）不想复制一份拖慢速度时。

> **小提醒**：当你声明一个引用类型变量时，必须在声明的同时初始化它，告诉它"你替谁站岗"。

------

**常引用：只许看，不许碰**

有些时候，我们想让函数能拿到外面的数据（不用把数据整个复制一份），但又不想让它改。这时候用**常引用**：

```cpp
void printValue(const int& v)    // 常引用，只能读不能改
{
    cout << v << endl;
    // v = 10;   ← 这句话编译都过不去！
}
```

`const int&` 翻译成人话就是："给你看，但别想动它一根寒毛。"

对于拿进来只读不写的参数，能用常引用就尽量用常引用，既安全又高效。

------

**返回值：函数的"出口"**

函数可以把结果还给调用的地方。还回去的方式就是 `return` 语句。

一个函数可以返回任意类型的值：

```cpp
int    getNumber()    { return 42; }
double getPi()        { return 3.14159; }
bool   isPositive()   { return true; }
char   getFirstChar() { return 'H'; }
```

如果函数什么都没打算还，写 `void` 就好：

```cpp
void sayHello()
{
    cout << "Hello!" << endl;
    // 没有 return 语句也不要紧
}
```

`void` 函数也可以写 `return;` 来提前退出，但不能往后跟任何值。

------

**关于 return 的一个常见误解**

初学者有时候会写这样的东西：

```cpp
int wrong()
{
    if (条件)
        return 1;
    // 如果条件不成立呢？函数走到了末尾却没有 return 任何东西！
}
```

编译器会给你报个警告。凡是非 `void` 的函数，必须保证在每一条可能的执行路径上都写了 `return`。

正确的姿势：

```cpp
int correct()
{
    if (条件)
        return 1;
    else
        return 0;
}

// 或者更简洁的写法：
int correct2()
{
    return 条件 ? 1 : 0;
}
```

------

**值返回 vs 引用返回**

函数还能返回引用——这听起来有点绕，但用好了很强大：

```cpp
// 返回两个 int 中较大的那个的引用
int& max(int& a, int& b)
{
    if (a >= b) return a;
    else        return b;
}

int main()
{
    int x = 10, y = 20;
    max(x, y) = 0;     // 把 y 改成 0！
    cout << "x: " << x << ", y: " << y << endl;  // x: 10, y: 0
}
```

因为 `max(x, y)` 返回的是 `y` 的引用，对返回值赋值其实就是改了 `y` 本身。这是值返回做不到的事。

**但有一个大坑**：千万不能返回**局部变量的引用**！

```cpp
int& danger()
{
    int temp = 42;
    return temp;    // 危险！temp 出了这个函数就不存在了！
}
```

`temp` 是局部变量，函数一结束它就寿终正寝了。你还回去一个已经死了的变量的引用，就像拿着一个过期的地址找人，天知道那个地址现在住着谁。

值返回就没有这个问题——它还回去的是**复制品**，正本死了也不影响副本。

------

**多返回值**

C++ 函数只有一个返回值。那要是想一次还回去多个东西怎么办？方法不少：

方法一：用引用参数。

```cpp
void divide(int a, int b, int& quotient, int& remainder)
{
    quotient  = a / b;
    remainder = a % b;
}
```

方法二：定义一个结构体，打包返还。

```cpp
struct Result
{
    int quotient;
    int remainder;
};

Result divide(int a, int b)
{
    return {a / b, a % b};
}
```

方法三：用标准库的 `pair` 或 `tuple`（以后会讲到）。

无论如何，记住一点就够：函数的参数是入口，返回值是出口。搞清楚了数据怎么进、怎么出，函数这个工具你就算真会用了。
