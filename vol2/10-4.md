# 换一种干活的方式 · 函数调用运算符定义

有些时候你想让一个对象看起来像一个函数——用圆括号直接"调用"它。这就是**函数对象(functor)**——一个重载了 `operator()` 的类。和普通函数相比，函数对象最大的杀招是它**能有状态**——它能"记住"之前的调用。

Lambda 表达式的背后原理就是函数对象（编译器为 Lambda 生成的匿名类里有一个 `operator()`）。所以学习函数对象也是理解 Lambda 的第一步。

------

## 一个带状态的函数对象

```cpp
#include <iostream>
#include <vector>
#include <algorithm>

class Accumulator
{
private:
    int sum = 0;               // 状态——普通函数做不到
public:
    int operator()(int x)      // 这个类现在可以像函数一样被"调用"
    {
        sum += x;
        return sum;
    }
};

int main()
{
    Accumulator acc;
    cout << acc(10) << "\n";    // 10  —— acc 看起来像函数
    cout << acc(20) << "\n";    // 30
    cout << acc(30) << "\n";    // 60
    // 每次调用都记住之前的总和，不需要外部变量
}
```

`acc(10)` 被编译器翻译成 `acc.operator()(10)`——就是调了一次名叫 `operator()` 的成员函数而已。但它让对象在语法上拥有了"可调用性"。

------

## 函数对象与标准算法配合

标准算法要的就是可调用对象。相比 Lambda，函数对象多一个关键优势：**它是命名类型，可以被模板精确推导、编译期内联**。

```cpp
#include <iostream>
#include <vector>
#include <algorithm>

class Descending
{
public:
    bool operator()(int a, int b) const
    {
        return a > b;
    }
};

int main()
{
    vector<int> v = {3, 1, 4, 1, 5, 9, 2, 6};
    sort(v.begin(), v.end(), Descending());   // 直接传入临时函数对象

    for (int x : v) cout << x << " ";          // 9 6 5 4 3 2 1 1
    cout << "\n";
}
```

因为 `Descending` 是一个具体的类型（不是指针），编译器能内联 `sort` 里每一次 `operator()` 的调用——同样场景用函数指针就做不到这个优化。

------

## 一个函数对象多个签名

重载多个 `operator()`，让一个对象能处理多种调用形式：

```cpp
class Logger
{
    ostream& out;
public:
    explicit Logger(ostream& os) : out(os) {}

    void operator()(const string& msg)    // 字符串消息
    {
        out << "[LOG] " << msg << "\n";
    }

    void operator()(const string& tag, int value)   // 带标签的数值
    {
        out << "[" << tag << "] " << value << "\n";
    }
};

int main()
{
    Logger log(cout);
    log("服务启动");              // [LOG] 服务启动
    log("CPU使用率", 78);        // [CPU使用率] 78
}
```

## 函数对象做模板参数——从类型级别传递行为

```cpp
template<typename Func>
void repeat(int times, Func f)
{
    for (int i = 0; i < times; ++i)
        f(i);
}

int main()
{
    auto printIndex = [](int i) { cout << "第 " << i << " 次\n"; };
    repeat(5, printIndex);
}
```

`Func` 可以是 Lambda、函数指针、或重载了 `operator()` 的类——一套模板，全都能接。这正是标准库算法的本质——靠模板 + 可调用对象统一千变万化的情况。

函数对象不是要你彻底替代 Lambda——实际上，在现代 C++ 中，大部分短小的临时可调用对象直接写 Lambda 更快。但当你需要一个"有名字的、可复用的、持有状态的可调用体"时，函数对象就是那个标准答案。
