# 换一种干活的方式 · 函数调用运算符定义

Lambda 表达式之所以能当函数用，背后的原理就是它所属的类重载了 **`operator()`**——函数调用运算符。

既然 Lambda 能做，我们当然也能在自己的类里做。重载 `operator()` 的对象又叫**函数对象**（functor），用起来和函数一模一样，但它比函数多了一样东西——它能有**状态**。

------

**一个带状态的函数对象**

```cpp
class Accumulator
{
private:
    int sum = 0;          // 状态——普通函数做不到这一点
public:
    int operator()(int x)
    {
        sum += x;
        return sum;
    }
};

Accumulator acc;
cout << acc(10) << endl;  // 10
cout << acc(20) << endl;  // 30
cout << acc(30) << endl;  // 60
// acc 看起来像一个函数，但它"记得"之前的累加值
```

每次 `acc(x)` 调用都是 `acc.operator()(x)` 的简写。这里 `acc` 明明是个对象，写法上却像函数调用——所以是"函数对象"。

和 Lambda 捕捉变量之后 "记住状态" 是同一个原理。写一个 Lambda `[sum = 0](int x) mutable { sum += x; return sum; }`，把它包装成一个 `auto f = ...` 变量，效果和上面的 Accumulator 几乎完全一样——实际上编译器给 Lambda 生成的闭包类内部就用 `operator()` 来实现调用。

------

**用函数对象自定义排序**

标准库的算法喜欢接收可调用对象。函数对象是其中一种，而且因为它是类型（不是指针），编译器可以轻松内联：

```cpp
class Descending
{
public:
    bool operator()(int a, int b) const
    {
        return a > b;
    }
};

vector<int> v = {3, 1, 4, 1, 5};
sort(v.begin(), v.end(), Descending());
// v = {5, 4, 3, 1, 1}
```

这里 `Descending()` 创建了一个临时的函数对象。因为 `operator()` 是内联可见的，编译器很可能直接内联排序过程中的每一次比较，而函数指针或 `std::function` 会多一层间接调用开销。这是性能敏感场景下函数对象依然比 `std::function` 有优势的原因。

------

**函数对象做参数**

你还可以把函数对象当成参数传——再次展现它的"像函数又像数据"特性：

```cpp
template<typename Func>
void repeat(int times, Func f)
{
    for (int i = 0; i < times; ++i)
        f(i);
}

// 使用
Accumulator acc;
repeat(5, ref(acc));    // 传引用，让 acc 在 repeat 里被修改
cout << acc(0) << endl;  // 能看到累积效果
```

因为函数对象有类型，模板可以推导出 `Func` 的具体类型，从而生成特化代码，实现零开销抽象——这正是 C++ 标准库算法高效率的核心。

------

**多重 `operator()`：一个对象承担多种调用形式**

一个类可以重载多个不同签名的 `operator()`：

```cpp
class Overloaded
{
public:
    int  operator()(int x)           { return x * 2; }
    int  operator()(int a, int b)    { return a + b; }
    void operator()(const string& s) { cout << s << endl; }
};

Overloaded ov;
cout << ov(21)       << endl;   // 42
cout << ov(10, 20)   << endl;   // 30
ov("Hello");                      // 打印 Hello
```

这就是为什么一个 Lambda 只能有固定的参数形式——它对应的是编译器生成的一个确定的 `operator()` 签名；但手写类完全可以装上好几个不同签名的 `operator()`，一个对象当多种函数用。
