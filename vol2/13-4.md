# 不可知的未来 · 随机数

有些时候你需要"随机"：

- 游戏刷一个随机宝箱。
- 密码学里生成密钥。
- 抽奖系统摇号码。
- 测试代码时生成随机数据来检查健壮性。

C 语言时代，大家都用 `rand()`——简单但质量很低，周期短，分布不均匀。C++11 引入了一套全新分层设计的随机数库，在 `<random>` 里。它将随机数的生成分为三道工序：随机设备提供种子，随机数引擎把种子变成随机数序列，分布把随机数序列塑形成你需要的范围。

------

**现代 C++ 随机数的正确姿势**

```cpp
#include <random>
#include <iostream>
using namespace std;

int main()
{
    random_device rd;                       // 真随机设备（或至少为种子提供一些熵）
    mt19937 gen(rd());                      // 梅森旋转引擎——高质量伪随机数生成器

    uniform_int_distribution<int> dist(1, 100);  // 均匀整数分布，范围 [1, 100]

    for (int i = 0; i < 5; ++i)
        cout << dist(gen) << " ";           // 生成五个 1~100 的随机整数
    // 比如输出：42 17 83 95 61

    return 0;
}
```

`mt19937` 是梅森旋转算法（Mersenne Twister）的标准实现，周期长达 $2^{19937} - 1$，足够几乎所有非密码用途。`random_device` 是一个系统特定的熵源——在大多数平台上它能拿到硬件或操作系统的真正随机数（如 `/dev/urandom` 或 RDRAND 指令），但标准并不强制它一定是真随机，有些嵌入平台可能退化成伪随机。对不需要密码学保证的场景而言这绰绰有余。

------

**引擎 + 分布 = 灵活组合**

库里有多种引擎和分布，可以按需混搭：

**引擎**（生成原始随机 bits）：
- `mt19937` / `mt19937_64` ——梅森旋转，通用首选。
- `ranlux48` ——统计学质量极高但性能较低，适合物理模拟类的高要求。
- `minstd_rand` / `minstd_rand0` ——最小标准线性同余，速度快但质量一般，仅用于内存极度紧张或速度极度敏感。
- `default_random_engine` ——由编译厂商选定的默认引擎，一般就是 mt19937 的包装，不建议依赖它——写清楚 `mt19937` 更靠谱。

**分布**（把原始 bits 塑形成你需要的范围和形状）：
- `uniform_int_distribution` ——均匀整数分布。
- `uniform_real_distribution` ——均匀实数分布。
- `normal_distribution` ——正态分布（高斯分布）。
- `bernoulli_distribution` ——0/1 分布（以给定概率返回 true）。

```cpp
normal_distribution<double> dist(70.0, 10.0);   // 均值 70，标准差 10
for (int i = 0; i < 5; ++i)
    cout << dist(gen) << " ";
// 可能输出：68.2 82.1 71.5 55.3 74.9
```

再加一个 `std::shuffle` 的用法——用随机引擎来随机打乱一个容器：

```cpp
vector<int> v = {1, 2, 3, 4, 5};
shuffle(v.begin(), v.end(), gen);
// v 现在是随机排序的，比如 {3, 1, 5, 4, 2}
```

C++17 的 `<algorithm>` 里 `sample` 也接受随机引擎，用于从序列中无放回抽样。

------

**注意**

- 不要每次调用随机函数时都重新创建一个新的引擎——这会不断重建内部状态，随机性大打折扣。引擎应该在程序开始时初始化一次，然后反复上喂给分布函数。
- `rand()` 和 `random_shuffle`（C++14 已标记为弃用，C++17 已移除）应被视为历史遗迹，新代码全部用 `<random>`。
- 密码学随机数请不要用 `mt19937`——它在统计领域表现优异，但不可抵御预测攻击。应使用操作系统提供的密码学安全随机接口（如 `CryptGenRandom`、`/dev/random` 或专门的密码库，C++ 标准库本身不提供密码级随机数生成器）。
