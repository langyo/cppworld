# 逻辑的空间 · 定长状态表 bitset

有些场合，你只需要表示某个东西的"开"或"关"——比如一个游戏角色的 32 种状态（无敌、隐身、加速、毒中……），或者记录 100 个灯的开或关。用 `int` 只能表达 32 个状态，用 `vector<bool>` 又有位压缩带来的微妙语义偏差（`vector<bool>` 实际上是标准库中一个著名的不合规范——它不返回 `bool&`）。

`std::bitset` 就在 `<bitset>` 头文件里等着你——它是一个长度在编译期确定的二进制位序列，每个位都能独立、自然地读写。

------

**创建 bitset**

```cpp
#include <bitset>
using namespace std;

bitset<8> b1;                  // 8 个位，全为 0
bitset<8> b2(42);              // 用二进制表示 42 → 00101010
bitset<8> b3(string("1100")); // 从字符串构造 → 00001100
```

`bitset` 的大小（如 `8`）必须是编译期常量，写在模板尖角括号内。

------

**操作位**

```cpp
bitset<8> b(string("10101010"));

b.set(0);              // 把第 0 位设为 1
b.reset(1);            // 把第 1 位清零
b.flip(2);             // 翻转第 2 位（0 变 1，1 变 0）

bool flag = b.test(3); // 检测第 3 位是否为 1（为 0 则返回 false）

b.set();               // 全部设为 1 ——相当于 b = ~bitset<8>(0)
b.reset();             // 全部清零
b.flip();              // 全部翻转

cout << b[0] << endl;  // 直接通过下标访问位（返回的是代理对象，不是 bool 引用）
cout << b.count() << endl;    // 一共有几个 1
cout << b.any() << endl;      // 至少有一个 1？
cout << b.none() << endl;     // 一个 1 都没有？
cout << b.all() << endl;      // 全是 1？
```

`b.test(pos)` 和 `b[pos]` 有什么区别？`test` 有越界检查（通过 `at` 式异常），`b[pos]` 和 vector 的 `[]` 一样不做检查。一般情况下 `b[pos]` 更快而且够用。

------

**位运算**

两个同长度的 `bitset` 直接可以进行位运算：

```cpp
bitset<8> a(string("1100"));
bitset<8> b(string("1010"));

cout << (a & b)  << endl;  // 1000
cout << (a | b)  << endl;  // 1110
cout << (a ^ b)  << endl;  // 0110
cout << (~a)     << endl;  // 11110011
cout << (a << 2) << endl;  // 00110000
```

结果直接输出就是二进制字符串，调试相当方便。这种内置的可打印性比直接用四则运算模拟位运算强了不止一个等级——不需要自己写 hexdump 才能看到底层值。

------

**bitset 适用场景**

- **编译期确定的标志位集合**——如编译器选项、硬件寄存器值、协议标志位。
- **集合运算**——并集（`|`）、交集（`&`）、对称差（`^`）在 bitset 上就是一条位运算指令，比 `set<int>` 用循环实现快上好几个数量级。
- **布尔数组的压缩替代品**——比 `bool arr[100]` 省 87.5% 内存。

**不适用场景**：大小在运行时才能确定时，`bitset` 不能用（它的大小必须是编译期常数）。这种场景应使用 `vector<bool>`（或更好的是 `vector<char>` 配合大小的显式管理，以避免 `vector<bool>` 不返回真正引用的坑）。

------

`bitset` 是很小众但精巧的工具——它的大小在编译期固定并不会在大多数商业软件里成为瓶颈，反而让你享有了 C++ 零开销抽象的位运算支持。哪天你需要管理一组有限的标志，"手写 int 位运算"不如"用一个写清了语义名字的 bitset 变量"来得靠谱。
