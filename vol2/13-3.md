# 逻辑的空间 · 定长状态表 bitset

有些场景你需要几十甚至几百个 on/off 开关——比如游戏角色身上挂着的一堆状态（无敌、隐身、加速、灼烧、冰冻、中毒……），或者记录一份 64 选项的配置掩码。用 `bool` 数组？一个标志一字节，浪费。用 `int` 位运算？读起来像密码。

`std::bitset<N>` 给你的答案——N 个二进制位，每一位独立读写，调试时直接输出为 0/1 字符串。它在 `<bitset>` 中。

------

## 创建和基本操作

```cpp
#include <bitset>
#include <iostream>

int main()
{
    bitset<8> b1;                     // 8 位全为 0
    bitset<8> b2(0b10101010);         // 用二进制字面量构造
    bitset<8> b3(string("1100")); // 从字符串构造："00001100"

    cout << "b2 = " << b2 << "\n";     // 直接输出二进制：10101010

    // 操作单一位
    b2.set(0);             // 第 0 位设 1
    b2.reset(1);           // 第 1 位清零
    b2.flip(2);            // 第 2 位翻转

    cout << "修改后: " << b2 << "\n";

    // 整体操作
    b2.set();              // 全部设 1
    b2.reset();            // 全部清零

    // 查询
    cout << "各位的 1 的个数: " << b2.count() << "\n";
    cout << "至少有一个 1？"   << b2.any()    << "\n";
    cout << "全是 0？"        << b2.none()   << "\n";
}
```

`b[pos]` 返回的是一个**代理对象**（不是 `bool&`），它能隐式转成 `bool` 且支持赋值。`b.test(pos)` 和 `b[pos]` 的区别如同 `vector::at` 和 `vector::operator[]`——前者越界抛异常，后者不检查。

------

## 位运算：当成整数来用

两个同长度的 `bitset` 天然支持全套位运算：

```cpp
bitset<8> a(string("1100"));
bitset<8> b(string("1010"));

cout << "a & b  = " << (a & b)  << "\n";   // 1000
cout << "a | b  = " << (a | b)  << "\n";   // 1110
cout << "a ^ b  = " << (a ^ b)  << "\n";   // 0110
cout << "~a     = " << (~a)     << "\n";   // 11110011
cout << "a << 2 = " << (a << 2) << "\n";   // 00110000
cout << "a >> 1 = " << (a >> 1) << "\n";   // 00000110
```

## 实战：游戏角色状态系统

用 `bitset` 做一个角色身上的 Buff/Debuff 管理：

```cpp
#include <bitset>
#include <iostream>
#include <string>
#include <vector>

// 定义每种状态的位索引
enum StatusIndex
{
    ST_INVINCIBLE = 0,   // 无敌
    ST_INVISIBLE  = 1,   // 隐身
    ST_SPEED_UP   = 2,   // 加速
    ST_BURNING    = 3,   // 灼烧（每帧扣血）
    ST_FROZEN     = 4,   // 冰冻（无法移动）
    ST_POISONED   = 5,   // 中毒
    ST_COUNT      = 6    // 状态总数——方便扩展
};

const char* StatusNames[] = {
    "无敌", "隐身", "加速", "灼烧", "冰冻", "中毒"
};

class Character
{
private:
    bitset<ST_COUNT> statuses;

public:
    void addStatus(StatusIndex s)
    {
        statuses.set(s);
        cout << "获得状态: " << StatusNames[s] << "\n";
    }

    void removeStatus(StatusIndex s)
    {
        statuses.reset(s);
        cout << "失去状态: " << StatusNames[s] << "\n";
    }

    bool hasStatus(StatusIndex s) const
    {
        return statuses.test(s);
    }

    void applyTickDamage()
    {
        if (hasStatus(ST_BURNING))
            cout << "灼烧伤害: -5 HP\n";
        if (hasStatus(ST_POISONED))
            cout << "中毒伤害: -2 HP\n";
    }

    bool canMove() const
    {
        return !hasStatus(ST_FROZEN);    // 冰冻时不能移动
    }

    void printStatuses() const
    {
        cout << "当前状态: " << statuses << " [";
        bool first = true;
        for (int i = 0; i < ST_COUNT; ++i)
        {
            if (statuses[i])
            {
                if (!first) cout << ", ";
                cout << StatusNames[i];
                first = false;
            }
        }
        cout << "]\n";
    }
};

int main()
{
    Character hero;
    hero.addStatus(ST_SPEED_UP);
    hero.addStatus(ST_POISONED);
    hero.addStatus(ST_FROZEN);

    hero.printStatuses();                    // 110100 [加速, 中毒, 冰冻]

    cout << "能移动？" << (hero.canMove() ? "是" : "否") << "\n";   // 否

    hero.removeStatus(ST_FROZEN);
    cout << "解除冰冻后，能移动？" << (hero.canMove() ? "是" : "否") << "\n"; // 是

    hero.applyTickDamage();     // 只对灼烧/中毒起作用
}
```

六个状态全存进一个对象里，占用不到一个 CPU 寄存器的大小，同时调试时一行 `statuses` 直接打印成 0/1 串——远比手写 `if (flags & (1 << 5))` 这种魔法数字清晰。

## 集合运算：bitset 的交并差

换一个角度——`bitset` 也可以看作是一个"最大元素数已知的小型集合"：

```cpp
bitset<32> setA;
setA.set(1); setA.set(3); setA.set(5);   // A = {1, 3, 5}

bitset<32> setB;
setB.set(3); setB.set(5); setB.set(7);   // B = {3, 5, 7}

auto intersection   = setA & setB;        // {3, 5}
auto unionSet       = setA | setB;        // {1, 3, 5, 7}
auto difference     = setA & ~setB;       // A 中有 B 中没有：{1}
auto symmetricDiff  = setA ^ setB;        // 对称差：{1, 7}
```

这些集合运算在一条 CPU 指令内完成，而用 `std::set<int>` 实现需要多条循环指令。在元素范围小且固定（如所有 ASCII 字符、所有游戏技能 ID）的场景下，`bitset` 比 `std::set` 快一个数量级以上。

------

## bitset 的局限

`bitset` 的大小在编译期写死——你不能在运行时根据用户输入来决定 `bitset<N>` 的 N 是多少。如果大小是运行时可变的，需要用 `std::vector<bool>`（或更推荐直接用 `std::vector<char>` 配合除 8 计算位偏移，因为 `vector<bool>` 不返回真正的引用这一点很多老手踩过坑）。
