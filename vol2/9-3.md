# 瓶子工厂 · 动态数组

前面在讲原生数组时我们就提到过：静态数组的大小是编译时定死的。你想在运行时由用户输入一个长度 n，然后创建一个 `int arr[n]`？老标准不答应（某些编译器可能有扩展支持，但不保证可移植）。

动态内存解决了这个限制，但也引入了手动管理的痛苦。

这一节我们把三块知识串起来：动态数组怎么来的、它为什么危险、以及更好的替代方案。

------

**new[] 和 delete[]：C++ 最早的"动态数组"**

```cpp
int n;
cin >> n;
int* arr = new int[n];     // 运行时动态分配数组，大小为 n

for (int i = 0; i < n; ++i)
    arr[i] = i;

delete[] arr;              // 别忘了中括号！
```

问题：
1. **得手动算大小**——你得另外记着 `n` 是多少，`arr` 本身不知道自己有多大。
2. **忘了 delete[] 就泄漏**——没有任何机制会提醒你。
3. **要扩容？得先分配更大的再搬，然后把旧的 delete[] 掉。**
4. **异常可能绕过 delete[]**——程序中途跳出去，你的指针就永远丢了。

这就好比你去租了一间仓库，老板让你自己记着仓库的地址和面积，忘了也不提醒——有没有更靠谱的"仓库租赁公司"？

有，标准库早就给我们准备好了：`vector`、`array`、`string`。

------

**vector：动态数组的最佳实践**

事实上，我们在第六章就学过——`vector` 就是"一个管理动态数组的容器"。它的内部就是用 `new[]` 和 `delete[]` 实现的（或者等效的分配器），但替你把所有"怎么扩容、什么时候析构"的问题全包了。

```cpp
int n;
cin >> n;
vector<int> v(n);        // 动态分配 n 个 int，全初始化为 0

for (int i = 0; i < n; ++i)
    v[i] = i * 2;

// 自动析构——不用写任何 delete，vector 自己还
```

而且 `vector` 还能动态增长——`push_back` 的时候，如果空间不够了，它会在内部偷偷去申请一块更大的内存并把旧数据搬过去。你不用费这个心。

------

**array：你需要固定大小的容器时**

静态数组有"退化指针"的毛病，但如果大小是编译期的常量，`std::array` 比原生数组更好：

```cpp
array<int, 10> arr = {1, 2, 3, 4, 5};   // 大小在编译期确定，且有容器的接口
// arr.size() 直接可用，不会退化
```

如果大小在编译期就能确定，`std::array` 是你的第一选择。它不分配堆内存，性能跟原生数组相同，但又提供了 `size()`、迭代器等标准接口，传参也不会退化。

------

**new[] 和 delete[] 还有用吗？**

在现代 C++ 里，直接用 `new[]` 和 `delete[]` 的情况已经非常少了。你可能会在以下场景见到它们：

1. **写底层库**（如自定义容器、内存池）时，需要手动管理内存。
2. **嵌入式系统**中标准库占内存太大或者干脆没有。
3. **遗留代码维护**——旧代码里遍地 new/delete，你维护它时得能看懂并学会逐步把它替换为智能指针或容器。

在应用层代码中，**能用 `vector` 就不用 `new[]`**，能用 `make_unique` 就不用 `new`。这是一个经过几十年 C++ 社区沉淀下来的共识。

------

**动态数组与智能指针的联手**

如果你确实需要直接分配动态数组但想利用智能指针的自动释放，C++14 后的 `unique_ptr` 可以管数组：

```cpp
auto arr = make_unique<int[]>(100);   // 一个管理 int[100] 的 unique_ptr
arr[0] = 42;                          // [] 可用
// delete[] 自动调
```

`shared_ptr` 管数组的情况要复杂一些——C++17 之前没有原生的 `shared_ptr<T[]>` 支持，需要用自定义删除器；C++17 起补齐了。但对多数初学者来说，遇到需要共享数组所有权的场景直接用 `vector<shared_ptr<T>>` 可能更直观。

------

**从旧代码一路进化到现代**

下面是一段简化的进化史——从最原始的手动管理到最现代的写法：

```cpp
// 阶段一：原始指针——容易忘记 delete
int* arr1 = new int[n];

// 阶段二：自我提醒，写 delete
delete[] arr1;   // 但如果中间有 return 或异常，就可能跳过去了

// 阶段三：包装成 vector——最佳选择
vector<int> arr2(n);

// 阶段四：如果用 unique_ptr 管数组（C++14）
auto arr3 = make_unique<int[]>(n);  // 有自动释放，但仍不如 vector 功能全
```

可以看到，现代 C++ 提供的工具层层递进，唯一的方向就是**让手动内存管理淡出应用层代码**。
