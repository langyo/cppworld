# 瓶子工厂 · 动态数组

重温一下原生数组最气人的地方——大小在编译期就得定死：

```cpp
int n;
std::cin >> n;
int arr[n];    // 绝大多数标准编译器不答应——编译期不知道 n 是多少
```

动态内存解了这个锁：用 `new[]` 在运行期按需要的大小分配数组。但同时也把"手动释放"这颗雷接了过来。C++ 的标准库则提供了更好的选择。

------

## new[] 和 delete[]：裸动态数组

```cpp
#include <iostream>

int main()
{
    int n;
    std::cout << "需要几个元素？";
    std::cin >> n;

    int* arr = new int[n];              // 运行期定大小——栈数组想都别想

    for (int i = 0; i < n; ++i)
        arr[i] = i * 10;

    for (int i = 0; i < n; ++i)
        std::cout << arr[i] << " ";

    delete[] arr;                       // 数组回收用的是 delete[]，不是 delete
}
```

缺点你肯定已经熟悉了：**大小得另记**（arr 自己不知道应该有多大）、**忘了 delete[] 就泄漏**、**扩容得先买新房再搬家**、**中间抛了异常 delete[] 到不了**——四个口子都在漏。

------

## 最优解：vector 本就是动态数组

事实上标准库的 `vector` 本质上就是对 `new[]`/`delete[]` 的完美包装——它替你处理了全部四个问题：

```cpp
#include <iostream>
#include <vector>

int main()
{
    int n;
    std::cout << "需要几个元素？";
    std::cin >> n;

    std::vector<int> v(n);              // 动态分配 n 个 int

    for (int i = 0; i < n; ++i)
        v[i] = i * 10;

    v.push_back(999);                    // 后面还能继续加——原生 new[] 做不到
    std::cout << "现在有 " << v.size() << " 个元素\n";

    // 析构是 vector 自己的事——不用写任何 delete
}
```

`push_back` 时如果容量不够，vector 会在背后悄悄分配一片更大的新内存、把原有元素全部移动过去、然后静默释放旧址——整个过程对你是透明的。扩容搬家有性能代价（大量 push_back 时可以通过提前 `reserve` 缓解），但远比手写搬家安全可靠。

------

## unique_ptr 也能管数组

C++14 起 `make_unique` 可以管动态数组：

```cpp
auto arr = std::make_unique<int[]>(50);    // 自动 delete[]
arr[0] = 42;
std::cout << arr[0] << "\n";
// 离开作用域自动释放——不需要写 delete[]
```

这比裸 `new[]` 安全，但依然不如 `vector` 功能多——`unique_ptr<int[]>` 没有 `size()`、`push_back` 等方法。它的适用场景是：你确实需要非 vector 管理的原始数组（比如给 C API 传参），但又想借助 RAII 管理生命期。

------

## 一条线从旧到新

```cpp
// C 风格 legacy——危险
int* data1 = new int[n];
delete[] data1;                    // 中间任何 return/异常都能跳过这行

// unique_ptr 包装——自动释放
auto data2 = std::make_unique<int[]>(n);
// 不费心写 delete[]

// vector——功能和安全性全能
std::vector<int> data3(n);
data3.push_back(extra);            // 还能接着往后接，前面两个都做不到
```

三层进化，唯一方向就是让手动的 `new[]` / `delete[]` 从你的应用层代码中完全消失。
