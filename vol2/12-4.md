# 无限剑制 · 可变参数模板

函数可以接受不定数量的参数——比如 `printf` 传入的格式字符串后面跟的若干个参数。C 语言用 `<cstdarg>` 的旧式可变参数来玩这一套，但它不安全——编译器不知道参数的类型，全靠调用者和 `printf` 的格式串对暗号。

C++11 引入的**可变参数模板**给出了一个类型安全、编译器全程可控的解决方案。它靠参数包（parameter pack）来定义"零或者多个参数"，用递归展开来逐项处理。

------

**参数包**

```cpp
template<typename... Args>         // Args 是一个模板参数包——零或多个不同的类型
void print(Args... args)           // args 是一个函数参数包——零或多个实参
{
    // ...
}
```

`...` 就是"包"的标志。调用：

```cpp
print(1);                    // Args = {int}, args = {1}
print(1, 2.5, "hello");      // Args = {int, double, const char*}, args = {1, 2.5, "hello"}
print();                     // Args = {}, args = {} —— 零个也可以
```

关键在怎么把这些参数逐个处理——这就需要**递归展开**。

------

**递归展开**

经典模式：一个"吃光"所有参数的递归函数：

```cpp
// 基础情况：没有参数了，停止递归
void print() { cout << "结束" << endl; }

// 递归情况：拿到第一个参数，把后面的参数包交给下一层递归
template<typename T, typename... Args>
void print(T first, Args... rest)
{
    cout << first << " ";
    print(rest...);      // 递归调用，参数包"减"掉一个参数
}

print("Hello", "C++", "World");
// 输出：Hello C++ World 结束
```

过程的展开：

```
print("Hello", "C++", "World")
→ 打 "Hello "，递归 print("C++", "World")
→ 打 "C++ "，递归 print("World")
→ 打 "World "，递归 print()
→ 打 "结束"，停
```

这个模式背下来：用一个非模板的、接受零参数的基极函数作为终止条件；用一个接受 `T first, Args... rest` 的模板递归函数一步步消耗参数包。编译器会在编译期一棵一棵地生成中间模板函数实例，每一个都少一个参数，最终调用到基函数。整个展开过程是完全编译期完成的，没有运行时循环或数组分配。

------

**sizeof...：参数包的元素个数**

```cpp
template<typename... Args>
int count(Args... args)
{
    return sizeof...(args);     // 参数个数
}
```

注意 `sizeof...` 后面必须直接跟包名，中间不能有任何括号或空格以外的间隔。这是编译期常量，可以用于 `array` 大小等编译期上下文中。

------

**折叠表达式：C++17 的简化写法**

递归展开写起来有时太啰嗦了。C++17 引入了**折叠表达式**：

```cpp
template<typename... Args>
auto sum(Args... args)
{
    return (args + ...);    // 折叠表达式：展开为 args1 + args2 + args3 + ...
}

cout << sum(1, 2, 3, 4, 5) << endl;    // 15
```

`(args + ...)` 是**一元右折**：从左到右展开为 `arg1 + (arg2 + (arg3 + ...))`。

`(... + args)` 是**一元左折**：`((arg1 + arg2) + arg3) + ...`。

除此之外还支持二元折叠（以一个初值起始）和一些通用运算符组合，详情可查阅标准手册。对于连加、连乘、连输出等模式，折叠表达式比递归模板简洁很多。

------

**std::make_shared 就是靠这招**

标准库中，`make_shared` 和 `make_unique` 能接受任意数量和类型的参数，就是因为它们内部是可变参数模板 + 完美转发——把调用者传来的任意参数组原样转给对象的构造函数。

```cpp
auto sp = make_shared<Point>(3, 5);   // 这里 3, 5 是以 Args... 的形式打包转给 Point 构造的
```

再比如，你可以写一个简单的 emplace 函数，帮你在容器里就地构造对象：

```cpp
template<typename Container, typename... Args>
void emplaceOne(Container& c, Args&&... args)
{
    c.emplace_back(forward<Args>(args)...);
}
```

这三个小节的内容——模板基础、完美转发、可变参数——组合在一起，就是 C++ 泛型编程核心工具集。学会它们之后，再看标准库里 `make_shared`、`emplace` 等函数的文档，你会发现它们其实就这点本事，没啥可怕的。
