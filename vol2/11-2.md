# 宗族的继承 · 基类与派生类

上一节我们用 Block 和它的子类展示了继承和虚函数的雏形，这里再来仔细拆解：继承是怎么一回事、派生类内部到底继承了哪些东西。

------

**继承的语法**

```cpp
class 派生类 : 继承方式 基类
{
    // 额外的成员
};
```

三种继承方式：

| 继承方式 | 基类的 public 成员 → 在派生类中变为 | 基类的 protected 成员 → |
| -------- | -------------------------------- | ----------------------- |
| `public` | public | protected |
| `protected` | protected | protected |
| `private` | private | private |

在实际工程中，**99% 的情况用 `public` 继承**。`protected` 和 `private` 继承主要用于更偏底层的设计（比如"用某类的实现但不应暴露其接口"），初学者暂时不用管。

直观来说，`public` 继承传达的就是一个"**是**"的关系——DirtBlock **是** Block。而 protected/private 继承传递的是"**凭借……实现**"的语义，属于另一套设计思路。

如果不安指定继承方式，`class` 默认是 `private` 继承（`struct` 默认是 `public` 继承——这是 `class` 与 `struct` 除了默认成员访问权限之外的另一个区别）。但强烈建议永远**显式写出继承方式**，不要依赖默认值。

------

**派生类能从基类获得什么**

```cpp
class Base
{
private:
    int secret;
protected:
    int family;
public:
    int everyone;
    void say() { cout << "Base" << endl; }
};

class Derived : public Base
{
public:
    void show()
    {
        // cout << secret;       // ❌ 基类的 private 成员——任何派生类都看不到
        cout << family;          // ✅ protected 成员——派生类能看到
        cout << everyone;        // ✅ public 成员——世界可见
        say();                   // ✅ 基类的 public 成员函数——世界也可见
    }
};

Derived d;
d.everyone = 10;    // ✅ public 继承来的 public 成员还是 public
d.say();            // ✅ 同上
// d.family;        // ❌ protected 成员外界看不见
```

概括就是一条：派生类拥有基类的全部成员（包括 private），但它只看得见 `public` 和 `protected` 的。外界的代码（非派生类）只看得见 `public` 的。private 成员存在于对象中，但只有基类自己的成员函数才能触碰它。

------

**派生类的构造函数**

派生类在自己的构造函数中，必须先**调用基类的构造函数**来初始化基类那部分数据，然后才初始化自己的成员：

```cpp
class Base
{
protected:
    int value;
public:
    Base(int v) : value(v) {}
};

class Derived : public Base
{
private:
    int extra;
public:
    Derived(int v, int e)
        : Base(v)      // 调基类的构造函数
        , extra(e)     // 初始化自己的
    {}
};
```

如果忘记调用基类构造函数，编译器会尝试调用基类的**默认构造函数**。如果基类没有默认构造函数（即你已自定义了构造函数且没有写 `= default` 的无参版本），编译就会报错。

初始化顺序永远是**基类在前，成员在后**，跟你在初始化列表里的书写顺序无关。所以不要试图在派生类的初始化列表中依赖未构造完成的基类部分——这是未定义行为。

------

**重定义与隐藏**

如果你在派生类里写了一模一样签名的非虚函数：

```cpp
class Base
{
public:
    void say() { cout << "Base" << endl; }
};

class Derived : public Base
{
public:
    void say() { cout << "Derived" << endl; }   // 隐藏（hide），而不是覆盖
};
```

这不是多态，这叫**名称隐藏**——派生类的 `say` 不给外界的代码任何机会看基类的版本，哪怕它名字一样。即使参数不一样，一样会隐藏：只要派生类定义了任何跟基类同名的函数（包括重载版本），基类的整个相同名字的全部被隐藏了。

可以通过 `using Base::say;` 声明把基类那个名字的所有重载版本重新暴露出来：

```cpp
class Derived : public Base
{
public:
    using Base::say;                    // 把基类的 say 重载集开放
    void say(int x) { cout << x << endl; }  // 派生类自己的追加版本
};
```

这在工程中很常见——派生类想给基类接口"做加法"（保持原有、再新加一两个重载），一行 `using` 就干净利落。

如果要实现真正的多态（派生对象通过基类接口调用派生类版本），就要靠下一节的重点——`virtual` 和 `override` 了。
