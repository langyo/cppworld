# 机械臂的自我修养 · 初识重载运算符

在第三章，我们学习了 C++ 内置的运算符——`+`、`-`、`*`、`/`、`==`、`<`……这些天生就能对 `int`、`double` 这样的内置类型进行操作。

但你有没有想过——如果是你自己定义的类呢？

```cpp
class Point
{
public:
    int x, y;
};

Point p1{1, 2}, p2{3, 4};
Point p3 = p1 + p2;   // ❌ 编译错误！编译器不知道 Point 怎么加
```

编译器不认识你的 `Point`，不知道两个 `Point` 相加是什么意思。这再正常不过——你得告诉它规则。

**运算符重载**就是做这件事的：让你可以定义一个运算符在你的自定义类型上的行为。就像你可以教机械臂学会"抓取"、"放下"、甚至"挥舞"，运算符重载让你教你的类理解"+"、"-"、"==" 等等。

------

**最简单的运算符重载**

我们给 `Point` 写一个加法：

```cpp
class Point
{
public:
    int x, y;

    Point operator+(const Point& other) const
    {
        return Point{x + other.x, y + other.y};
    }
};

Point p1{1, 2}, p2{3, 4};
Point p3 = p1 + p2;        // p3.x = 4, p3.y = 6
```

`operator+` 就是"如何执行加法"的说明书。当我们写 `p1 + p2` 时，编译器内部把它翻译成了 `p1.operator+(p2)`——本质就是调用一个叫 `operator+` 的成员函数。

你可以当场验证一下——先写 `Point p1{1, 2}, p2{3, 4};`，然后分别打印 `(p1 + p2).x` 和 `(p1 + p2).y`，看看是不是 4 和 6。

------

**成员 vs 友元：两种定义位置的抉择**

上面的 `operator+` 是作为成员函数定义的。还有一个常见的写法：作为**非成员函数**（通常声明为类的友元）：

```cpp
class Point
{
public:
    int x, y;

    friend Point operator+(const Point& a, const Point& b);
};

Point operator+(const Point& a, const Point& b)
{
    return Point{a.x + b.x, a.y + b.y};
}
```

作为成员函数时，左侧操作数必须是这个类的对象（因为成员函数隐式地需要 `this`）。作为非成员函数时，左右两边都可以做类型转换。

一个典型例子是 `ostream` 的 `operator<<`——它必须写成非成员函数（左侧是 `ostream&`，不是你自定义的类）：

```cpp
ostream& operator<<(ostream& os, const Point& p)
{
    os << "(" << p.x << ", " << p.y << ")";
    return os;
}
```

如果你强行把它写成 Point 的成员函数，就会变成 `p << cout` 这种反直觉的写法——不可取。

大原则是：**对称的二元运算符（`+`、`-`、`==`）如果左右两侧地位平等，最好写成非成员函数；单目或天然左偏的（`=`、`[]`、`()`）写成成员函数。** 遇到拿不准的情况，先写成成员函数，编译不过再考虑改成友元。

------

**重载 `==` 与 `!=`**

帮 `Point` 学会比较：

```cpp
class Point
{
public:
    int x, y;

    bool operator==(const Point& other) const
    {
        return x == other.x && y == other.y;
    }

    bool operator!=(const Point& other) const
    {
        return !(*this == other);   // 用 == 来实现 !=，DRY 原则
    }
};

Point a{3, 5}, b{3, 5}, c{1, 2};
cout << (a == b) << endl;   // true
cout << (a != c) << endl;   // true
```

注意 `!=` 是直接调用了 `==` 再取反——这样以后就算比较逻辑变了，也只需要改 `operator==` 一个地方。

------

**重载 `<<` 与 `>>`**

这个操作非常实用——以后你就可以这样写了：

```cpp
Point p{10, 20};
cout << p << endl;    // 输出：Point(10, 20)
// cin >> p;          // 从输入流读取坐标
```

实现：

```cpp
#include <iostream>
using namespace std;

class Point
{
private:
    int x, y;
public:
    Point(int a = 0, int b = 0) : x(a), y(b) {}

    friend ostream& operator<<(ostream& os, const Point& p)
    {
        os << "Point(" << p.x << ", " << p.y << ")";
        return os;  // 记得返回 os，这样才能支持连续输出
    }

    friend istream& operator>>(istream& is, Point& p)
    {
        is >> p.x >> p.y;
        return is;  // 同样返回 is，支持连续输入
    }
};
```

返回 `ostream&` 是为了支持链式输出，像 `cout << a << b << c;` 这样。如果返回 `void`，链就断了。

------

**一条永远不要忘记的原则**

运算符重载的目的是**让你的类型用起来像内置类型一样自然**。理想情况下，别人用你的类时甚至不需要知道运算符是重载过的——它就应该表现得跟直觉一致。

这意味着：
- `+` 不要干减法的事。
- `==` 要对称——`a == b` 和 `b == a` 的结果必须相同。
- 重载了 `<` 就最好也重载 `>`（但不用全部手写，可以用小的实现大的）。

**不要滥用**——如果你的运算符行为连你自己都要想半天才能解释清楚，那它就是不合适的。比如给一个文件类重载 `!` 来表示"删除文件"——好看吗？不，这是坑自己。
