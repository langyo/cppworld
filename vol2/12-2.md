# 容器类型的传递 · 转发模板

你写好了一个不错的函数模板。它接收参数，做点事，把参数再传给另一个函数。这个"传递"过程看起来很简单——参数进来，参数出去。但在模板的世界里，"转发"恰恰是一个非常容易出错的重灾区。

比如说，你想写一个通用的工厂函数——由你创建对象，交给调用者：

```cpp
template<typename T, typename Arg>
T create(Arg arg)
{
    return T(arg);
}
```

然后你发现 `Arg` 被拷贝了。如果外面传的是个右值——本打算"移交资源"——你这儿对 `arg` 多拷贝了一遍，效率嘎嘎掉。

再比如你传引用进来，里面因为不对，引用的特性被丢了。或者你传 const 进来，template 把 const 也草率处理掉了。

**完美转发**就是来解决这些"传进去的东西被走形"的问题的。它的目标很简单：实参无论是什么类型、什么值类别（左值还是右值），都能"原汁原味"地传到下一站。

------

**通用引用**

首先得认识一个新概念——**通用引用**（也叫转发引用）。它的写法是 `T&&`，但只有在 `T` 是**被推导出来的模板参数**时才是通用引用：

```cpp
template<typename T>
void wrapper(T&& arg)   // T&& 自动识别是左值还是右值
{
    // ...
}
```

调用：

```cpp
int x = 42;
wrapper(x);       // T 推导为 int&，arg 是 int&（左值引用）
wrapper(42);      // T 推导为 int，arg 是 int&&（右值引用）
```

当外面传的是左值，`T` 被推导为 `int&`，这样 `T&&`（即 `int& &&`）经过**引用折叠**规则变成 `int&`（一个引用加一个引用还是引用）。

当外面传的是右值，`T` 被推导为 `int`，`T&&` 保持为 `int&&`。

简单说：只要在模板中看到 `T&&` 并且 `T` 是由调用推导的，这个东西就会在"左值引用"与"右值引用"之间自动选择。不用你写两个重载。

------

**std::forward：完美转发的核心**

`std::forward` 是配合通用引用使用的：如果原始参数是左值，`forward` 把它作为左值传下去；如果原始参数是右值，`forward` 把它作为右值传下去。

```cpp
template<typename T>
T create(T&& arg)     // 通用引用
{
    // arg 在函数内部本身是一个左值——因为它有名有姓有地址
    return T(forward<T>(arg));   // forward 还原了原本的左右值属性
}
```

注意在 `create` 函数里面，`arg` 本身就变成了一个**左值**——它有自己的名字，你可以取它的地址。如果你直接写 `T(arg)`，那就会对 `arg` 进行拷贝（哪怕外面传进来的是右值）。`forward<T>(arg)` 就是在这个时机把右值身份"复原"回来。

`forward` 的一个常见记忆写法是：**给 `forward` 的模板参数加上 `T` 表示"如果外面传给我的时候类型是 X&，那就按左值算；否则按右值算"。** 不过你只需记住 `forward<T>(arg)` 这一个配方就行——跟 `move` 完全不同：`move` 是"无论如何都变成右值"，`forward` 是"还原原始身份"。

------

**应用案例：make_unique 的实现**

C++14 引入了 `make_unique`，它的内部就大量依赖完美转发：

```cpp
template<typename T, typename... Args>
unique_ptr<T> make_unique(Args&&... args)
{
    return unique_ptr<T>(new T(forward<Args>(args)...));
}
```

这是模板加转发的一个经典组合——接受任意类型、任意多个参数，然后"原样"转给 `T` 的构造函数，不丢信息、不多复制、不误判左右值。

------

**现实：转发+可变参数=终极接口**

如果你想写一个日志库的入口函数：

```cpp
template<typename... Args>
void log(const string& format, Args&&... args)
{
    printf(format.c_str(), forward<Args>(args)...);
}
```

这里 `Args&&...` 是可变参数包+通用引用的复合招，把参数"一个不少的忠实转发"变成了几行代码的事。下一节我们会专门探讨可变参数模板。

完美转发是模板编程中最重要的实用技术之一——不懂它，写出的模板函数很可能在调用者传右值时悄悄拷贝了一份，而你自己根本没意识到。
