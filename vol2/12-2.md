# 容器类型的传递 · 转发模板

你写好了一个漂亮的函数模板，把参数收进来，做点处理，然后传给下一个函数。这个"转手"的过程中，最容易犯的错误就是**参数被走形了**——外面传的是右值（想要移动），里面被当成左值又悄悄地拷贝了一次。

**完美转发**的目标：不管传进来的是什么类型、什么身份（左值还是右值），都原样交给下一站。

------

## 别从概念学，从问题出发

看一个经典坑——写一个通用工厂函数：

```cpp
template<typename T, typename Arg>
T create(Arg arg)
{
    return T(arg);      // 这里 arg 触发了一次不必要的拷贝
}

int main()
{
    std::string s = create<std::string>(std::string("Hello"));
    // 期望：临时 string 直接移动到新构造的 string 里
    // 实际：arg 被拷贝了一遍——因为函数内部 arg 多了一个名字，就成左值了
}
```

`arg` 一进入函数体，因为它有了名字、可以取地址，就变成了左值。`T(arg)` 看到的 `arg` 不再是当初那个临时右值——它会触发拷贝构造而不是移动构造。

------

## 通用引用：`T&&` 不永远是右值引用

当一个函数参数写成 `T&&` 并且 `T` 是模板推导出来的类型时，这就不是常规的"右值引用"——它是一个**通用引用**（也叫转发引用）。它能自动分辨调用者传进来的是左值还是右值：

```cpp
template<typename T>
void inspect(T&& arg)
{
    // arg 在里面永远是个左值（有名有姓）
}

int main()
{
    int x = 42;
    inspect(x);        // T 推导为 int&，int& &&  → int& （引用折叠）
    inspect(100);      // T 推导为 int， int&&    → int&&
}
```

引用折叠规则：`&` + `&` = `&`，`&` + `&&` = `&`，`&&` + `&` = `&`，`&&` + `&&` = `&&`。所以当调用方传左值时，`T` 被推导为 `int&`，`T&&` 变成了 `int& &&` → `int&`——一个左值引用。

------

## `std::forward`：把身份还给参数

有了通用引用，我们还需要 `forward` 在函数体内把这些参数的身份"还原"——把右值当作右值、把左值当作左值传下去：

```cpp
#include <iostream>
#include <utility>
#include <memory>
#include <string>

template<typename T, typename... Args>
std::unique_ptr<T> make(Args&&... args)
{
    // forward 根据 Args 推导出来的原始身份还原左右值
    return std::unique_ptr<T>(new T(std::forward<Args>(args)...));
}

struct Widget
{
    std::string name;
    Widget(const std::string& n) : name(n)
    {
        std::cout << "拷贝构造 Widget: " << name << "\n";
    }
    Widget(std::string&& n) : name(std::move(n))
    {
        std::cout << "移动构造 Widget: " << name << "\n";
    }
};

int main()
{
    std::string title = "Alpha";
    auto w1 = make<Widget>(title);                // title 是左值 → Widget 拷贝构造
    auto w2 = make<Widget>(std::string("Beta"));  // 临时 string 是右值 → Widget 移动构造
}
```

输出：

```
拷贝构造 Widget: Alpha
移动构造 Widget: Beta
```

如果不用 `forward`、直接写 `new T(args...)`，两行都会走拷贝构造——因为 `args` 在函数体内有了名字、成了左值。`forward` 是唯一能把"右值身份"还回去的手段。

------

## 直观区分：move vs forward

```cpp
void sink(int& x)  { std::cout << "左值 (" << x << ")\n"; }
void sink(int&& x) { std::cout << "右值 (" << x << ")\n"; }

template<typename T>
void relay(T&& arg)
{
    // sink(arg);                  // 永远左值——arg 有名字
    // sink(std::move(arg));       // 永远右值——无脑转
    sink(std::forward<T>(arg));    // 还原原始身份——该左就左，该右就右
}
```

------

## 总结：完美转发的配方

1. 参数写 `T&&`（通用引用），让 `T` 被推导。
2. 传递时写 `std::forward<T>(arg)`，让身份还原。
3. 如果和可变参数配合（如同上面 `make` 的例子），就用 `Args&&...` + `forward<Args>(args)...`。

这三条你背熟，写出来的模板函数在转发效率上就和中止库一个水准了。
