# 特殊人格 · 模板特例化

模板的理念是"一套代码适用所有类型"。但有时候，某一种具体类型需要完全不同的实现。比如标准库的 `vector<bool>` ——为了节省内存，它用位压缩而非每个 bool 占一字节。

**模板特例化**就是为"通用中的特殊"而生。当你写了一个函数模板或类模板，又想对某个特定类型给予专属待遇——就写它的特例化版本。

------

## 全特化：给某个具体类型写一套单独的

```cpp
// 通用版本
template<typename T>
class Comparator
{
public:
    bool greater(const T& a, const T& b) const
    {
        return a > b;
    }
};

// 对 const char* 的全特化版本——比较字符串内容，不比较地址
template<>
class Comparator<const char*>
{
public:
    bool greater(const char* a, const char* b) const
    {
        return strcmp(a, b) > 0;
    }
};

int main()
{
    Comparator<int> ci;
    cout << ci.greater(5, 3) << "\n";          // 1（5 > 3）

    Comparator<const char*> cs;
    cout << cs.greater("zoo", "apple") << "\n"; // 1（zoo 字典序在 apple 之后）
}
```

全特化写法：原模板的 `template<typename T>` 变成 `template<>`，类名后跟 `<具体类型>`。

函数模板只支持全特化，不支持偏特化。如果需要偏特化一个函数的行为，变通方法是把它包装成类模板的静态成员函数。

------

## 偏特化：按"模式"特化

不想特化到某一个具体的类型，而是特化到一种**类型的模式**——比如"所有指针类型"或"两个类型相同时"：

```cpp
// 通用版本
template<typename T>
class Storage { /* 默认实现 */ };

// 偏特化——如果 T 是指针
template<typename T>
class Storage<T*>
{
    // 针对指针类型的特殊处理（如自动 delete 等）
};

// 偏特化——两个类型相同时
template<typename T>
class Pair<T, T>
{
    // 同类型对子的特化逻辑
};
```

标准库的 `std::is_pointer<T>`、`std::is_same<A, B>` 等类型特质就是用偏特化 + 继承生成的——通用版本返回 `false_type`，特化版本返回 `true_type`。这个模式虽然看着复杂，原理就是一个"判断匹配特化规则 → 吐出 bool 常量"的机械过程。

------

## 写好一个判断"是否是指针"的类型特质

```cpp
// 通用——非指针类型
template<typename T>
struct is_pointer
{
    static constexpr bool value = false;
};

// 偏特化——匹配指针模式
template<typename T>
struct is_pointer<T*>
{
    static constexpr bool value = true;
};

int main()
{
    cout << is_pointer<int>::value << "\n";    // 0（false）
    cout << is_pointer<int*>::value << "\n";   // 1（true）
    cout << is_pointer<string*>::value << "\n"; // 1
}
```

标准库的 `std::is_pointer` 比这更复杂（它还要去除顶层 const/volatile 等），但原理完全相同——模板匹配规则 + 偏特化 = 编译期类型判定。

------

## 变量模板也可以特化

C++14 起模板变量也能特化：

```cpp
template<typename T>
constexpr T pi = T(3.14159265358979);

template<>
constexpr float pi<float> = 3.14f;    // 对 float 不需要那么高的精度

int main()
{
    cout << pi<double> << "\n";           // 3.14159265358979
    cout << pi<float> << "\n";            // 3.14
}
```

------

特例化本质是把模板编程从"一刀切"扩展成"一套规则+批量例外"的模式。实际项目中，自定义特例化的出现频率不高——但看标准库源码时，处处都是。理解它，你才能看懂为什么 `vector<bool>` 特殊、为什么 `is_pointer` 能编译期返回 bool、以及那些令人眼花缭乱的 `<type_traits>` 到底是什么机制在运作。
