# 偷懒的诀窍 · 合成函数与虚析构

如果你不写构造函数、析构函数、拷贝/移动函数，编译器会"替你省事"——帮你逐一合成一套默认的。绝大多数时候这套默认行为没问题。但当你跨进了继承、或手动管起了资源，这些合成就可能变成暗坑。

------

## 编译器偷偷帮你生成啥

如果你完全没有声明，编译器会自动合成这六个：

1. 默认构造函数
2. 析构函数
3. 拷贝构造函数
4. 拷贝赋值运算符
5. 移动构造函数（C++11）
6. 移动赋值运算符（C++11）

生成的规则是"逐个成员"走它们各自该走的流程——内置类型成员不初始化，类类型成员调它们对应构造/析构。

这里面有一条非常容易被忽视的规则：**你只要手动声明了析构函数，编译器就不再自动生成移动构造函数和移动赋值运算符**。后果就是：你的类在需要移动时，会静默降级为拷贝——没有任何警告，悄无声息。

**解决方案**：如果你手写了析构但又需要移动，用 `= default` 把它俩要回来：

```cpp
class MyClass
{
public:
    ~MyClass() { /* 关闭日志文件句柄 */ }
    MyClass(MyClass&&) = default;              // 把移动构造要回来
    MyClass& operator=(MyClass&&) = default;   // 把移动赋值要回来
};
```

------

## 虚析构——任何基类都不该省略

没有虚析构的继承是经典炸弹：

```cpp
class Base
{
public:
    ~Base() { cout << "Base 析构\n"; }
};

class Derived : public Base
{
private:
    int* data;
public:
    Derived() : data(new int[100]) {}
    ~Derived() { delete[] data; cout << "Derived 析构\n"; }
};

int main()
{
    Base* ptr = new Derived();
    delete ptr;      // 只输出 "Base 析构"——data 代表 100 个 int 永远泄漏了！
}
```

`ptr` 静态类型是 `Base*`，`Base` 的析构不是虚的 → 编译器老老实实只调了 `Base::~Base()`，派生类部分的清理全被跳过去了。

**修正只有一行**：

```cpp
virtual ~Base() = default;
```

这条规则很硬：任何有可能被继承的类（即你预期将来有派生类指向它的指针去 delete），都必须把析构标 `virtual`。这条没做到位就是内存泄漏的无声炸药。

------

## `= default` 和 `= delete`

- `= default`：告诉编译器"我就要默认的行为，帮我生成"。适用于想把被压制掉的默认函数恢复出来的场景。
- `= delete`：禁止调用。适用于"不希望外界拷贝我的资源"，比如 `unique_ptr`：

```cpp
class NonCopyable
{
public:
    NonCopyable() = default;
    NonCopyable(const NonCopyable&) = delete;           // 禁止拷贝
    NonCopyable& operator=(const NonCopyable&) = delete;
    NonCopyable(NonCopyable&&) = default;               // 允许移动
    NonCopyable& operator=(NonCopyable&&) = default;
};
```

## 三/五法则的口诀

如果你需要手动写以下五个函数中的**任何一个**：

- 析构函数
- 拷贝构造函数
- 拷贝赋值运算符
- 移动构造函数
- 移动赋值运算符

那大概率这五个你都需要整理一遍（至少前三者必须成组处理）——因为你正在管理不普通的资源，编译器逐个成员的默认行为在资源所有权上迟早要坑你。建议的顺序：先确定资源的生命周期和转移规则，再逐一写出（或用 `= default`/`= delete` 明确表达意图），最后确保基类的虚析构不漏掉。
