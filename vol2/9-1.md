# 无中生有 · 初识动态内存

到目前为止，我们创建的变量都住在**栈**（Stack）上。栈内存的特点是：你在哪个花括号里声明了变量，等出了这个花括号，变量就自动销毁，所占的空间也自动归还了。方便是方便，但也有两个局限：

1. **大小固定**——编译时就得知道要多大空间。你不能在运行时才决定"嘿，这数组得根据用户输入来定大小"。
2. **生命期可预计**——变量活了多久取决于它的作用域，出了作用域就没命了。有时你希望数据活得长一些——比如函数把计算结果还回去还在用，数据结构还需要保持这块数据。

这就需要**动态内存**——在运行时从**堆**（Heap）上申请空间，大小和生命期都由程序员自己决定。这部分空间不会随着作用域结束而自动释放——你得手动归还。

------

**new 与 delete：最原始的动态内存工具**

C++ 中，用 `new` 申请空间，用 `delete` 归还空间：

```cpp
int* p = new int;       // 在堆上申请一个 int 的空间，p 指向它
*p = 42;                // 往这块空间里写值
cout << *p << endl;     // 读值 => 42
delete p;               // 归还空间——千万记得！
```

`new` 做了两件事：
1. 在堆上分配一块足以装下 `int` 的内存。
2. 返回这块内存的地址（一个指针）。

`delete` 做相反的事——把你申请的那块内存回收，还给操作系统。

如果你忘了 `delete`，那块堆里的内存就一直占坑不还——这叫**内存泄漏**。时间长了，程序会越占越多，最终系统扛不住。

------

**动态分配数组**

```cpp
int n;
cin >> n;
int* arr = new int[n];       // n 是运行时才决定的，这对栈数组是做不到的

for (int i = 0; i < n; ++i)
    arr[i] = i * 10;

delete[] arr;                // 注意：释放数组要用 delete[]，不是 delete！
```

`new[]` 和 `delete[]` 必须配对。如果用 `new[]` 申请、却用 `delete`（忘了中括号）来还，行为是**未定义**的——可能导致只释放了第一个元素、后面的全泄漏，也可能直接崩溃。

------

**动态分配对象**

```cpp
class Point
{
public:
    int x, y;
    Point(int a, int b) : x(a), y(b) {}
};

Point* p = new Point(10, 20);   // 在堆上创建 Point 对象，同时调构造函数
cout << p->x << ", " << p->y << endl;  // -> 用来通过指针访问成员
delete p;
```

`new` 也可以调用构造函数。`new Point(10, 20)` 相当于在堆上"造"了一个 `Point(10, 20)` 的对象。

------

**堆与栈的关键区别**

| | 栈 | 堆 |
| ---- | ---- | ---- |
| 谁来管理 | 编译器自动管 | 程序员手动管 |
| 大小 | 通常有限（MB 级） | 系统能给的都行 |
| 速度 | 极快（移动栈指针就行） | 较慢（要查表找空闲块） |
| 适合放什么 | 小的、临时的、确定大小的 | 大的、不确定大小的、活的久的 |
| 会不会泄漏 | 不可能（自动释放） | 可能（忘记 delete） |

------

**有了 new/delete，为什么还要智能指针？**

你可能会想："只要我每次 new 完记得 delete，不就没事了？"

理论上对，但实际上：

- 函数可能有多条返回路径，每条都得记得 delete。
- 中间可能抛了个异常，delete 还没走到，程序就跳去 catch 了。
- 代码被别人改来改去，后来的人可能不清楚这块内存是谁该还的，漏掉了。

手动 new/delete 就像用完煤气要记得关阀门——简单归简单，忘了一次后果可就不堪设想。编译器不会检查你忘了没有，于是这成了 C++ 程序中最臭名昭著的内存 bug 来源。

C++11 引入的**智能指针**就是为此而生的——它帮你自动管理"什么时候该释放内存"。你把 new 出来的指针交到智能指针手里，后面的一切你都无需操心——我们下一节就来讲它们。
