# 绘制蓝图 · 初试定义模板

回忆一下：函数重载让我们可以为不同的类型写同名函数。但你发现没有——重载了几个版本的函数之后，代码其实相当一致，仅仅是类型不一样罢了：

```cpp
int max(int a, int b)           { return a > b ? a : b; }
double max(double a, double b)  { return a > b ? a : b; }
// 每多一种类型就要再抄一遍……烦不烦？
```

C++ 的**模板**就是为了解决这个问题而生的。它让你写一次代码，让编译器为不同的类型各自生成对应的版本——你写的是蓝图，编译器照着蓝图替你生产出具体代码。

------

**函数模板：一个最简单的开始**

```cpp
template<typename T>
T max(T a, T b)
{
    return a > b ? a : b;
}

// 使用
int a = max(3, 5);               // T 被推导为 int
double d = max(3.14, 2.71);      // T 被推导为 double
string s = max(string("z"), string("a")); // T 被推导为 string
```

`template<typename T>` 是在说："这里不写死类型，我用 `T` 代表一种还不确定的类型，等调用的时候再定。"

当编译器看到 `max(3, 5)`，它推导出 `T = int`，然后像复印机一样生成一份 `max` 的 `int` 版本。这整个过程是在编译期完成的，运行时不增加任何开销——也就是**零成本抽象**。

一些老代码用 `template<class T>` 写法——`class` 和 `typename` 在这里是**同义**的。不过 `typename` 更直观（T 可以是任何类型，不限于类类型），现代 C++ 倾向于用 `typename`。

------

**使用多个模板参数**

```cpp
template<typename T1, typename T2>
auto add(T1 a, T2 b) -> decltype(a + b)    // 尾置返回类型
{
    return a + b;
}

auto result = add(3, 4.5);    // T1 = int, T2 = double, result = 7.5
```

当两个参数类型不同时，返回类型需要告诉编译器"返回 `a + b` 的结果类型"。`auto ... -> decltype(...)` 这个语法叫**尾置返回类型**，靠着 `decltype` 让编译器根据表达式的实际类型自己把返回类型抠出来。

如果省略尾置返回类型直接写 `auto add(T1 a, T2 b)`——这在 C++14 起，`auto` 返回类型在仅有一个确定返回语句时是可以推断的。所以 C++14 及以上可以直接写：

```cpp
template<typename T1, typename T2>
auto add(T1 a, T2 b)
{
    return a + b;
}
```

------

**类模板：泛型的容器**

类也可以做成模板。标准库中最典型的类模板就是 `vector<T>`——`T` 可以是任何类型。

```cpp
template<typename T>
class Box
{
private:
    T content;
public:
    Box(const T& item) : content(item) {}
    T get() const { return content; }
    void set(const T& item) { content = item; }
};

Box<int> intBox(42);
Box<string> strBox("Hello");
```

注意：`Box<int>` 和 `Box<string>` 是两个**完全不同的类型**，由编译器各自生成一份完整的类定义。你不可以在需要 `Box<int>` 的地方塞一个 `Box<string>`。

------

**模板的声明与定义通常放在头文件中**

模板不是普通的函数或类——它是"蓝图"。编译器需要看到蓝图的完整内容才能在调用时为你生成具体版本。所以，**模板的定义通常都要写在头文件中**，不能像普通函数那样把声明放 `.h`、实现放在 `.cpp`。

如果你把它们分开，链接期会找不到实现，报一堆 `undefined reference`。有些构建模式（如显式实例化）可以绕过，但对初学者来说，记住"模板全要写头文件里"足够了。

------

**非类型模板参数**

模板参数不一定是类型，也可以是一个**编译期已知的值**：

```cpp
template<typename T, size_t N>
class FixedArray
{
private:
    T data[N];       // N 必须在编译期已知，因为这是静态数组大小
public:
    size_t size() const { return N; }
};

FixedArray<int, 10> arr;   // 一个能装 10 个 int 的静态数组
```

`std::array<int, 5>` 就是这么实现的——第二个模板参数是大小，一个编译期的整数。

非类型模板参数只能是**编译期的常量**（整型、枚举、指针等，C++20 起也支持浮点和类类型）。你不能传一个运行时才能得到的值给非类型参数。

模板是 C++ 泛型编程的基石。从现在到以后，只要你写 `vector<int>`、`map<string, int>`，你都在用模板。下几节，我们探索它更强大、更灵活的用法。
