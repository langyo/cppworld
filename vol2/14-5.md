# 俄罗斯套娃 · 局部类与嵌套类

类不一定非得放在全局作用域里。你可以把它塞进另一个类的内部（嵌套类），甚至塞进一个函数体的内部（局部类）。这些"套娃"手段在特定场景下能让你把类型的可见性控制在最窄的范围内——外面的代码连这个类型名字都碰不到。

------

## 嵌套类：把辅助类型关在大类里面

经典场景：二叉树的节点只在树的内部有用，不需要让外部看到。

```cpp
#include <iostream>
#include <memory>

class BinaryTree
{
private:
    struct Node                    // 嵌套类——对 BinaryTree 外部世界完全不可见
    {
        int value;
        std::unique_ptr<Node> left;
        std::unique_ptr<Node> right;

        Node(int v) : value(v) {}
    };

    std::unique_ptr<Node> root;

    // 递归辅助——也藏起来
    void insert(Node*& node, int value)
    {
        if (!node)
        {
            node = new Node(value);
            return;
        }
        if (value < node->value)
            insert(node->left.get() ? &(*node->left) : nullptr, value);
        // ... 实际代码会更完整，这里保留示意
    }

    void inorder(const Node* node) const
    {
        if (!node) return;
        inorder(node->left.get());
        std::cout << node->value << " ";
        inorder(node->right.get());
    }

public:
    void insert(int value)
    {
        // root 裸指针到 unique_ptr 的转换需要妥善处理；此处略
    }

    void print() const
    {
        inorder(root.get());
        std::cout << "\n";
    }
};
```

外部代码完全不知道 `Node` 这个类型的存在。你一眼就能看出"Node 只是 BinaryTree 的实现细节"。

## 嵌套类与访问权限：他们不是天然的朋友

嵌套类只是**名字上的隶属关系**，并不会自动获得访问宿主类 `private` 成员的特权。反过来也是——宿主类也不能自由读取嵌套类的 `private` 成员：

```cpp
class Outer
{
private:
    int secret = 42;

    class Inner
    {
    public:
        void peek(Outer& o)
        {
            // std::cout << o.secret;    //  不能访问，Inner 不是 Outer 的友元
        }
    };

public:
    void test()
    {
        Inner in;
        // std::cout << in.somePrivateMember;   //  Outer 也不能访问 Inner 的私有成员
    }
};
```

如果确实需要互相访问私有成员，需要显式的 `friend` 声明。但想想看——如果两个类需要彼此访问私有细节，那它们之间的耦合已经深到是时候重新审视设计的时候了。

## 局部类：塞在函数里的类

一个类可以在函数体内部定义——它只在这个函数内部可见和可用：

```cpp
#include <iostream>
#include <vector>
#include <algorithm>

void topKReport(const std::vector<int>& data, int k)
{
    // 只在本次调用里出现的辅助类型
    class TopKAccumulator
    {
    private:
        std::vector<int> heap;     // 最小堆维护 Top-K
        int k;
    public:
        TopKAccumulator(int k_) : k(k_) { heap.reserve(k); }

        void push(int val)
        {
            heap.push_back(val);
            std::push_heap(heap.begin(), heap.end(), std::greater<int>());
            if (heap.size() > k)
            {
                std::pop_heap(heap.begin(), heap.end(), std::greater<int>());
                heap.pop_back();
            }
        }

        void print() const
        {
            auto copy = heap;
            std::sort(copy.begin(), copy.end(), std::greater<int>());
            for (int x : copy) std::cout << x << " ";
            std::cout << "\n";
        }
    };

    TopKAccumulator acc(k);
    for (int x : data)
        acc.push(x);

    std::cout << "最大的 " << k << " 个数: ";
    acc.print();
}

int main()
{
    std::vector<int> scores = {85, 92, 78, 95, 88, 76, 99, 81};
    topKReport(scores, 3);     // 输出: 最大的 3 个数: 99 95 92
}
```

整个 `TopKAccumulator` 类完完全全只活在 `topKReport` 这个函数的一次调用里。外面不仅不知道这个类型的存在，连"这个函数内部有人实现了 Top-K"这件事也完全没有痕迹。

局部类有一些不自由的限制：它的成员函数必须在类内部直接定义（不能在外面写 `TopKAccumulator::push`），不能有静态成员，不能访问函数外的变量（除非通过参数传进来）。这些限制加上它的极窄作用域，使它更适合"一个函数内部的临时重型逻辑"，而不是日常随手用。

## 用嵌套类做迭代器

标准库中绝大多数容器的迭代器就是用嵌套类实现的——比如 `std::vector<int>::iterator` 就是 `std::vector` 内部定义的一个嵌套类（或类型别名指过去的）。你要自定义容器时也得这么干——把迭代器类定义成容器的 public 嵌套类，外部用 `MyContainer::iterator` 来引用。

```cpp
class IntRange
{
private:
    int start, end;

public:
    IntRange(int s, int e) : start(s), end(e) {}

    class iterator
    {
    private:
        int current;
    public:
        explicit iterator(int c) : current(c) {}
        int operator*() const { return current; }
        iterator& operator++() { ++current; return *this; }
        bool operator!=(const iterator& other) const { return current != other.current; }
    };

    iterator begin() const { return iterator(start); }
    iterator end()   const { return iterator(end); }
};

int main()
{
    IntRange range(1, 5);
    for (int x : range)
        std::cout << x << " ";    // 1 2 3 4
}
```

这里的 `iterator` 嵌套在 `IntRange` 里面，"迭代器"和"范围"两个概念的就近绑定在代码上格外自然——`IntRange::iterator`，不懂的人也一眼能看明白它在谁身上干活。

------

嵌套类和局部类都不是你每天要写的工具，但它们解决了一类问题：**类型的可见域应该有多窄**。一个只服务于特定逻辑的辅助类型，放在全局域是信息泄露，放在函数域或类域则是精准的封装。下次你在写"这个 struct 应该只在这里用"的时候，想想把它往里面缩一缩，正是 C++ 信息隐藏给您的最后一重手段。
