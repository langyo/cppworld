# 匹配想法 · 正则表达式

你在某个字符串里要找特定的模式——比如把一个文本里的所有邮箱地址提取出来，或者判断用户输入的手机号格式对不对。靠手写循环和字符判断也能做，但代码又臭又长又一堆边界情况。

**正则表达式**就是为"模式匹配"而生的专用语言。C++11 把它收入标准库，放在 `<regex>` 头文件里。

------

**一个最简单的匹配**

```cpp
#include <iostream>
#include <regex>
#include <string>
using namespace std;

int main()
{
    string text = "我的邮箱是 someone@example.com，请联系我。";

    regex pattern(R"(\w+@\w+\.\w+)");    // 邮箱匹配模式

    smatch matches;
    if (regex_search(text, matches, pattern))
        cout << "找到邮箱: " << matches[0] << endl;
    // 输出：找到邮箱: someone@example.com

    return 0;
}
```

`R"(...)"` 是 C++ 的**原始字符串字面量**。平时字符串里的反斜杠需要写 `\\`，在正则里反斜杠本来就多，如果再加上 C++ 的转义，就变成了 `\\w` ——看着累。用原始字符串就可以原样写入 `\w`、`\.` 等，无需二次转义。原始字符串的格式是 `R"分隔符(内容)分隔符"`，分隔符可自定义；常用的就是最简单的 `R"(...)"`。

`regex_search` 在字符串中搜索第一个匹配项并填入 `matches`。`matches[0]` 是整个匹配到的内容。

------

**正则表达式语法速查**

常用模式元素：

| 模式 | 含义 |
| ---- | ---- |
| `\d` | 一个数字 |
| `\w` | 一个"单词字符"：字母、数字、下划线 |
| `\s` | 一个空白字符（空格、制表符、换行） |
| `.` | 任意一个字符 |
| `+` | 前面的模式出现 1 次或更多次 |
| `*` | 前面的模式出现 0 次或更多次 |
| `?` | 前面的模式出现 0 次或 1 次 |
| `{n}` | 前面的模式正好出现 n 次 |
| `[abc]` | 中括号内的任意一个字符 |
| `(pattern)` | 捕获组——匹配的同时把匹配的部分单独记下来 |

------

**捕获组：不仅匹配，还摘取**

```cpp
string text = "出生日期：1995-03-21";

regex pattern(R"((\d{4})-(\d{2})-(\d{2}))");   // 三个捕获组
smatch matches;

if (regex_search(text, matches, pattern))
{
    cout << "完整匹配: " << matches[0] << endl;   // 1995-03-21
    cout << "年: "       << matches[1] << endl;   // 1995
    cout << "月: "       << matches[2] << endl;   // 03
    cout << "日: "       << matches[3] << endl;   // 21
}
```

`matches[0]` 永远是整个匹配段落，`matches[1]` 是第一组括号捕获的内容，`matches[2]` 是第二组……以此类推。

------

**regex_match vs regex_search vs regex_replace**

| 函数 | 含义 |
| ---- | ---- |
| `regex_match` | 整个字符串**完全匹配**模式——从头到尾一丝不差 |
| `regex_search` | 在字符串中**找到**任何匹配模式的**子串** |
| `regex_replace` | 把匹配的部分**替换**成另一个字符串 |

```cpp
regex pattern(R"(\d+)");
string s = "abc 123 def";

cout << regex_match(s, pattern) << endl;      // false——"abc 123 def" 不全是数字
cout << regex_search(s, pattern) << endl;     // true——其中包含了 "123"

string replaced = regex_replace(s, pattern, "***");
cout << replaced << endl;                     // abc *** def
```

------

**迭代匹配：找出所有的匹配项**

```cpp
string text = "电话号码：123456，另一个号码：987654，还有一个：555555";
regex pattern(R"(\d+)");

auto begin = sregex_iterator(text.begin(), text.end(), pattern);
auto end   = sregex_iterator();

for (auto it = begin; it != end; ++it)
    cout << "找到: " << it->str() << endl;
// 输出三次——123456, 987654, 555555
```

`sregex_iterator` 用起来和标准的容器迭代器一模一样——从这个角度说，它让"搜索所有匹配"变成了一种遍历行为，可以直接塞进 `for_each` 甚至跟 `vector` 互传迭代器。`s` 前缀表示它是处理 `string` 的；对应的还有 `cregex_iterator` 用于 C 风格字符串。

------

**正则标志**

`regex` 构造时可以指定标志：

```cpp
regex pattern(R"(hello)", regex::icase);   // 忽略大小写——Hello、HELLO 都匹配
```

其它常用标志：`regex::multiline`（`^` 和 `$` 匹配每行的行首行尾而不是整个文本的首尾）、`regex::dotall`（`.` 也可以匹配换行符）。这些跟其它语言的正则标志非常类似。

------

正则表达式本身就是一个博大精深的体系——这里只是一个速览，让你能够拿起它们就开始匹配字符串。要注意的是，C++ regex 在某些编译环境下性能不是最快的（尤其是 GCC 的旧版本），处理超大量文本时建议先做基准测试。但对于一般批量的日志解析、表单验证，它足够胜任。
