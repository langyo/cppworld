# 换一种走路的方式 · 递增递减运算符定义

如果 `Point` 表示二维平面上一个点的位置，让点移动最自然的操作就是"加一个偏移"。但有时候，我们只希望在某个方向上挪一步——比如往右走一格，或往上走一格：

```cpp
Point p{10, 20};
++p;    // 往右一步？往上一步？还是都有？
```

这时候我们可以重载 `++` 运算符。

不过 `++` 有一个小陷阱：它有前置（`++p`）和后置（`p++`）两种形式。C++ 用一个巧妙的机制来区分它们。

------

**重载前置 `++`**

前置版本没有额外参数，返回修改后的对象的引用：

```cpp
class Point
{
public:
    int x, y;

    Point& operator++()       // 前置 ++
    {
        ++x;
        ++y;
        return *this;         // 返回修改后的自己
    }
};

Point p{10, 20};
++p;              // p 现在是 {11, 21}
```

返回引用 `Point&` 而不是值 `Point` 是为了支持链式调用，比如 `++(++p)`。如果返回值类型，第二次 + + 会在一个临时的副本上操作，白白丢失第一次的结果。

------

**重载后置 `++`**

后置版本需要一个额外的 `int` 参数来区分——这个 `int` 只是个**占位符**，用来告诉编译器"这是后置版本"，你不用传实际的值进去，编译器会自动当它不存在：

```cpp
class Point
{
public:
    int x, y;

    Point operator++(int)    // 后置 ++，注意 int 占位
    {
        Point old = *this;   // 先保存旧值
        ++x;
        ++y;
        return old;          // 返回旧值
    }
};

Point p{10, 20};
Point q = p++;   // q = {10, 20}（旧值），p = {11, 21}（新值）
```

后置版本返回的是旧值的**副本**（不是引用！），因为 `old` 是个局部变量，函数结束后就销毁了，只能考拷贝的方式把值带出去。

通常前置版本比后置版本效率更高（少了一次拷贝），所以优先用前置 `++`。除非你真的需要"先用旧值再自增"的语义（比如经典 for 循环尾部），否则别写 `p++`。

------

**重载递减 `--`**

完全对称，把 `++` 换成 `--` 就行：

```cpp
class Point
{
public:
    int x, y;

    Point& operator--()       // 前置 --
    {
        --x; --y;
        return *this;
    }

    Point operator--(int)     // 后置 --
    {
        Point old = *this;
        --x; --y;
        return old;
    }
};
```

------

**举一反三：`+=` 和 `-=`**

递增递减是"一次一步"，但很多时候我们想"一次走好几步"——`+=` 更合适：

```cpp
class Point
{
public:
    int x, y;

    Point& operator+=(const Point& other)
    {
        x += other.x;
        y += other.y;
        return *this;
    }
};
```

然后你就可以这样走路了：

```cpp
Point p{10, 20};
Point moveRight{1, 0};
Point moveUp{0, 1};

p += moveRight;     // {11, 20}
p += moveRight;     // {12, 20}
p += moveUp;        // {12, 21}
```

有了 `+=`，`+` 也就顺手了——通常你可以用 `+=` 来实现 `+`：

```cpp
Point operator+(Point a, const Point& b)
{
    a += b;     // 复用 +=
    return a;   // 返回副本（不修改原来的 a——这里 a 本身是按值传进来的）
}
```

这样维护起来也省心——但凡要改加法逻辑，只需要改 `operator+=` 一处，`operator+` 自然就跟上了。

------

总结：**递增递减、复合赋值（`+=`、`-=`）返回引用，算术运算（`+`、`-`）返回值（副本）**。这是 C++ 社区的共识写法，照这个模板来就不会出错。
