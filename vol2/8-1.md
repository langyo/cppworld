# 取之不竭 · 顺序容器

在第六章我们认识了 `vector`——一列能随时加长缩短的小格子。它是标准库里出场率最高的容器，但并非唯一的容器。

C++ 标准库提供了好几种**顺序容器**——它们的共同特点是：元素按顺序排好，每个元素位于固定的位置。区别就在于"在什么位置插入/删除"的效率不同，以及额外占用的内存不同。

------

**顺序容器全家福**

| 容器 | 全称 | 一句话概括 |
| ---- | ---- | ---------- |
| `vector` | 向量 | 尾部增删飞快，随机访问快，中间的增删慢 |
| `deque` | 双端队列 | 两头增删飞快，随机访问快 |
| `list` | 双向链表 | 任意位置增删飞快，不能随机访问 |
| `forward_list` | 单向链表 | 只有 forward 方向，比 list 更省内存 |
| `array` | 定长数组 | 上一章讲过了，大小固定 |

大多数时候选 `vector`，因为现代 CPU 喜欢连续内存，缓存命中率高。当需要在头部频繁增删时考虑 `deque`；当需要在中间反复插入删除以致于 vector 的搬移成本忍不了时，再考虑 `list`。

------

**deque——前后都能快速存取**

`deque` 读作 "deck"，全称 double-ended queue（双端队列）。跟 `vector` 相比，它多支持了高效的头插和头删：

```cpp
#include <deque>
using namespace std;

deque<int> d = {1, 2, 3};
d.push_back(4);      // 尾插 => {1, 2, 3, 4}
d.push_front(0);     // 头插 => {0, 1, 2, 3, 4}
d.pop_back();        // 尾删 => {0, 1, 2, 3}
d.pop_front();       // 头删 => {1, 2, 3}
```

除此之外，`deque` 的其它用法和 `vector` 几乎一样——下标、迭代器、`at()`、`size()` 一应俱全。

很多人（包括不少面试官）以为 `deque` 必须是用链表实现，其实 C++ 标准没有规定确切的实现方式。最常见的实现是**分段连续**：用一块指针数组指到多段固定大小的物理连续块，逻辑上给人"一整块可随机访问的空间"的感受。它在头部插入的时间复杂度是 O(1)（常数级），而 `vector` 是 O(n)（要把所有元素整体后移）。

------

**list——双向链表**

`list` 是标准的双向链表。每个元素有指向前一个和后一个的指针。在任意位置插入删除都是常数时间，但不能通过下标随机访问（没有 `[]` 运算符）：

```cpp
#include <list>
using namespace std;

list<int> lst = {1, 2, 3};
lst.push_back(4);
lst.push_front(0);

// 不能写 lst[2]，只能靠迭代器
for (auto it = lst.begin(); it != lst.end(); ++it)
    cout << *it << " ";
```

`list` 自带一些专门的成员函数——`sort()`、`merge()`、`reverse()`、`unique()`、`remove()` 等——它们比标准库 `<algorithm>` 里同名的通用算法对链表更高效，因为后者依赖随机访问。

什么时候用 list？当你确认要频繁地在中部插入删除、且数据的量很大导致 vector 的整体搬迁忍无可忍时。否则前面说的"CPU 喜欢连续内存"这一条就够让 vector 稳赢绝大部分场景。

------

**forward_list——单向链表**

`forward_list` 是 C++11 引入的，每个节点只存一个指向下一个的指针，更省内存，但只能单向遍历：

```cpp
#include <forward_list>
using namespace std;

forward_list<int> fl = {1, 2, 3};
fl.push_front(0);   // 只有头插，没有尾插（没存尾指针）
```

说实话，在大部分应用代码中 `forward_list` 的出镜率远比 `list` 还低——除非你极度在意那几个字节的内存开销并且就是想用链表。

------

**选择容器的原则**

一个常用的决策流程：

1. 默认用 `vector`。
2. 如果需要频繁在头部增删，考虑 `deque`。
3. 如果需要在中间频繁增删且数据量真的很大导致 vector 太痛苦了，考虑 `list`。
4. 如果空间是最最紧张的资源，考虑 `forward_list`。
5. 如果大小在编译期就能定死并且想要容器接口，考虑 `array`。

对于初学者，把 `vector` 用熟是最重要的。等你真的写出了"插入 10 万条数据、中间还要频繁删开头"的程序，再回头看其它容器也不迟。

------

**容器通用的操作**

以下操作对所有顺序容器（除了 `array` 的大小不可变、`forward_list` 的限制多一点）几乎都是通用的：

```cpp
// 构造
容器 c;                // 空容器
容器 c(n);             // n 个默认初始化的元素
容器 c(n, val);        // n 个 val
容器 c(other);         // 拷贝
容器 c = {1, 2, 3};    // 初始化列表

// 大小
c.size();
c.empty();
c.max_size();          // 最大容量——人一辈子用不到

// 增删
c.push_back(val);      // 尾插（forward_list 没有）
c.pop_back();          // 尾删（forward_list 没有）
c.clear();             // 清空

// 比较
c1 == c2;
c1 < c2;               // 字典序比较

// 赋值和交换
c1 = c2;
c1.swap(c2);           // O(1) 交换两个同类型容器的内容
```

特别要点一下 `swap`：它是 O(1)——只交换两个容器内部指向数据的指针，不搬元素。如果你需要一个"空容器"来接盘，`c.clear()` 或 `容器().swap(c)` 这两种写法都是常见手段（后者在 C++11 之前用的更多）。
