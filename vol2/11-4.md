# 遗产分配 · 访问控制与派生类作用域

面向对象的第三条腿是封装。在继承语境下，封装具体聚焦于一个关键问题：**派生类能看到基类的什么东西？外面再通过派生类又能看到什么？**

------

## 访问修饰符作用的总结

| 修饰符 | 自身类内部 | 派生类内部 | 外部代码（非派生） |
| ------ | ---------- | ---------- | ------------------ |
| `public` | 可以 | 可以 | 可以 |
| `protected` | 可以 | 可以 | **不可以** |
| `private` | 可以 | **不可以** | **不可以** |

`protected` 就是专门为继承准备的——让派生类能"接管"一部分基类的内部设施，同时保持对外部世界的封闭。

```cpp
class Shape
{
protected:
    int color;          // 派生类可以读/写 color
    void setStyle() {}  // 派生类可以调 setStyle
public:
    void draw() const { cout << "绘制图形\n"; }
};

class Circle : public Shape
{
public:
    void applyTheme()
    {
        color = 0xFF0000;       //  OK——派生类看得见 protected 成员
        setStyle();             //  OK——protected 成员函数也能调
    }
};

int main()
{
    Circle c;
    c.draw();                   //  OK——public 成员
    // c.color = 0;             //   外部看不到 protected
}
```

------

## 继承方式给访问级别"再加一层门禁"

大多数时候我们只用 `public` 继承：

```cpp
class Circle : public Shape { /* ... */ };
```

它保持原有的访问级别不降级——基类的 `public` 部分在派生类眼里依然是 `public`。

`protected` 继承和 `private` 继承是少见得多的设计选择：

```cpp
class InternalImpl : private Base { /* ... */ };
```

`private` 继承相当于"我用了基类的实现，但外面谁也看不出来"。现代 C++ 中大多数这类需求可以用组合（把 Base 作为成员变量）替代，`private` 继承主要存在于底层的设计模式实现中。

------

## 派生类中的作用域查找

当你在派生类的函数内写一个名字，编译器按这个顺序找：

1. 当前派生类内部
2. 直接基类内部
3. 间接基类（基类的基类）
4. 全局作用域

内层名字会掩盖外层的同名者：

```cpp
class Base
{
public:
    int value = 10;
    void show() { cout << "Base\n"; }
};

class Derived : public Base
{
public:
    int value = 20;            // 掩盖了 Base::value

    void show()                // 掩盖了 Base::show
    {
        cout << "Derived " << value << "\n";       // 20（自己的）
        cout << "Base " << Base::value << "\n";    // 10（显式用 Base::）
    }
};
```

有一个不同的情况：如果 `Base` 中的 `show` 是 `virtual` 的且 `Derived` 的 `show` 写了 `override`，那调用的就是多态覆盖而不是掩盖——这是动态绑定和名称掩盖之间的分界线，需要按场景准确区分。

## using 声明解除遮盖

如果派生类定义了任意一个跟基类同名的函数（包括重载的不同参数版本），基类那个名字的**全部重载版本**会被遮盖。用 `using` 把基类的版本恢复出来：

```cpp
class Base
{
public:
    void action(int x)    { cout << "Base int " << x << "\n"; }
    void action(double x) { cout << "Base double " << x << "\n"; }
};

class Derived : public Base
{
public:
    using Base::action;     // 把基类 action 的所有重载版本都开放出来

    void action(const string& s)    // 自己在上面再加一个版本的 action
    {
        cout << "Derived string " << s << "\n";
    }
};

int main()
{
    Derived d;
    d.action(10);           // Base int 10  —— 走基类版本
    d.action(3.14);         // Base double 3.14
    d.action("hello");      // Derived string hello —— 自己的追加版本
}
```

`using` 声明在派生类为基类接口"做加法"时的利器——原有的整组重载保留无悖，派生类自己在上面再挂新版本。
