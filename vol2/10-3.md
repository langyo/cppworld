# 换一种交流的方式 · 成员访问运算符定义

有些类天生就该"扮演"别的身份。比如智能指针——用起来像普通指针，但背地里帮你自动 `delete`。达成这种伪装靠两个运算符：**`*`（解引用）和 `->`（箭头）**。

------

## 从零写一个最小智能指针

```cpp
#include <iostream>

template<typename T>
class UniquePtr
{
private:
    T* raw;

public:
    explicit UniquePtr(T* p) : raw(p)
    {
        std::cout << "获取资源\n";
    }

    ~UniquePtr()
    {
        std::cout << "释放资源\n";
        delete raw;
    }

    // 禁用拷贝——独占所有权
    UniquePtr(const UniquePtr&) = delete;
    UniquePtr& operator=(const UniquePtr&) = delete;

    // 支持移动
    UniquePtr(UniquePtr&& other) noexcept : raw(other.raw)
    {
        other.raw = nullptr;
    }

    // * 运算符——像普通指针一样解引用
    T& operator*() const
    {
        return *raw;
    }

    // -> 运算符——像普通指针一样访问成员
    T* operator->() const
    {
        return raw;
    }

    // 判空
    explicit operator bool() const
    {
        return raw != nullptr;
    }
};
```

试着用一下——它用起来跟指针几乎完全一样，但不用担心 `delete`：

```cpp
struct Point
{
    int x, y;
    Point(int a, int b) : x(a), y(b) {}
};

int main()
{
    UniquePtr<Point> p(new Point(10, 20));

    std::cout << "x = " << p->x << ", y = " << p->y << "\n";  // -> 自动生效
    (*p).x = 99;                                               // *  自动生效

    if (p)
        std::cout << "指针非空\n";

    // 离开作用域时自动 delete——不需要手动释放
}
```

这里 `p->x` 被编译器翻译成 `(p.operator->())->x`——先调 `operator->` 拿到 `Point*`，再对 `Point*` 做原生的 `->`。这个链式解引用机制是 C++ 标准特意支持的：只要 `operator->` 返的东西还能再 `->`，编译器就会一直解下去。

## `[]` 下标运算符——把类伪装成数组

如果你写了一个包装类，想让它支持 `wrapper[3]` 这种用法，重载 `[]` 就行：

```cpp
#include <vector>
#include <iostream>
#include <stdexcept>

class SafeVector
{
private:
    std::vector<int> data;

public:
    void push(int x) { data.push_back(x); }

    // 可写版
    int& operator[](size_t index)
    {
        return data[index];
    }

    // 只读版（const 对象调这个）
    const int& operator[](size_t index) const
    {
        return data[index];
    }

    // 可选：带越界检查的 at()
    int& at(size_t index)
    {
        if (index >= data.size())
            throw std::out_of_range("下标越界！");
        return data[index];
    }

    size_t size() const { return data.size(); }
};

int main()
{
    SafeVector sv;
    sv.push(10);
    sv.push(20);
    sv.push(30);

    sv[1] = 200;                       // 通过 [] 修改
    for (size_t i = 0; i < sv.size(); ++i)
        std::cout << sv[i] << " ";     // 通过 [] 读取
    std::cout << "\n";

    try
    {
        sv.at(10);                     // 越界——抛异常
    }
    catch (const std::out_of_range& e)
    {
        std::cerr << "异常: " << e.what() << "\n";
    }
}
```

const 版本和非 const 版本的 `[]` 要成对出现——因为 `const SafeVector` 对象只能调用带 `const` 修饰的成员函数。

------

这三个运算符——`*`、`->`、`[]`——加起来给了你一个能力：**让你的类用起来就像它正扮演的那个东西**。智能指针像指针，自定义容器像数组。调用者不需要知道底层有多复杂，看到 `->` 就知道怎么用。这也是为什么标准库的 `unique_ptr`、`shared_ptr`、`vector`、`map` 这些工具用起来如此顺滑——它们的作者正是把这些运算符打磨到了极致。
