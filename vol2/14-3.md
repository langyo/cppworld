# 摆脱单亲状态 · 多重继承与虚继承

前面讲的继承都是"一个人一个爹"——一个派生类只从一个基类来。C++ 允许一个类同时拥有多个父类。这个机制就是**多重继承**。

把话说在前头：多重继承在绝大多数日常代码中用不到。但有两样事你必须靠它——一是同时实现多个纯虚接口（这很安全），二是在翻旧代码和大型框架（比如 Qt、Unreal 的对象模型）时能看懂它。

------

## 基本语法

```cpp
class A
{
public:
    int a;
    void show() { std::cout << "A\n"; }
};

class B
{
public:
    int b;
    void show() { std::cout << "B\n"; }
};

class C : public A, public B     // 同时继承两个基类
{
public:
    int c;
};

int main()
{
    C obj;
    obj.a = 1;          // 来自 A
    obj.b = 2;          // 来自 B

    // obj.show();      //  二义——A 有 show，B 也有 show
    obj.A::show();      //  显式指明调 A 版本的 show
    obj.B::show();
}
```

多重继承遇到同名函数时必须显式用 `基类名::` 指明走哪条路。

------

## 钻石问题（菱形继承）

多重继承最经典的坑——一个基类被两条路间接继承下来，最终导致最底层的类中有两份基类数据：

```
     Person（name 字段）
    /      \
Student   Teacher（各继承了一份 name）
    \      /
   Assistant（得到两份 name！）
```

```cpp
class Person
{
public:
    std::string name;
};

class Student : public Person {};      // Person 的一份拷贝
class Teacher : public Person {};      // Person 的另一份拷贝

class Assistant : public Student, public Teacher {};

int main()
{
    Assistant a;
    // a.name = "张三";    //  二义——Student 里有一个 name，Teacher 里也有一个
    a.Student::name = "张三";
    a.Teacher::name = "李四";
}
```

"助理是人吗？"——“是，但他是两个不同的人（两份数据）。”这显然不符合直觉。

------

## 虚继承：让祖辈只有一个

在 `Student` 和 `Teacher` 的继承中用 `virtual` 告诉编译器："Person 只留一份"：

```cpp
class Person
{
public:
    std::string name;
};

class Student : virtual public Person {};    // 虚继承
class Teacher : virtual public Person {};    // 虚继承

class Assistant : public Student, public Teacher {};

int main()
{
    Assistant a;
    a.name = "张三";           //  只有一个 name，不再二义
    std::cout << a.name << "\n";
}
```

虚继承的代价：虚基类的初始化由**最底层的类直接负责**（而非由中间层代劳），内部的 vptr 布局也更复杂。因此不要无脑把所有继承都标 `virtual`——只在出现钻石结构时才用。

------

## 多重继承最安全的用法：实现多个接口

```cpp
class IDrawable
{
public:
    virtual void draw() const = 0;
    virtual ~IDrawable() = default;
};

class ISerializable
{
public:
    virtual std::string serialize() const = 0;
    virtual ~ISerializable() = default;
};

// 一个类实现两个接口——完全没有数据冲突
class Button : public IDrawable, public ISerializable
{
public:
    void draw() const override
    {
        std::cout << "[绘制按钮]\n";
    }

    std::string serialize() const override
    {
        return "{ \"type\": \"button\" }";
    }
};

int main()
{
    Button btn;
    btn.draw();

    IDrawable& d = btn;
    ISerializable& s = btn;
    std::cout << s.serialize() << "\n";
}
```

纯虚接口没有数据成员，多重继承它们不会产生钻石问题。很多时候 "C++ 没有 interface 关键字"这件事用一个简朴的多重纯虚继承就原地解决了。

------

多重继承在日常业务代码中应该谨慎使用（尤其是带数据成员的），但在接口设计、框架组织和跨语言调用层它是绕不开的基础功。把它和虚继承理解透了，再去看任何庞大的 C++ 代码仓库，对象的继承树都不会再让你害怕。
