# 铁索连环 · 变长元组 tuple

前面我们学过 `pair`——两样东西绑一起。但真实世界中经常要把不止两样东西一次性传递：比如一个人的姓名、年龄、地址，或者一次函数调用的多个返回值。

`tuple` 就是**任意个任意类型"绑在一起"的包**。在 `<tuple>` 头文件中。

------

**创建 tuple**

```cpp
#include <tuple>
#include <string>
using namespace std;

// 一条语句把姓名、年龄、分数绑在一起
tuple<string, int, double> student("张三", 20, 95.5);

// 或者用 make_tuple 自动推导
auto info = make_tuple("李四", 22, 88.0, "计算机系");
// info 的类型是 tuple<const char*, int, double, const char*>
```

------

**提取 tuple 中的元素**

```cpp
auto student = make_tuple("张三", 20, 95.5);

// 方式一：get<索引> —— 索引必须是编译期常量
cout << get<0>(student) << endl;  // "张三"
cout << get<1>(student) << endl;  // 20
cout << get<2>(student) << endl;  // 95.5
// get<3>(student);               // ❌ 编译错误，索引超出范围

// 方式二：结构化绑定（C++17）
auto& [name, age, score] = student;
cout << name << " " << age << " " << score << endl;
```

`get<索引>` 要求索引在编译期已知——用变量不行。它是零成本的：编译完成后，拿 tuple 的第二个字段和访问一个结构体第二个成员的效率完全一样。

结构化绑定在 C++17 中得到支持，用它来解包 tuple 无论写法还是可读性都是最清爽的选择。

------

**tuple 的常见用途**

1. **函数多返回值**：

```cpp
tuple<int, int, double> getStats(const vector<int>& v)
{
    int min = v[0], max = v[0];
    double sum = 0;
    for (int x : v)
    {
        if (x < min) min = x;
        if (x > max) max = x;
        sum += x;
    }
    return {min, max, sum / v.size()};   // C++17 起可直接用初始化列表返回
}

auto [min, max, avg] = getStats(data);
```

2. **多元素打包传递**：不想为临时组合专门定义一个 struct 时，`tuple` 就很方便。

3. **智能比较前奏**：可以用 `tie` 来实现一次性同时比较多个字段的排序——`tie(a.field1, a.field2) < tie(b.field1, b.field2)`。`tie` 是 `make_tuple` 的"引用版本"，返回一个各元素均为引用的 tuple，不产生拷贝。

------

**字典序比较：tuple 自带排序逻辑**

两个同构的 tuple（元素类型两两匹配）可以直接用 `<`、`==` 比较，按元素顺序字典序比较：

```cpp
tuple t1(1, string("a"));
tuple t2(1, string("b"));
cout << (t1 < t2) << endl;  // true——第二字段 "a" < "b"
```

------

**操作 tuple 的工具**

| 工具 | 作用 |
| ---- | ---- |
| `make_tuple` | 用已有的值创建 tuple |
| `tie` | 创建"引用型" tuple——元素全为引用，用于解包或逻辑比较 |
| `tuple_cat` | 拼接两个 tuple |
| `tuple_size<T>` | 获取 tuple 的元素个数，编译期常量 |
| `tuple_element<N, T>` | 获取 tuple 第 N 个位置的类型 |

`tie` 最常用：当你需要一次性把一组有名字的变量"打包成引用组"做字段级比较或直接解构时，`tie(a, b, c)` 就是答案——它生成的 tuple 指针向原来的变量，修改它会直接改原变量。

------

tuple 就是"不写 struct 的多元素组合"。对于临时性的、一次性把几样东西传来传去的场景，它很趁手。但如果这组数据以后要在多个地方反复出现，意味着它正在逐渐成为一个独立的抽象——那就值得为它取个名字，把它变成一个 struct 或 class，让类型名字本身成为一种文档式的说明。
