# 电报机 · 字符串流

有时候，我们想对一段字符串做"流式"操作——就像从键盘读数据一样从字符串中"读"，或者像往屏幕上打印一样"写"到一个字符串里。

C++ 提供了**字符串流**（string stream），藏在 `<sstream>` 头文件里。它三兄弟：

| 类 | 用途 |
| ---- | ---- |
| `istringstream` | 从字符串中"读" |
| `ostringstream` | "写"到一个字符串里 |
| `stringstream` | 既读又写 |

它们用起来和 `cin`/`cout`、文件流完全一样的语法，只是数据源从键盘/文件变成了内存中的字符串。如果你已经会 `cin`/`cout`，再学字符串流最多花十分钟。

------

**ostringstream：把输出装进字符串**

```cpp
#include <iostream>
#include <sstream>
#include <string>
using namespace std;

int main()
{
    ostringstream oss;

    oss << "姓名: " << "张三" << ", ";
    oss << "年龄: " << 25 << ", ";
    oss << "分数: " << 95.5;

    string result = oss.str();   // 把流里的内容导出为 string
    cout << result << endl;
    // 输出: 姓名: 张三, 年龄: 25, 分数: 95.5

    return 0;
}
```

在需要"拼接多个不同类型的数据到一个字符串中去"的时候，`ostringstream` 比 `+` 运算符拼接 `string` 更干净高效——尤其是当拼接次数很多时，它不会像反复 `+` 那样制造出一个又一个中间临时字符串对象。

------

**istringstream：从字符串中解析数据**

这是最常被用到的字符串流。比如拿到一行 "10 20 30" 这样的字符串，你想把数字一个一个提取出来：

```cpp
string line = "10 20 30";
istringstream iss(line);

int a, b, c;
iss >> a >> b >> c;         // a=10, b=20, c=30
```

或者从格式化的一行里按类型分开读：

```cpp
string record = "Alice 25 95.5";
istringstream iss(record);

string name;
int age;
double score;

iss >> name >> age >> score;
cout << name << " " << age << "岁，分数 " << score << endl;
```

字符串流和 `cin` 一样，空白字符是天然的分隔符，所以很适合解析以空格分隔的数据。

------

**完整的数据解析流程**

字符串流真正发威的场景是这样的：你在文件里读到一行行文本，每行有不同数量的数字，你需要全部解析出来：

```cpp
string line;
while (getline(cin, line))           // 一行行读
{
    istringstream iss(line);
    int sum = 0, val;
    while (iss >> val)               // 一个数字一个数字地读
        sum += val;
    cout << "本行数字之和: " << sum << endl;
}
```

每一行用一个新的 `istringstream` 包裹起来，然后就可以像读 cin 那样一直 `>>`，直到流里没东西为止。

------

**stringstream：读写双向**

```cpp
stringstream ss;
ss << 100 << " " << 200;     // 写入

int a, b;
ss >> a >> b;                // 读取 => a=100, b=200
```

不太常见。大部分时候 `istringstream`（只读）和 `ostringstream`（只写）已经够用，而且语义更清晰，一眼就能看出这段代码是读还是写。

------

**值转换的另一种方式**

在引入 `to_string` 和 `stoi` 等函数之前（C++98 时代），字符串流是实现数字和字符串互转的常用手段：

```cpp
// 数字 → 字符串（现在有 to_string 了，不过这个写法依然可行）
ostringstream oss;
oss << 3.14159;
string s = oss.str();

// 字符串 → 数字
istringstream iss("42");
int n;
iss >> n;
```

有了 `to_string` / `stoi` / `stod` 之后，字符串流转字符串的需求减少了一些，但处理复杂格式化时它仍然不可或缺。比如你要生成一串带前导零、指定小数位数的格式化数字——用流操纵符（`setw`、`setprecision` 等）直接写在流里，比 `to_string` 灵活得多。

------

**和其他流的配合**

字符串流的最大价值之一就是"连接各种流"——比如从一个文件流读入，交给字符串流做格式解析，最后从字符串流提取出强类型的值。文件流负责从磁盘拿字节，字符串流负责把它理解成数字和单词，两个职责完全分开了。

同样地，在写出复杂格式的输出文件时，可以先用 `ostringstream` 在内存中组装好完整的一行，再一次性写进 `ofstream`，避免频繁的小块 IO——这在网络编程或大文件输出时对性能有明显帮助。
