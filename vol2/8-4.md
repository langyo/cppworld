# 摄取与排放数据 · 文件输入输出

到目前为止，程序的所有输入输出都是在黑窗口里进行的——程序一关，数据不翼而飞。现实中，程序需要把数据存进文件，下次打开还能接着用；或者读别人写好的文件，提取信息。

C++ 标准库通过 `<fstream>` 头文件来支撑文件读写。它中三个核心类——`ifstream`（输入文件流）、`ofstream`（输出文件流）和 `fstream`（输入输出合一）——用起来和 `cin`/`cout` 是同一套语法，只是方向从屏幕换成了文件。

------

## 写入文件

```cpp
#include <iostream>
#include <fstream>

int main()
{
    std::ofstream file("output.txt");    // 创建输出文件流，自动关联到 output.txt

    if (!file)                            // 一定要检查打开是否成功
    {
        std::cerr << "无法打开文件！\n";
        return 1;
    }

    file << "Hello, File!\n";
    file << "圆周率 ≈ " << 3.14159 << "\n";

    // file 对象退出作用域时自动关闭，你也可以显式调 file.close()
    return 0;
}
```

创建 `ofstream` 时传文件名，内部会自动打开。写起来和 `cout` 一模一样——`<<` 运算符照搬。

默认打开是"覆盖模式"——文件若已存在，原有内容会被清掉。如果想**往后面追加**而不是覆盖，加上 `std::ios::app`：

```cpp
std::ofstream file("output.txt", std::ios::app);
file << "新追加的一行\n";
```

## 读取文件

```cpp
#include <iostream>
#include <fstream>
#include <string>

int main()
{
    std::ifstream file("input.txt");

    if (!file)
    {
        std::cerr << "无法打开文件！\n";
        return 1;
    }

    std::string line;
    int lineNum = 1;

    while (std::getline(file, line))
    {
        std::cout << lineNum << ": " << line << "\n";
        ++lineNum;
    }

    return 0;
}
```

`getline` 每次读到换行为止（换行符本身不被存入 `line`）。`while (getline(file, line))` 读一行、检查是否成功——读到文件末尾或出错了，自动退出循环。

除了逐行读，还有逐词读和逐字符读：

```cpp
// 逐词读（空白字符分隔）
std::string word;
while (file >> word)
    std::cout << "[" << word << "] ";

// 逐字符读（包括空格和换行）
char ch;
while (file.get(ch))
    std::cout << ch;
```

## 打开模式

打开文件时可附加多个模式标志，用 `|` 组合：

| 标志 | 含义 |
| ---- | ---- |
| `std::ios::in` | 读模式（`ifstream` 默认） |
| `std::ios::out` | 写模式（`ofstream` 默认） |
| `std::ios::app` | 在末尾追加 |
| `std::ios::ate` | 打开后立即定位到文件末尾 |
| `std::ios::binary` | 二进制模式 |
| `std::ios::trunc` | 如文件已存在，清空 |

```cpp
std::ofstream file("data.bin", std::ios::out | std::ios::binary);  // 二进制写入
```

为什么需要二进制模式？因为在 Windows 上，文本模式会自动把 `\n` 转换成 `\r\n` 写入、再在读取时转回来。二进制模式不做任何转换——原样来原样去。处理图片、音频、自定义序列化格式时，必须指定 `binary`，否则数据就被悄悄破坏掉了。

## 同时读写：fstream

```cpp
#include <iostream>
#include <fstream>
#include <string>

int main()
{
    std::fstream file("data.txt", std::ios::in | std::ios::out | std::ios::trunc);

    file << "Hello!\n";
    file << "第二行\n";

    file.seekg(0);       // 把读指针移回文件开头
    std::string line;
    while (std::getline(file, line))
        std::cout << "读到: " << line << "\n";

    return 0;
}
```

读和写各有一个"文件指针"。`seekg`（seek get）移读指针，`seekp`（seek put）移写指针。`seekg(0)` 回到文件头重新开始读。

## 实际工具：合并两个文本文件

学到这里，顺手写一个真正有用的工具——把两个文本文件的行交织合并写入新文件：

```cpp
#include <iostream>
#include <fstream>
#include <string>

int main(int argc, char* argv[])
{
    if (argc < 4)
    {
        std::cerr << "用法: " << argv[0] << " <文件1> <文件2> <输出文件>\n";
        return 1;
    }

    std::ifstream f1(argv[1]);
    std::ifstream f2(argv[2]);
    std::ofstream out(argv[3]);

    if (!f1 || !f2 || !out)
    {
        std::cerr << "错误：无法打开某个文件\n";
        return 1;
    }

    std::string line1, line2;
    while (std::getline(f1, line1) && std::getline(f2, line2))
    {
        out << line1 << "\n" << line2 << "\n";
    }

    std::cout << "合并完成，结果在 " << argv[3] << "\n";
    return 0;
}
```

文件 IO 的 "打开 → 检查 → 读取/写入 → 自动关闭" 模式，在 C++ 的文件流里已经是一种肌肉记忆式的标准写法。掌握了它，日志系统、配置文件解析器的底层部分你就都接得住了。
