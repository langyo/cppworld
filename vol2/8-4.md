# 摄取与排放数据 · 文件输入输出

到目前为止，我们所有的 `cin` 和 `cout` 都只对着屏幕打交道——程序一关，所以的数据不翼而飞。真正的程序需要把数据存到文件里，下次打开还能接着用。

C++ 标准库通过 `<fstream>` 头文件提供了文件读写的能力。它中的三个核心类——`ifstream`（输入文件流）、`ofstream`（输出文件流）和 `fstream`（输入输出文件流）——用起来和 `cin`/`cout` 是同一套语法，只是把方向从键盘/屏幕换成了文件。

------

**写入文件**

```cpp
#include <fstream>
#include <iostream>
using namespace std;

int main()
{
    ofstream file("output.txt");    // 创建一个输出文件流，关联到 output.txt

    if (!file)     // 检查是否打开成功
    {
        cerr << "无法打开文件！" << endl;
        return 1;
    }

    file << "Hello, File!" << endl;
    file << "第二行文字" << endl;
    file << "圆周率 ≈ " << 3.14159 << endl;

    // 文件会在 file 对象析构时自动关闭，也可以手动：
    // file.close();

    return 0;
}
```

创建 `ofstream` 对象时就指定了文件名——构造函数会在内部尝试打开这个文件。写入的语法和 `cout` 几乎一模一样，流运算符 `<<` 直接照搬。

当你运行完程序后，打开 `output.txt`，就能看到刚写进去的内容。

`ofstream` 默认以"覆盖"模式打开：如果文件已存在，原有内容会被清空。如果想**追加**而不是覆盖，加 `ios::app` 模式：

```cpp
ofstream file("output.txt", ios::app);   // append 模式，新的内容贴在末尾
```

------

**读取文件**

```cpp
#include <fstream>
#include <iostream>
#include <string>
using namespace std;

int main()
{
    ifstream file("input.txt");

    if (!file)
    {
        cerr << "无法打开文件！" << endl;
        return 1;
    }

    // 方式一：逐词读取（空白字符分隔）
    string word;
    while (file >> word)
        cout << "[" << word << "] ";

    // 方式二：逐行读取
    string line;
    while (getline(file, line))
        cout << line << endl;

    // 方式三：逐个字符读取
    char ch;
    while (file.get(ch))
        cout << ch;

    return 0;
}
```

- `file >> word` 读取到空白字符（空格、制表符、换行）为止——如果你想读入以空格分隔的 token，用这个。
- `getline(file, line)` 一次读到换行符为止，换行符自身不存进 line 里。
- `file.get(ch)` 一次读一个字符，包括空格和换行都不跳过。

注意 `while (file >> word)` 这种写法——流对象在布尔上下文中会自动判定"上一次读取是否成功"。当读到文件末尾或中途出错时，它变成 false，循环自动退出。这是文件操作中最常见的循环模式。

------

**多种打开模式**

打开文件时可以指定模式标志，通过 `|` 组合：

| 标志 | 含义 |
| ---- | ---- |
| `ios::in` | 读模式（ifstream 默认） |
| `ios::out` | 写模式（ofstream 默认） |
| `ios::app` | 追加模式——每次写入都在末尾 |
| `ios::ate` | 打开后立刻把文件指针定位到末尾 |
| `ios::binary` | 二进制模式——不进行文本转换 |
| `ios::trunc` | 如果文件已存在，清空 |


```cpp
ofstream file("data.bin", ios::out | ios::binary);   // 以二进制模式写入
ifstream file("log.txt", ios::in | ios::ate);        // 打开并定位到末尾
```

默认情况下（不指定 binary标志），文件以**文本模式**打开——在 Windows 上，`\n` 在写入时会转换成 `\r\n`，读取时再转回来。**二进制模式**不做任何转换，原样读写。处理图片、音频、序列化数据等非文本内容时，必须指定 `ios::binary`，否则你的数据会被悄无声息地损坏。

------

**fstream：同时读写**

`fstream` 类可以同时对一个文件进行读写：

```cpp
fstream file("data.txt", ios::in | ios::out);

if (!file)
{
    // 如果文件还不存在，新建一个
    file.open("data.txt", ios::out);
    file.close();
    file.open("data.txt", ios::in | ios::out);
}

file << "Hello!" << endl;

// 回到文件开头读取
file.seekg(0);                    // seek get —— 移动读取位置
string line;
getline(file, line);

cout << "读到: " << line << endl;
```

读和写各有独立的"文件指针"——用 `seekg()`（seek get）移动读指针，用 `seekp()`（seek put）移动写指针。`tellg()` / `tellp()` 用来查询当前的读/写位置。

`seekg(0, ios::end)` 表示相对于文件末尾偏移 0（即跳到文件尾），`seekg(0, ios::beg)` 表示相对于文件开头偏移 0（回到开头）。第二个参数省略时默认就是相对于开头。实际代码中最常用的是 `seekg(0)`（回文件头开始重新读）和 `seekp(0, ios::end)`（追加写）。

------

**二进制文件的读写**

文本文件的 `<<` 和 `>>` 不适合二进制数据（比如一个整数写进文本文件是"42"（两字节字符），但用二进制方式它是 4 个字节的原始内存表示）。二进制读写要用 `write()` 和 `read()`：

```cpp
// 写入二进制数据
ofstream out("binary.dat", ios::binary);
int x = 12345;
out.write(reinterpret_cast<const char*>(&x), sizeof(x));
out.close();

// 读取二进制数据
ifstream in("binary.dat", ios::binary);
int y;
in.read(reinterpret_cast<char*>(&y), sizeof(y));
cout << y << endl;    // 12345
```

`write` 和 `read` 接受的第一个参数是 `char*` 的指针，然后加上要读写多少个字节。因为大多数数据不是 char 类型，所以要用 `reinterpret_cast<const char*>(&x)` 或者 C 风格的 `(const char*)&x` 来转一次。看着有点吓人，但原理很简单：把内存里的原始字节"照抄"到文件里。

------

**文件操作中的常见错误**

1. 文件打不开 → 最常见的是路径不对、文件被占用、或者没权限。永远检查 `if (!file)`。
2. 读完忘了检查 eof → 在 `while (!file.eof())` 里循环是个经典坑，因为 eof 只有在"尝试读过末尾之后"才会被触发，可能导致最后一条数据被处理两次。应该直接用 `while (getline(file, line))` 或 `while (file >> x)` 这种惯用法——它们在读取失败时立即终止循环。
3. 忘了关闭流 → 流对象析构时会自动关闭。但如果程序还没跑完就需要打开另一个文件覆盖这个文件，必须显式 `close()`。
4. 文本模式和二进制模式混用 → 想读写原始字节数据就不要用 << 和 >>，用 `write()` 和 `read()` 外加 `ios::binary`。
