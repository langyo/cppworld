# 换一种变脸的方式 · 类型转换运算符定义

有时候你造了一个类型，却在需要另一个类型的地方也想直接用——不希望每次都敲 `obj.toDouble()`，而是希望直接写 `obj` 让编译器自己转化。

类型转换运算符就是干这个的。它让你自定义"我这个类在遇上需要某种类型的时候，应该怎么把我转过去"。

------

## 基本语法

```cpp
class Distance
{
private:
    double meters;
public:
    Distance(double m) : meters(m) {}

    operator double() const          // 类型转换运算符：Distance → double
    {
        return meters;
    }
};

int main()
{
    Distance d(1.5);

    double len = d;                  // 自动将 Distance 转成 double
    std::cout << len << "\n";        // 1.5
    std::cout << d + 0.5 << "\n";    // 2.0 —— d 先转 double 然后 +0.5
}
```

`Distance` 在遇到需要 `double` 的地方，自动把自己转化——调用者不需要知道转化是怎么发生的。

------

## 隐式转换的陷阱

转换运算符不加约束就可能在你完全没意想的场景中静默执行，把代码行为带偏：

```cpp
class Stream
{
    bool valid;
public:
    operator bool() const { return valid; }   // 危险的隐式！
};

Stream s;
int x = s + 5;    // 竟然编译通过了——s 被隐式转成 bool (0/1)，再 + 5
```

上面不是你想要的行为。如果想保留在 `if` 中可用的好处，又防止它偷偷参加算术运算，标上 `explicit`：

```cpp
class SafeStream
{
    bool valid;
public:
    explicit operator bool() const { return valid; }
};

SafeStream ss;

if (ss) { /* ... */ }                 //  OK——if/while/for/! 等语境允许 explicit 的隐式调用
// int x = ss + 5;                    //  编译错误——explicit 阻止
bool b = static_cast<bool>(ss);       //  OK——显式转换
```

标准库中的 `std::ifstream`、`std::unique_ptr` 的 `operator bool` 都是 `explicit` 的——既能让你在条件判断中直接使用，又杜绝了莫名其妙跑到算术表达式里的风险。

------

## 更多场景：通过转换运算符实现流畅的 API

```cpp
class Percentage
{
private:
    double value;    // 内部存小数形式 (0.0 ~ 1.0)
public:
    explicit Percentage(double v) : value(v / 100.0) {}

    // 用于输出——输出成 "85%"
    explicit operator std::string() const
    {
        return std::to_string(static_cast<int>(value * 100)) + "%";
    }

    // 用于计算——转成乘数 (0.85)
    explicit operator double() const
    {
        return value;
    }
};

int main()
{
    Percentage p(85.0);                    // 85%
    std::cout << static_cast<std::string>(p) << "\n";    // "85%"

    double discount = 200 * static_cast<double>(p);      // p 转成 0.85 → 170
    std::cout << "折后价: " << discount << "\n";
}
```

同一个对象，在不同场景下可以有完全不同的转化方式。转 `double` 拿乘数，转 `string` 拿去显示——各自明确的身份，各走各的通路。

------

## 避免一张类型上开太多转换门

如果一个类同时提供 `operator int()`、`operator double()` 甚至 `operator string()`，外加一个接收 `int` 的构造函数——那当你传这个类给任何接受数字或字符串的函数时，编译器就会进入选择困难（"我该走转成 int 进去还是转成 double 进去？还是把参数转成我的类型走你的重载？"）。

原则：**每个类型最多一条隐式转换通路**。如果必须提供多条，那就全部标上 `explicit`，让调用者自己选。
