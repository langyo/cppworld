# 小别致 · 枚举与联合类

枚举和联合体都是 C 就有的老家伙。但到了 C++11，它们各自迎来了一次像样的翻新——老的工具加上类型安全的新包装，可用性一下子改善了不少。

------

**传统枚举的痛**

```cpp
enum Color { Red, Green, Blue };
enum Mood  { Happy, Sad, Blue };   // ❌ 重名冲突——Blue 被用了两次！
```

C 风格的枚举会将值名字泄露到**外层作用域**——两个叫 `Blue` 的枚举值撞在一起，编译不过。而且这种枚举的值可以隐式转换成整数，却无法将整数隐式转回枚举，类型安全形同虚设。

------

**枚举类（enum class）：C++11 的类型安全方案**

```cpp
enum class Color { Red, Green, Blue };
enum class Mood  { Happy, Sad, Blue };   // ✅ 通过枚举名隔离，不冲突

Color c = Color::Red;
Mood  m = Mood::Blue;
// int x = c;           // ❌ 不能自动转为 int——必须显式转
int x = static_cast<int>(c);    // ✅ 显式转
```

`enum class` 三件事全部解决了——域隔离、名称不污染外层、禁止隐式转 int。默认所有新代码都用 `enum class`，无例外。

------

**指定底层类型**

枚举也可以指定底层整型类型——比如只想用一个字节存储：

```cpp
enum class Permission : uint8_t     // 底层类型为 uint8_t（一个字节）
{
    Read    = 1 << 0,     // 0b0001
    Write   = 1 << 1,     // 0b0010
    Execute = 1 << 2      // 0b0100
};

// 组合权限
uint8_t canReadWrite = static_cast<uint8_t>(Permission::Read)
                     | static_cast<uint8_t>(Permission::Write);
```

C++11 起枚举类均可指定底层类型，传统的 C 枚举也可（写 `enum Color : int { ... }`）。

------

**union：叠加型的"时空共享"体**

union 允许多个成员共享同一个内存位置——同一时间只存储其中一个成员的值。这就好像一张"热铺位床"——同一时刻只能睡一个人，但夜里睡的人和白班睡的人可以是不同的人：

```cpp
union Value
{
    int   intVal;
    double doubleVal;
    char*  strVal;
};
```

union 的大小等于它最大成员的大小（加上可能的对齐封装）。读写值的时候，由程序员自己记住"当前激活的是哪个成员"——读错成员拿到的是垃圾。

C++17 推出了**标记联合体** `std::variant`（在 `<variant>` 头文件中），它是类型安全的 union——它内部记录了当前激活的是哪个替代类型，用 `get` 或 `visit` 安全访问。对于新代码，一般直接用 `variant`，union 主要用于底层库或者语言边界上。

------

**联合体的进阶特性**

C++11 起 union 的成员已经可以是拥有构造/析构函数的类类型（但你需要手动管理它们的生命周期，调用 placement new 和显式析构）。C++17 为 union 提供了一点模板支持。日常代码中遇到的 union 大多是 C 语言的遗留——如果你读了旧代码、某硬件接口、或二进制协议的解析部分时见到大坨大坨的 union 和 `#pragma pack`，不必惊慌，原理就是这些。
