# 小别致 · 枚举与联合类

枚举和联合体都是 C 语言的遗产。C++11 把它们洗礼成了类型安全的现代工具。会用它们，游戏状态机、权限系统、网络协议解析这些硬活才写得干净。

------

## 枚举类：告别 C 风格枚举的乱象

老式 C 枚举有两个让人牙疼的问题：名称泄露到外层作用域、值能偷偷转成 int。

```cpp
// C 风格枚举
enum Color { Red, Green, Blue };
enum Mood  { Happy, Sad, Blue };    // 报错——Blue 被 Color 用过了！
```

C++11 的 `enum class` 三件事一步到位：

```cpp
enum class Color { Red, Green, Blue };
enum class Mood  { Happy, Sad, Blue };    // 各自的 Blue，互不干扰

Color c = Color::Red;          // 必须带限定名
Mood  m = Mood::Blue;
// int x = c;                  //  不允许隐式转 int
int x = static_cast<int>(c);   //  想转必须显式来
```

以后再也不要写旧式 `enum`。`enum class` 把作用域隔离和类型安全一并治了。

------

## 指定底层类型，精确控制大小

默认的枚举底层类型是 `int`。当你要把一堆标志位打包成字节或跟硬件寄存器严格匹配时，可以精确指定：

```cpp
enum class Permission : uint8_t     // 只占一个字节
{
    None     = 0,
    Read     = 1 << 0,    // 0b0001
    Write    = 1 << 1,    // 0b0010
    Execute  = 1 << 2     // 0b0100
};

// 枚举值做位运算——先用 static_cast 转回底层整数
uint8_t rw = static_cast<uint8_t>(Permission::Read)
           | static_cast<uint8_t>(Permission::Write);
```

如果想把位运算写得更舒服，可以给 `Permission` 重载 `|` 运算符——但大多数代码里直接用 `static_cast` 然后位运算，已经很清晰了。

------

## 用枚举类做状态机

枚举最实用的场景之一是有限状态机。比如一个简单的 TCP 连接状态：

```cpp
enum class ConnectionState : uint8_t
{
    Closed,
    Listening,
    SynSent,
    Established,
    FinWait,
    TimeWait
};

class TcpConnection
{
private:
    ConnectionState state = ConnectionState::Closed;

public:
    bool connect()
    {
        if (state != ConnectionState::Closed)
        {
            cerr << "当前不能发起连接\n";
            return false;
        }
        state = ConnectionState::SynSent;
        cout << "发送 SYN...\n";
        // ……收到 SYN-ACK 后……
        state = ConnectionState::Established;
        cout << "连接已建立\n";
        return true;
    }

    void close()
    {
        state = ConnectionState::Closed;
        cout << "连接已关闭\n";
    }

    ConnectionState getState() const { return state; }
};
```

相比用 `int` 加一堆 `#define` 或 `const int` 来表示状态，`enum class` 让编译器帮你兜着底——绝不会不小心把 `2`（一个无意义的整数）赋给 `state`，或把两个不同语境的状态值混用。

## union：多类型共享同一块空间

union 让多个成员共用同一块内存——同一时刻只有其中一个成员是"活着"的。它的大小等于最大成员的大小。

```cpp
union VariantValue
{
    int    intVal;
    double dblVal;
    char   strVal[16];
};

int main()
{
    VariantValue v;
    v.intVal = 42;

    cout << "int 值: " << v.intVal << "\n";

    // 现在覆盖它！
    v.dblVal = 3.14159;
    cout << "double 值: " << v.dblVal << "\n";

    // 危险——如果去读 v.intVal，拿到的是浮点位模式画出的整数，完全不是 42
}
```

C++17 提供了替代品 `std::variant` ——它是类型安全的 union，内部记录了"当前哪个类型在活"：

```cpp
#include <variant>
#include <iostream>
#include <string>

int main()
{
    variant<int, double, string> value;
    value = 42;
    value = 3.14159;
    value = string("Hello");

    // 安全取出——通过 get 或 visit
    if (auto* p = get_if<string>(&value))
        cout << "字符串值: " << *p << "\n";
    else if (auto* p = get_if<int>(&value))
        cout << "整数值: " << *p << "\n";
}
```

对于新代码，`std::variant` 是首选——它让你不用自己记"当前哪个成员是活的"。union 现在主要用于底层库、二进制协议解析、或对每一个字节都需要精确控制的嵌入式场合。

------

union 偶尔还有一个变种叫**匿名 union**——不提供类型名字，直接把成员暴露在结构体的命名空间中。这在某些需要"同一组 bit 以多种身份被访问"的底层代码中出现，但极少在应用层代码中使用。

枚举和联合体这两个老兵，经过 C++11 的现代化改造后不再是弱类型时代的遗留物。`enum class` 是任何需要表示"有限个选择"的地方的第一选择——状态机、命令码、配置项、位标志、协议标识符；`std::variant` 则是需要在运行时灵活切换类型的现代方案。
