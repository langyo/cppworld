# 意念造山河 · lambda 表达式

在前面第八章我们接触过几行像这样的东西：

```cpp
[](int x) { return x > 5; }
```

这玩意儿叫 **Lambda 表达式**，C++11 引入。它的作用很纯粹——让你在需要函数的地方**直接就地写一个函数**，而不用跑到外面去单独定义一个函数再传函数指针。

这在跟算法库搭伙时非常趁手：想筛选？想排序？想对每个元素干点什么？写一个 Lambda，几步搞定。

------

**最简单的 Lambda**

```cpp
auto add = [](int a, int b) { return a + b; };
cout << add(3, 5) << endl;   // 8
```

拆开来看：

| 部分 | 说明 |
| ---- | ---- |
| `[]` | 捕获列表——决定 Lambda 体内可以访问外部哪些变量 |
| `(int a, int b)` | 参数列表——和普通函数一样 |
| `{ return a + b; }` | 函数体——和普通函数一样 |

Lambda 本质上是编译器帮你自动生成的一小个匿名的类（称为闭包类型），这个类带有一个 `operator()` 成员函数——也就是说，一个 Lambda 表达式就是一个可调用对象。

------

**捕获列表：Lambda 体内如何访问外面的东西**

Lambda 可以"抓取"外部变量进去用。怎么抓，由 `[]` 里的捕获列表决定：

```cpp
int x = 10, y = 20;

// 按值捕获——抓取 x 的副本，函数体内改的是副本，不影响外面的 x
auto f1 = [x]() { return x + 1; };     // x 在内部是只读的

// 按引用捕获——抓取 x 的引用，函数体内能改外面的 x
auto f2 = [&x]() { x++; };             // 可以修改外面的 x

// 全部按值捕获（用 = 前缀）
auto f3 = [=]() { return x + y; };     // x 和 y 都以副本方式进入

// 全部按引用捕获（用 & 前缀）
auto f4 = [&]() { x = 0; y = 0; };     // x 和 y 都以引用方式进入

// 混合——y 按值，其余按引用
auto f5 = [&, y]() { x++; /* y 是只读副本 */ };
```

注意按值捕获的变量在 Lambda 体内默认是常量的，不能修改。如果你确实要在按值捕获的 Lambda 体内改"副本"，需要在参数列表后面加 `mutable`：

```cpp
int count = 0;
auto f = [count]() mutable { count++; return count; };
// f() 每次调用返回的值依次+1，但外部的 count 不变化
```

`mutable` 在算法回调中比较冷门，更像是一种允许你临时动一下"私有副本"的免死令牌。大多数时候不需要它。

------

**Lambda 与泛型算法：标准搭档**

Lambda 最常见的出场形式就是给泛型算法当"判断依据"或"操作函数"：

```cpp
vector<int> v = {1, 5, 3, 8, 2, 7, 4, 6};

// 排序——由大到小
sort(v.begin(), v.end(), [](int a, int b) { return a > b; });

// 查找第一个能整除 3 的元素
auto it = find_if(v.begin(), v.end(), [](int x) { return x % 3 == 0; });

// 每个元素都翻倍
for_each(v.begin(), v.end(), [](int& x) { x *= 2; });

// 删除偶数
v.erase(remove_if(v.begin(), v.end(), [](int x) { return x % 2 == 0; }),
        v.end());
```

在 Lambda 出现之前的 C++98，你必须为这些算法在远处写一个命名函数或函数对象类，定义位置和调用位置相隔太远，代码来回跳着读。Lambda 把逻辑代码放在离使用地点最近的地方，可读性好很多。

------

**泛型 Lambda（C++14）**

如果你不想指定参数类型，可以用 `auto`：

```cpp
auto add = [](auto a, auto b) { return a + b; };
cout << add(3, 5) << endl;      // int + int => 8
cout << add(1.5, 2.7) << endl;  // double + double => 4.2
cout << add(string("Hello, "), string("World!")) << endl; // string 拼接也支持
```

这个叫**泛型 Lambda**——参数写 `auto`，编译器会按调用时的实参类型自动为你生成对应的模板版本。本质上跟写了个函数模板差不多，只不过你没有显式定义那个模板本身。

泛型 Lambda 是 C++14 带来的特性，在 C++11 中写 Lambda 必须显式写出参数类型。

------

**返回值类型推断**

大多数情况下编译器能自动推断 Lambda 的返回值类型，你不需要写：

```cpp
auto f = [](int a, int b) { return a + b; };  // 编译器知道返回 int
```

但如果函数体中有多个 `return` 语句且返回类型不一样，或者想显式指定返回类型，可以用尾置返回类型：

```cpp
auto f = [](int a, int b) -> double { return a + b; };  // 显式返回 double
```

尾置返回类型用 `->` 指定，写在参数列表后面、函数体前面。跟函数模板的尾置返回类型写法一模一样。

------

**捕获 this**

在类的成员函数里面写 Lambda 时，如果你想用类的成员变量，要么用 `[=]` 或 `[&]` 隐式捕获 `this`，要么显式写 `[this]`：

```cpp
class Counter
{
    int value = 0;
public:
    auto getLambda()
    {
        return [this]() { value++; };    // 捕获了 this，就能用 value
    }
};
```

C++20 起还支持 `[*this]`——把整个对象拷贝一份进 Lambda 里（在短暂的短生命周期 Lambda 中用得少，主要是异步场景中延长对象寿命）。

------

**Lambda vs 函数指针 vs 函数对象**

| 方式 | 写法难度 | 性能 |
| ---- | -------- | ---- |
| Lambda 表达式 | 简洁 | 优（编译器通常会内联） |
| 函数指针 | 简单但类型约束多 | 一般（难以内联） |
| 函数对象（class 重载 operator()） | 繁琐 | 优 |

在现代 C++ 中，Lambda 已经成为编写短小可调用对象的首选方式。除非你需要复用同一段逻辑很多次（那把它写成个正规函数更合适），否则就大胆用 Lambda。
