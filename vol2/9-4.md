# 意念造山河 · lambda 表达式

第八章我们在介绍泛型算法时用过好几次这样的小东西：

```cpp
[](int x) { return x > 5; }
```

它叫 **Lambda 表达式**，C++11 引入。它能让你在需要传函数的地方**当场写一个小函数**，不用跑出去定义一个独立的命名函数——代码的逻辑和逻辑的调用者放在同一个视野里，读起来顺，改起来也直接。

------

## 从外部函数到 Lambda：一个对比

假设要找一个 vector 里第一个大于 5 的数。

**C++98 写法**——你得在外面单独定义一个函数或函数对象类：

```cpp
bool greaterThan5(int x) { return x > 5; }
// ……然后在很远的地方……
auto it = find_if(v.begin(), v.end(), greaterThan5);
```

**Lambda 写法**——就地写，就地用：

```cpp
auto it = find_if(v.begin(), v.end(),
                       [](int x) { return x > 5; });
```

不用跳转到文件其他地方去看 `greaterThan5` 写的是什么——它就在你正在读的那行代码的旁边。

Lambda 本质上是一个被编译器自动翻译成一个匿名类 + `operator()` 的东西。这个匿名类被称为闭包类型（closure type），`operator()` 的行为就是你写在大括号里的那段代码。所以 Lambda 其实也是函数对象的一种，只是你不用手写类了。

------

## Lambda 的结构

```cpp
auto add = [](int a, int b) { return a + b; };
cout << add(3, 5) << "\n";   // 8
```

拆开看四部分：

| 部分 | 含义 |
| ---- | ---- |
| `[]` | 捕获列表——告诉 Lambda 体内可以访问外部哪些变量、怎么访问 |
| `(int a, int b)` | 参数列表——和普通函数一样 |
| `{ return a + b; }` | 函数体——和普通函数一样 |
| 返回值 | 编译器自动推断，此处是 `int` |

------

## 捕获：把外面的东西带进去

Lambda 体内可以用外面的变量——但必须通过捕获列表声明"怎么拿到"它们：

```cpp
int threshold = 10;

// 按值捕获 threshold —— 拿它的副本进 Lambda 体，外部改了不影响副本
auto f1 = [threshold](int x) { return x > threshold; };

// 按引用捕获 threshold —— 共享同一个变量，Lambda 体内改它就等于改外面
auto f2 = [&threshold](int x) { return x > threshold; };

// 全按值捕获（=）——用到的所有外部变量都以副本方式拉进来
auto f3 = [=](int x) { return x > threshold; };

// 全按引用捕获（&）——用到的所有外部变量都以引用方式拉进来
auto f4 = [&](int x) { threshold = x; };   // Lambda 体内可以改外面的 threshold
```

按值捕获进来的变量在 Lambda 体内默认是 `const` 的——不能修改。如果你确实想在 Lambda 体内改动**按值捕获的副本**（改动只影响副本，外面不受影响），在参数列表后面加 `mutable`：

```cpp
int count = 0;
auto counter = [count]() mutable { return ++count; };
cout << counter() << "\n";   // 1
cout << counter() << "\n";   // 2
cout << count << "\n";       // 0 —— 外面的 count 纹丝未动
```

## Lambda 与标准算法：天然战友

```cpp
vector<int> v = {1, 5, 3, 8, 2, 7, 4, 6};

// 降序排——Lambda 里定义比较规则
sort(v.begin(), v.end(), [](int a, int b) { return a > b; });

// 找第一个能整除 3 的数
auto it = find_if(v.begin(), v.end(), [](int x) { return x % 3 == 0; });
if (it != v.end())
    cout << "找到了：" << *it << "\n";

// 把每个元素翻倍
for_each(v.begin(), v.end(), [](int& x) { x *= 2; });

// 筛掉所有小于 5 的元素
v.erase(remove_if(v.begin(), v.end(), [](int x) { return x < 5; }),
        v.end());
```

`remove_if` 把"不应删除的"元素挤到前面，返回一个新末尾，`erase` 从新末尾开始真正删掉尾部的废弃元素——这是"remove-erase惯用法"，搭配 Lambda 最常见。

------

## 泛型 Lambda（C++14）

C++14 起参数可以用 `auto`——写一次 Lambda，应对多种被调类型：

```cpp
auto add = [](auto a, auto b) { return a + b; };

cout << add(3, 5) << "\n";                       // int + int => 8
cout << add(1.5, 2.7) << "\n";                   // double + double => 4.2
cout << add(string("Hello, "), "World!")    // string 拼接
          << "\n";
```

泛型 Lambda 的本质是一个模板——`[](auto a, auto b) { return a + b; }` 编译器会为其生成模板化的 `operator()`，对每一种调用类型生成对应的实例。

------

## 捕获 this

在类的成员函数里写 Lambda，如果需要用到这个类的成员变量，必须显式或隐式地捕获 `this`：

```cpp
class Counter
{
    int value = 0;
public:
    auto getIncrementer()
    {
        return [this]() { return ++value; };   // 拿 this 进来，就能用 value
    }
};

int main()
{
    Counter c;
    auto inc = c.getIncrementer();
    cout << inc() << "\n";   // 1
    cout << inc() << "\n";   // 2
}
```

按引用捕获 `[&]` 和按值捕获 `[=]` 也会隐式捕获 `this`（各自按照引用/值的方式）。C++20 引入 `[*this]`——把整个对象复制一份带进 Lambda，让异步代码实现对象生命周期独立。

------

## 返回值类型推断

绝大多数情况不需手写返回类型，编译器能推出来。但如果你需要显式指定（比如两个 return 语句返回不同类型想要统一），用尾置返回类型：

```cpp
auto f = [](int a, int b) -> double { return a + b; };
//                            ^^^^^^^^  强制返回 double
```

不加 `-> double` 的话，编译器会推成 `int`，因为 `a + b` 是 `int`。尾置返回类型让你能强制把结果类型往宽了指。

------

Lambda 是 C++11 之后日常写代码最常用的工具之一。只要你的代码里有"这里需要一个短小的函数"的地方——排序规则、查找条件、遍历操作、回调寄存器——它会大大降低代码的隔阂感。把相关的逻辑紧凑地放在一起，是 Lambda 送给你最实际的便利。
