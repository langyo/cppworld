# 划分领地 · 命名空间

你可能从一开始就注意到 `std::cout` 里那个奇怪的 `std::` 前缀。它不是什么语法糖——它是一个**命名空间**，用来把"cout 这个名"跟其它人写的 `cout` 隔离开来。

说白了，命名空间就是一个带有名字的作用域。一个项目里可能有几十个库——保不齐两个库各定义了同名的类或函数。如果没有命名空间，名字就会撞车，编译器也分不清你想用的是哪一个。

------

**定义命名空间**

```cpp
namespace MyEngine
{
    class Renderer { /* ... */ };
    class Window   { /* ... */ };

    void initialize()
    {
        cout << "引擎初始化" << endl;
    }
}
```

然后使用：

```cpp
MyEngine::Renderer r;
MyEngine::initialize();
```

`::` 是作用域运算符。`MyEngine::Renderer` 就是在说"去 MyEngine 这个领地找 Renderer"。

------

**`using` 声明**

如果不想到处写 `MyEngine::`，可以用 `using` 把一部分名字引入当前作用域：

```cpp
using MyEngine::Renderer;
using MyEngine::initialize;

Renderer r;
initialize();
```

与之前看到的 `using namespace std;` 不同——`using namespace` 是把整个命名空间的所有名字一次性全部引入；`using` 单个名字则精细得多，只露出你需要的几个名字，风险小得多。

------

**命名空间嵌套**

命名空间可以嵌套：

```cpp
namespace Engine
{
    namespace Graphics
    {
        class Shader { /* ... */ };
    }
}

Engine::Graphics::Shader s;
```

C++17 起嵌套命名空间可以更简洁地写在一行——`namespace Engine::Graphics { ... }`。

------

**匿名命名空间**

如果你声明了一个匿名命名空间：

```cpp
namespace
{
    int internalCounter = 0;
    void privateHelper() { /* ... */ }
}
```

那其中的东西对**这个源文件**可见，对其它源文件完全隐藏。它相当于给每个成员加了个隐式的 `static` 限定符，但比 `static` 更通用（C++ 逐渐淘汰文件级的 `static` 声明，推荐用匿名命名空间代替）。

**匿名命名空间是现代 C++ 中实现文件级内部链接（internal linkage）的标准方式，`static` 全局变量和函数应逐步替换为匿名命名空间。**

------

**命名空间别名**

如果某命名空间长得令人发指：

```cpp
namespace chrono = chrono;    // 给 chrono 起个短号
chrono::seconds(5);
```

这在处理深度嵌套的库命名空间时特别管用——比如 `namespace fs = std::filesystem;`。

------

**内联命名空间**

```cpp
namespace MyLib
{
    inline namespace v2    // 默认使用最新版本
    {
        void feature() { cout << "新版功能" << endl; }
    }
    namespace v1
    {
        void feature() { cout << "旧版功能" << endl; }
    }
}
```

调用时，写 `MyLib::feature()` 实际落到 `v2`（因为 `v2` 是 inline）。如果想用旧版，显式指定 `MyLib::v1::feature()`。

内联命名空间主要用于库的版本管理——保持二进制兼容的同时允许用户在不同版本的 API 间显式选择。标准库内部也用它来做 ABI 版本控制。

------

命名空间布局的是一个项目的逻辑格局，大项目靠它避免命名污染。总体原则：**自己写的代码放进自己的命名空间，用别人的东西时尽量用 `using` 单名而不是 `using namespace`。** 这习惯一养成，省你一生的调试时间。
