# 一一对应 · 关联容器

顺序容器是按位置存数据的——第 0 号、第 1 号、第 2 号……访问的时候你得知道元素在哪个位置。但有些时候，我们想按**键**来查数据，而不是按位置。

比如：用学号查学生信息，用单词查它的释义，用日期查当日的天气。位置不告诉你答案，键才告诉你答案。

这就要求一种新的容器——**关联容器**。它内部自动按照键来组织数据，让你能以对数时间查到元素，而不是一条条地遍历。

------

**两大核心：map 和 set**

| 容器 | 解释 | 例子 |
| ---- | ---- | ---- |
| `map` | 键值对：每个键对应一个值 | 学号 → 学生；单词 → 释义 |
| `set` | 只有键没有值 | 黑名单；已访问过的 URL 集合 |

`map` 像一本字典——你查一个单词，得到它的解释。`set` 像一个集合——你问"这个单词在我们的单词表里吗？"，答案是"在"或"不在"。

这是最常用的两个关联容器。我们先从 `map` 开始。

------

**map：键值对**

```cpp
#include <iostream>
#include <map>
#include <string>
using namespace std;

int main()
{
    map<string, int> phoneBook;        // 姓名 → 电话号码

    // 添入数据
    phoneBook["Alice"] = 12345678;
    phoneBook["Bob"]   = 87654321;
    phoneBook["Carol"] = 55667788;

    // 查询
    string name = "Bob";
    cout << name << " 的电话是 " << phoneBook[name] << endl;

    // 遍历
    for (const auto& entry : phoneBook)
        cout << entry.first << ": " << entry.second << endl;
    // 输出顺序是 Alice → Bob → Carol（按 key 排序过的！）

    return 0;
}
```

关键点：
1. `map<string, int>` 表示"键是 string，值是 int"。
2. `phoneBook["Alice"] = 12345678` 会往 map 里塞一对键值——如果键 `"Alice"` 不存在，则创建；如果存在，则更新它的值。
3. 遍历输出结果是按 key 的字典序排好的，不是按插入顺序。
4. 每个元素是一个 `pair`，`first` 是键，`second` 是值。

> 用 `[]` 访问一个不存在的键会发生什么？它会悄悄把这对键值**创建出来**（值是用默认初始化，对 int 来说就是 0）。如果你只是想知道某个键在不在，或者不能容忍这种静默创建，那就用 `find()`（马上会讲到）。

------

**set：去重集合**

`set` 只存键，不存值。它不仅自带去重，插入重复的元素会被无视：

```cpp
#include <set>
using namespace std;

set<string> wordSet;

wordSet.insert("apple");
wordSet.insert("banana");
wordSet.insert("apple");      // 插入过的不再插入
wordSet.insert("cherry");

cout << wordSet.size() << endl;   // 3，不是 4

for (const auto& word : wordSet)
    cout << word << " ";          // apple banana cherry（已排序）
```

和 `map` 一样，`set` 中的元素也自动排好了序。

------

**查找：find()**

`[]` 在找不到键的时候会插入一个空值，这不是我们期望的纯查询行为。纯查询用 `find()`：

```cpp
map<string, int> phoneBook;
phoneBook["Alice"] = 12345678;

auto it = phoneBook.find("Alice");
if (it != phoneBook.end())
    cout << "找到了: " << it->second << endl;
else
    cout << "未找到" << endl;
```

`find` 返回一个迭代器——如果找到了，迭代器指向对应的键值对；如果没找到，返回 `end()`。

所以 `if (map.find(key) != map.end())` 是查键在不在的标准写法。

除了 find，还有一个 `count()`，对于 `map`/`set`（没重复键的容器）来说，它返回 0 或 1（因为键唯一）。但 `find()` 比 `count()` 更能完整满足"检测存在性+获取值"的需求，所以实际代码中 `find()` 更常用。

------

**插入与删除**

```cpp
map<string, int> dict;

// 插入的三种姿势
dict.insert({"hello", 1});                       // 初始化列表
dict.insert(make_pair("world", 2));               // make_pair
dict.insert(pair<string, int>("goodbye", 3));     // 显式构造

// insert 返回一个 pair：first 是指向元素的迭代器，second 是 bool（插入成功为 true）
auto result = dict.insert({"hello", 100});
if (!result.second)
    cout << "hello 已存在，值未更新" << endl;

// 删除
dict.erase("world");      // 按 key 删
dict.erase(dict.begin()); // 按迭代器删

// 如果键不存在，erase 不会报错（因为没有异常行为需要抛），直接无事发生
```

`insert` 的返回值很多时候被人忽略，但它其实是很好的"防重复"机制——如果你想保证键唯一或者在重复时做一些处理，检查 `result.second` 就行。

------

**multimap 与 multiset：允许重复键**

上面两个容器要求键是唯一的，有时我们需要允许相同的键出现多次：

```cpp
#include <map>
multimap<string, string> authors;     // 一个作者可以有多个作品
authors.insert({"鲁迅", "呐喊"});
authors.insert({"鲁迅", "彷徨"});
authors.insert({"老舍", "骆驼祥子"});

// 查找所有某个键的元素
auto range = authors.equal_range("鲁迅");   // 返回一对迭代器
for (auto it = range.first; it != range.second; ++it)
    cout << "鲁迅的作品: " << it->second << endl;
```

`multiset` 同理——允许重复元素的 `set`。

`equal_range()` 返回一个 `pair<iterator, iterator>`，表示所有匹配键范围的起止。注意它对 `multimap`/`multiset` 很有用，但普通 `map` 也可以用（只是范围长度永远 ≤ 1）。

------

**底层原理：红黑树 vs 哈希**

`map` 和 `set` 底层通常使用**红黑树**（平衡二叉查找树），因此：
- 键必须支持 `<` 运算（或者提供一个自定义比较函数）。
- 查找 / 插入 / 删除的时间复杂度是 O(log n)。
- 元素按顺序排列。

C++11 引入了**无序**版本：`unordered_map`、`unordered_set`、`unordered_multimap`、`unordered_multiset`。它们基于**哈希表**，特点是：

- 平均查找 / 插入 / 删除是 O(1)。
- 元素没有顺序。
- 键必须支持哈希运算。

也就是说——如果你只关心"查得快"而不关心"按顺序排"，`unordered_map` 往往比 `map` 好（尤其是大量数据时）。如果你希望遍历出来的结果是有序的（比如按字母排的字典），那就用 `map`。

```cpp
#include <unordered_map>
unordered_map<string, int> dict;   // 用哈希表，速度可能更快但输出无顺序
```

对初学者来说，先用熟 `map` 即可——在数据量不大时 O(log n) 和 O(1) 你根本感受不到差距。
