# 与硬件背靠背 · C++、C语言与汇编的协同

C++ 的独特之处在于它能顺畅调用 C 的函数库，也能通过中间层让 C 调用 C++ 封装的代码，甚至可以在程序里插一段汇编指令。这种跨语言的互通能力在处理系统 API、老旧 C 库和性能敏感代码时极为关键。

------

## 从 C++ 调 C

C 编译出的函数名不会被"修饰"（name mangling），而 C++ 编译器为了支持重载，内部的函数符号名会带有参数类型的编码信息。要让 C++ 能够找到一个 C 编译产出的函数，必须用 `extern "C"` 告诉 C++ 编译器："这几个人按 C 的链接规矩来"。

```cpp
extern "C"
{
    #include "some_c_library.h"
}

// 或在单个函数上声明
extern "C" int legacyFunction(int x);
```

有了 `extern "C"`，链接器就能在 C 生成的目标文件中找到函数符号。`extern "C"` 也无法作用于 C++ 特有的重载——被它包裹的函数名是按 C 规则唯一名称存道的，因此包裹区域里不能有同名不同参数的函数重载。

------

## 从 C 调 C++

反过来——在 C++ 这端暴露几个 `extern "C"` 包装的函数：

```cpp
// CppLib.cpp
extern "C"
{
    int compute(int a, int b)
    {
        return a * b + a;
    }
}
```

C 那边只需声明：

```c
// main.c
extern int compute(int a, int b);

int main()
{
    int result = compute(3, 5);    // 调用 C++ 编译产出的函数
    return 0;
}
```

编译时把 C 文件和 C++ 文件的目标文件一起链接就行。

C 没法直接冲着 C++ 的类对象去调用成员函数——只能绕道经过一层**不透明的 handle** 封装。惯用做法是：C++ 侧定义类完整实现，对 C 只暴露 `void*` 手柄和几个 C 函数：

```cpp
// CppWrapper.h（供 C 包含）
#ifdef __cplusplus
extern "C" {
#endif

typedef void* EngineHandle;

EngineHandle engine_create();
void         engine_destroy(EngineHandle h);
void         engine_render(EngineHandle h);

#ifdef __cplusplus
}
#endif
```

```cpp
// CppWrapper.cpp（C++ 实现）
#include "Engine.hpp"

EngineHandle engine_create()
{
    return new Engine();
}

void engine_destroy(EngineHandle h)
{
    delete static_cast<Engine*>(h);
}

void engine_render(EngineHandle h)
{
    static_cast<Engine*>(h)->render();
}
```

C 侧就拿着 `EngineHandle`（一个不认识的 `void*`）到处传，内部在 C++ 侧用 `static_cast` 恢复成原来的 `Engine*`。SDL、OpenAL 等许多跨语言的 C API 都是靠这种模式包装的。

## 内联汇编

C++ 编译器允许你直接在代码中嵌入汇编指令。当你需要用到 C++ 语法层面无法表达的硬件特性（如 CPUID、RDTSC、或某些 SIMD 指令的特化路径）时，这是最后的后手牌。

```cpp
#ifdef _MSC_VER
    __asm { mov eax, 1 }
#else
    __asm__("movl $1, %%eax" : : : "%eax");
#endif
```

汇编在不同编译器、不同 CPU 架构上完全不通——x86 的 MSVC 内联汇编代码不可能在 ARM 的 Clang 上编译。所以只应该在没有可移植的 C++ 替代方案时才拿出它。

------

## 调用约定

当 C++ 和 C 函数互相调用时，双方对参数如何传递、栈由谁清理必须意见一致——这便是**调用约定**。`extern "C"` 默认使用平台默认的 C 调用约定（通常是 `__cdecl`），不需要额外声明。如果你调用操作系统或某个 Dll 库中明确标注了调约的函数，则必须匹配：

```cpp
extern "C" __stdcall int WinApiFunction(int x);
```

Windows 上常见的调约包括 `__cdecl`（调用方清栈）、`__stdcall`（被调用方清栈）、`__fastcall`（寄存器传参优化）。调约不配对的最典型报就是函数返回时栈失衡导致程序直接崩溃。
