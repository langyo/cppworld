# 同时进行的任务 · 并行与并发库

单核频率不再像以前那样年年翻倍了，但 CPU 里的核心数量倒是从两个变成四个、八个、几十个。要让程序跑得更快，得像厨房炒菜一样学会"多用几口锅同时炒"，而不是只在一个锅上猛火干烧。

C++11 把多线程从各平台的私货库里提拔进了标准库——`<thread>`、`<mutex>`、`<condition_variable>`、`<future>`、`<atomic>`——让你写一次多线程代码，到处都能编译。

------

## 第一根线程

创建一个线程，把你要执行的任务（函数、Lambda 等）传进去就行：

```cpp
#include <iostream>
#include <thread>

void greet(const std::string& name)
{
    std::cout << "你好，" << name << "，来自子线程！\n";
}

int main()
{
    std::thread t(greet, "Steve");   // 创建线程，立刻开始执行 greet("Steve")
    t.join();                         // 主线程停下来等 t 跑完
    std::cout << "主线程继续\n";
    return 0;
}
```

`std::thread` 构造时传入可调用对象和参数，线程即启动——不需要 `start()` 之类的调用。`join()` 让当前线程停下来等子线程收工。如果不 `join` 也不 `detach`，`thread` 对象在析构时会调用 `std::terminate` 一把崩掉整个程序。

## 多线程一起跑

起两个线程，各自做不同的工作：

```cpp
#include <iostream>
#include <thread>
#include <vector>
#include <chrono>

void count(int id, int n)
{
    for (int i = 1; i <= n; ++i)
    {
        std::cout << "[线程 " << id << "] 第 " << i << " 步\n";
        std::this_thread::sleep_for(std::chrono::milliseconds(100));
    }
}

int main()
{
    std::thread t1(count, 1, 5);
    std::thread t2(count, 2, 3);

    t1.join();
    t2.join();

    std::cout << "两个线程都跑完了\n";
    return 0;
}
```

运行后你会看到两个线程的输出交错出现——这就是并发。`std::this_thread::sleep_for` 让当前线程暂停一小段时间，模拟在做工作量。

## 数据竞争与互斥锁

多线程最经典的问题：两个线程同时改同一变量。不加保护，结果不可预测：

```cpp
int counter = 0;     // 共享变量，无保护

void increment(int n)
{
    for (int i = 0; i < n; ++i)
        ++counter;   // 这里有数据竞争——两个线程可能同时读-改-写
}
```

用 `std::mutex` 加上一把锁，保证同一时刻只有一个人能操作：

```cpp
#include <iostream>
#include <thread>
#include <mutex>

int counter = 0;
std::mutex mtx;

void increment(int n)
{
    for (int i = 0; i < n; ++i)
    {
        std::lock_guard<std::mutex> lock(mtx);   // 构造时加锁，离开作用域自动解锁
        ++counter;
    }
}

int main()
{
    std::thread t1(increment, 100000);
    std::thread t2(increment, 100000);

    t1.join();
    t2.join();

    std::cout << "counter = " << counter << "\n";   // 一定是 200000
    return 0;
}
```

`lock_guard` 是 RAII 风格的锁——加锁和解锁都不需要手动调用，出了大括号自动解锁，即使中间抛异常也不会忘了。C++17 起更推荐用 `scoped_lock`，它能同时锁多个 mutex 并保证不会死锁：

```cpp
std::mutex mtx1, mtx2;
// ...
std::scoped_lock lock(mtx1, mtx2);   // 同时锁两把，顺序有保证，不会死锁
```

## 原子操作——无锁的计数

对于那些只需要"加一"、"减一"的简单场景，`std::atomic` 提供了一种比互斥锁轻量得多、直接在硬件层面保证原子性的方案：

```cpp
#include <iostream>
#include <thread>
#include <atomic>
#include <vector>

std::atomic<int> counter(0);     // 原子类型——无需 mutex

void increment(int n)
{
    for (int i = 0; i < n; ++i)
        counter.fetch_add(1);     // 原子加 1，不会被打断
}

int main()
{
    std::thread t1(increment, 100000);
    std::thread t2(increment, 100000);

    t1.join();
    t2.join();

    std::cout << "counter = " << counter << "\n";   // 200000，且比互斥锁版本快
    return 0;
}
```

`fetch_add` 在硬件层面保证"读-加-写"是一个不可分割的整体。原子操作是无锁数据结构的基础，但对于复杂的逻辑（比如"借钱之前要先检查余额够不够"，涉及多个步骤），`mutex` 仍然是首选——原子操作只能保护单一的原子步骤，不能保护复合逻辑。

## 条件变量：线程之间发信号

生产-消费模式是并发编程中的核心模型。一个线程生产数据，另一个线程消费数据——消费者必须在有数据时才干活，没有就等。

`std::condition_variable` 负责这个"等-通知"机制：

```cpp
#include <iostream>
#include <thread>
#include <mutex>
#include <condition_variable>
#include <queue>

std::queue<int> dataQueue;
std::mutex mtx;
std::condition_variable cv;
bool done = false;

void producer()
{
    for (int i = 1; i <= 10; ++i)
    {
        {
            std::lock_guard lock(mtx);
            dataQueue.push(i);
            std::cout << "生产: " << i << "\n";
        }
        cv.notify_one();          // 通知等待的消费者
        std::this_thread::sleep_for(std::chrono::milliseconds(50));
    }

    {
        std::lock_guard lock(mtx);
        done = true;
    }
    cv.notify_one();
}

void consumer()
{
    while (true)
    {
        std::unique_lock lock(mtx);
        // wait 会在条件为 false 时释放锁并等待；被通知后重新获取锁并检查条件
        cv.wait(lock, [] { return !dataQueue.empty() || done; });

        while (!dataQueue.empty())
        {
            int val = dataQueue.front();
            dataQueue.pop();
            lock.unlock();               // 处理数据时可以短暂放锁
            std::cout << "    消费: " << val << "\n";
            lock.lock();
        }

        if (done && dataQueue.empty())
            break;
    }
}

int main()
{
    std::thread t1(producer);
    std::thread t2(consumer);

    t1.join();
    t2.join();
    return 0;
}
```

搞懂了条件变量，你就搞懂了线程池、异步任务队列等绝大多数并发框架的底层原理。

## future 与 async：异步执行的返回结果

`std::async` 是并发编程中最便捷的方式——把任务丢到另一个线程，拿到一个"未来会给你结果"的凭证（`std::future`），然后你可以先做别的事，等需要用结果时再取：

```cpp
#include <iostream>
#include <future>
#include <chrono>

int heavyComputation(int x)
{
    std::this_thread::sleep_for(std::chrono::seconds(2));   // 模拟耗时运算
    return x * x;
}

int main()
{
    // 异步执行 heavyComputation(10)
    std::future<int> result = std::async(std::launch::async, heavyComputation, 10);

    // 主线程这期间可以干别的
    std::cout << "正在计算中……\n";

    // 真正需要结果时才阻塞等待
    int answer = result.get();
    std::cout << "结果是: " << answer << "\n";    // 100
}
```

`std::launch::async` 强制启动一个新的线程去跑任务。你也可以用 `std::launch::deferred` 让任务延迟到调用 `get()` 时才在当前线程里执行（惰性求值）。未指定启动策略的 `async` 会让系统自己决定——但"让系统决定"在行为上不完全可预测，明确指定 `launch::async` 是更好的习惯。

## 并行算法——开箱即用的并发

C++17 给标准库的算法加上了执行策略：你可以在 `sort`、`for_each` 等算法上传一个 `std::execution::par`，让它们自动启用多线程并行：

```cpp
#include <algorithm>
#include <vector>
#include <execution>
#include <iostream>

int main()
{
    std::vector<int> v(1000000);
    // 用并行策略对百万个元素排序
    std::sort(std::execution::par, v.begin(), v.end());
    std::cout << "并行排序完成\n";
}
```

注意并行策略只是一个提示——编译器可能帮你多线程优化，也可能当看不见。具体效果取决于你的标准库实现（MSVC 对并行算法的支持较好，libstdc++ 和 libc++ 也在逐步跟进）。

------

多线程编程的坑非常多——数据竞争、死锁、优先级反转、伪共享……但标准库为我们提供的基本工具覆盖了大部分日常需求。在项目中先掌握 `thread` + `mutex` + `lock_guard` 的经典组合，再逐步引入 `async`、`atomic` 和并行算法，是稳扎稳打上升的路线。
