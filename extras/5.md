# 地心探索 · 内存分配控制

默认的 `new` 和 `delete` 给你拿来即用的堆内存，但对于特定场景——尤其游戏引擎、实时系统、高频交易——开发者往往需要更精确地控制"内存在哪里分配、怎么分配、何时释放"。

------

**重载类自己的 operator new/delete**

你可以专门为某个类定制内存分配行为：

```cpp
class MyObject
{
public:
    static void* operator new(size_t size)
    {
        cout << "自定义的 new！分配 " << size << " 字节" << endl;
        return ::operator new(size);   // 底层仍然走全局 new 来拿原始内存
    }

    static void operator delete(void* ptr)
    {
        cout << "自定义的 delete！" << endl;
        ::operator delete(ptr);
    }
};

MyObject* obj = new MyObject();
delete obj;
```

上面只是简单包装了全局 new/delete——实际上你也可以使用自己的预分配池、对齐存储、跟踪内存泄漏等。

`operator new` 接收的是要分配多少字节（`size_t`），而你可以在它和 `operator delete` 中间作任何追踪日志、缓冲池分配、统计等操作。如果只是新包装原有全局 new/delete（像上面例子那样），意义不大——真正的价值在于完全替换内存来源。

------

**分配器：给容器定制内存源**

标准库容器的内存管理是可替换的——你可以写一个**自定义分配器**，让 `vector`、`map` 等容器的内存在你指定的池里分配而不是在全局堆上：

```cpp
template<typename T>
class MyAllocator : public std::allocator<T>
{
public:
    T* allocate(size_t n)
    {
        cout << "自定义分配 " << n << " 个元素" << endl;
        return std::allocator<T>::allocate(n);
    }
};

vector<int, MyAllocator<int>> v;    // 这个 vector 用我的分配器
```

自定义分配器是高级主题——涉及 `rebind`、`propagate_on_container_move_assignment` 等一大堆 API 细节。生产级别的自定义分配器通常不是为了去监视日志，而是为了从固定缓冲池、共享内存、或 CPU cache 亲和的 NUMA 节点分配器里获取内存。

------

**内存池**

写一个简单的内存池思想：

```cpp
class MemoryPool
{
    char pool[4096];       // 预分配一片连续内存
    size_t offset = 0;
public:
    void* allocate(size_t size)
    {
        if (offset + size > sizeof(pool))
            throw bad_alloc();
        void* p = pool + offset;
        offset += size;
        return p;
    }

    void reset() { offset = 0; }
};
```

这种**单调递增（bump allocator）**的内存池——只增不减，直到整个游戏帧结束一口气释放——在游戏引擎和图形渲染中很常见。它的分配成本几乎为零（仅一次指针边界检查和一次递增），但无法单独释放池中的某个对象。如果需要更灵活的池（支持释放），可以考虑 free list 或 slab allocator 等变种。

------

**placement new：在"预定位置"构造对象**

```cpp
char buffer[sizeof(MyObject)];
MyObject* obj = new (buffer) MyObject();   // 在 buffer 位置构造对象，不从堆上分配
obj->~MyObject();                           // placement new 的对象必须显式析构
```

placement new 不分配新内存，而是只在给定的地址上执行构造函数，常用于内存池和自定义分配器中——把"分配内存"和"构造对象"两步骤分离。

------

这些都属于底层利器——不是每个程序员都会在日常工作中动它们，但了解这些机制，才能真正理解 C++ 把"完全控制"这四个字做到了什么程度。
