# 启动《Minecraft》 · 系统调用库

有些事，只靠标准库完成不了——比如列出一个目录下的所有文件、启动另一个外部程序、读写在程序外设定的环境变量、或是通过网络收发数据。这些事情必须走到语言之外，直接跟操作系统打交道。

C++ 的标准库在 C++17 时终于用 `<filesystem>` 解决了文件系统操作的可移植难题。至于子进程和网络，至今仍主要依赖平台 API 或第三方库。我们逐个来把它们用活。

------

## 遍历文件夹：`std::filesystem`

C++17 献给我们的一个重磅礼物就是 `<filesystem>`。它在 `std::filesystem` 命名空间中提供了一整套文件系统操作，跨 Windows / Linux / macOS。

先来一个实际的例子——列出当前目录下所有的文件和子文件夹：

```cpp
#include <iostream>
#include <filesystem>

namespace fs = std::filesystem;

int main()
{
    fs::path current = fs::current_path();       // 获取当前工作目录的路径
    std::cout << "当前目录: " << current << "\n\n";

    for (const auto& entry : fs::directory_iterator(current))
    {
        std::cout << (entry.is_directory() ? "[目录] " : "[文件] ")
                  << entry.path().filename()     // 只显示文件名，不带完整路径
                  << "  (" << entry.file_size() << " 字节)\n";
    }

    return 0;
}
```

`fs::path` 是文件路径的跨平台表示——它自动处理 Windows 上的 `\` 和 POSIX 上的 `/`。`fs::directory_iterator` 让"遍历一个目录"变得像遍历 `vector` 一样简单——一个范围 for 就出来了。

`entry.is_directory()` 和 `entry.file_size()` 让我们一眼就能区分文件和文件夹，顺便看看文件有多大。

## 创建、复制、删除文件

继续往下走——这些操作现在都是标准库的一部分了：

```cpp
#include <filesystem>
#include <fstream>
#include <iostream>

namespace fs = std::filesystem;

int main()
{
    fs::path dir = "my_test_folder";
    fs::path file = dir / "hello.txt";   // / 运算符拼接路径，跨平台自动处理分隔符

    // 创建目录
    if (fs::create_directory(dir))
        std::cout << "目录 " << dir << " 创建成功\n";

    // 创建并写入文件
    {
        std::ofstream out(file);
        out << "Hello, Filesystem!\n";
    }

    // 验证文件存在，打出大小
    if (fs::exists(file))
        std::cout << "文件 " << file << " 存在，大小: "
                  << fs::file_size(file) << " 字节\n";

    // 复制文件
    fs::path copy = dir / "hello_copy.txt";
    fs::copy(file, copy, fs::copy_options::overwrite_existing);

    // 删除文件
    fs::remove(file);

    // 递归删除整个文件夹（C++17 无内置递归删除，需手动遍历；C++20 加了 remove_all）
    // 此处演示逐个删除：
    for (const auto& entry : fs::directory_iterator(dir))
        fs::remove(entry.path());
    fs::remove(dir);

    std::cout << "全部清理完毕\n";
    return 0;
}
```

注意 `dir / "hello.txt"` 这种写法——`fs::path` 重载了 `/` 运算符，用来拼接路径片段。写出来的代码读起来就像在命令行里敲路径一样自然。

`fs::copy_options::overwrite_existing` 让复制操作在目标已存在时覆盖它——不传这个选项的话，目标存在就抛异常。

## 递归遍历子文件夹

`directory_iterator` 只遍历一层。如果要递归进入所有子文件夹，换用 `recursive_directory_iterator`：

```cpp
#include <iostream>
#include <filesystem>
namespace fs = std::filesystem;

int main()
{
    fs::path root = fs::current_path();

    for (const auto& entry : fs::recursive_directory_iterator(root))
    {
        // 用缩进表示层级
        int depth = entry.depth();               // 当前在第几层
        for (int i = 0; i < depth; ++i)
            std::cout << "  ";

        std::cout << (entry.is_directory() ? "[D] " : "[F] ")
                  << entry.path().filename() << "\n";
    }

    return 0;
}
```

运行一下——你会看到整个项目的文件树被打印出来，每个文件/文件夹前面按嵌套深度缩进。这不就是自己写了个简易的 `tree` 命令？

## 环境变量

从操作系统的环境变量里读值，用 `std::getenv`（来自 `<cstdlib>`）：

```cpp
#include <iostream>
#include <cstdlib>

int main()
{
    const char* path = std::getenv("PATH");
    if (path)
    {
        std::cout << "PATH 的内容:\n";
        // Windows 上用 ; 分隔，Linux/macOS 上用 : 分隔
        std::string pathStr(path);
        std::cout << pathStr << "\n";
    }
    else
    {
        std::cout << "PATH 环境变量不存在\n";
    }

    return 0;
}
```

`getenv` 返回的指针不需要释放——它指向系统内部维护的环境变量副本。你只能读，不能通过它修改环境变量的值。想设置环境变量，Windows 上用 `_putenv`，POSIX 上用 `setenv`。

## 启动外部程序

### 简单版：`std::system`

```cpp
#include <cstdlib>

int main()
{
    // 在命令行里执行 dir（Windows）或 ls（Linux/macOS）
#ifdef _WIN32
    int ret = std::system("dir");
#else
    int ret = std::system("ls -la");
#endif
    std::cout << "命令返回码: " << ret << "\n";
    return 0;
}
```

`std::system` 会启动一个 shell，执行你给的字符串，等执行完了把返回值给你。在最简单"敲个命令看看结果"的场景够用了，但它有几个致命短板：

- 你拿不到被调程序的**输出内容**——只能看到它直接打印在终端上。
- 弹出命令行窗口（Windows）或嵌套 shell，不能精细控制。
- 如果在不信任的环境下执行用户输入的字符串，安全风险非常大。

### 进阶版：用管道捕获输出

如果你想**读到**外部程序的输出（比如调 `python --version` 拿到版本字符串），就要用 `popen`（POSIX）或 `_popen`（Windows）：

```cpp
#include <cstdio>
#include <iostream>
#include <string>
#include <array>

std::string exec(const char* cmd)
{
    std::array<char, 128> buffer;
    std::string result;

#ifdef _WIN32
    FILE* pipe = _popen(cmd, "r");
#else
    FILE* pipe = popen(cmd, "r");
#endif

    if (!pipe)
        return "命令执行失败";

    while (fgets(buffer.data(), buffer.size(), pipe) != nullptr)
        result += buffer.data();

#ifdef _WIN32
    _pclose(pipe);
#else
    pclose(pipe);
#endif

    return result;
}

int main()
{
    std::string output = exec("python --version");
    std::cout << "Python 版本: " << output;
    return 0;
}
```

`popen` 打开一个管道（pipe），把外部命令的标准输出接到你的程序里，你像读文件一样一行行读回来。这就是很多自动化脚本或构建工具背后"调命令、拿结果、做判断"的基础原理。

## 做一个自己的工具：批量文件重命名

把上面几样东西拼起来，写一个实际工具——遍历某个文件夹中的所有 `.txt` 文件，统一在文件名前面加上一个前缀：

```cpp
#include <iostream>
#include <filesystem>
#include <string>

namespace fs = std::filesystem;

int main(int argc, char* argv[])
{
    if (argc < 3)
    {
        std::cerr << "用法: " << argv[0] << " <文件夹路径> <前缀>\n";
        return 1;
    }

    fs::path folder(argv[1]);
    std::string prefix(argv[2]);

    if (!fs::exists(folder) || !fs::is_directory(folder))
    {
        std::cerr << "错误: " << folder << " 不存在或不是文件夹\n";
        return 1;
    }

    int count = 0;
    for (const auto& entry : fs::directory_iterator(folder))
    {
        if (!entry.is_regular_file())
            continue;

        if (entry.path().extension() != ".txt")
            continue;

        fs::path newName = entry.path().parent_path()
                         / (prefix + entry.path().filename().string());

        fs::rename(entry.path(), newName);
        std::cout << "重命名: " << entry.path().filename()
                  << " -> " << newName.filename() << "\n";
        ++count;
    }

    std::cout << "完成，共处理 " << count << " 个文件\n";
    return 0;
}
```

编译运行：

```bash
# Windows
renametool.exe C:\notes 备份_

# Linux/macOS
./renametool ~/notes 备份_
```

这就是一个直接从零写出、立刻用得上的命令行小工具。它背后就靠 `fs::directory_iterator`、`fs::path`、`fs::rename` 这几个 API，加起来不到四十行代码。

## 网络：标准库之外的疆土

C++23 引入了 `std::net` 命名空间，但目前在广泛使用的编译器上还不可用。实际项目中，网络编程的主流方案是第三方库：

- **Boost.Asio**：最老牌的跨平台异步网络库，支持 TCP/UDP/串口/定时器/信号等。
- **cpp-httplib**：单头文件、零依赖的 HTTP 客户端 + 服务端库。想快速发个 GET 请求或起个本地 REST 服务，引入一个头文件就够了。
- **libcurl**：C 语言的网络传输库，通过 C++ 包装后可用于 HTTP/FTP/SMTP 等几乎所有协议。

一个快速的 cpp-httplib 示例感受一下——发起一次 HTTP GET 请求：

```cpp
#include <httplib.h>
#include <iostream>

int main()
{
    httplib::Client cli("https://api.github.com");

    auto res = cli.Get("/users/langyo");
    if (res && res->status == 200)
        std::cout << res->body << "\n";
    else
        std::cerr << "请求失败\n";

    return 0;
}
```

网络编程不是标准 C++ 的内置能力，但用第三方库包装起来后同样可以简洁优雅——上面的示例只需要把 `httplib.h` 这一个头文件放进项目，无需编译任何源文件，就能在 Windows / Linux / macOS 上直接编译运行。

------

`<filesystem>` 已是 C++17 标配，让文件系统的操作变得可移植、可阅读、不需要写 `#ifdef _WIN32` 了。而 `popen` 和网络库补上了"与外互通"的另外两根筋。学会这些，你的 C++ 工具就已经能走出黑窗口，去真正地影响操作系统里的一物一件了。
