# 这片土地的旧时光 · 详解 C++98 与新版 C++ 的不同

C++ 是一门持续演进了几十年的语言。如果你在旧项目里一头撞上 C++98 的代码，或者某位老前辈训你"别用那些花里胡哨的新特性，老语法就够用了"，你需要知道它们到底差在哪、新的到底好在哪里。

------

**变量初始化**

C++98 喜欢用一个等号加括号：

```cpp
int x = 0;
vector<int> v;
// v = {1, 2, 3};   // ❌ C++98 不支持初始化列表
```

C++11 统一了初始化语法——花括号 `{}`（统一初始化和初始化列表）：

```cpp
int x{0};
vector<int> v{1, 2, 3};
map<string, int> dict{{"a", 1}, {"b", 2}};  // 嵌套初始化
```

花括号初始化不会发生窄化转换（如 `int x{3.14}` 直接编译错误），比等号初始化更严格安全。

------

**auto 关键字**

C++98 里写迭代器类型长到吐：

```cpp
for (vector<int>::const_iterator it = v.begin(); it != v.end(); ++it)
```

C++11 起直接用 `auto`：

```cpp
for (auto it = v.begin(); it != v.end(); ++it)
```

而且结合范围 for：

```cpp
for (const auto& x : v)    // C++11 范围 for
    cout << x << " ";
```

`auto` 不光用来写迭代器——它推任意可推导的类型，并能减少代码中因为类型调整而重复修改同一类型的多处显式声明。只要是"类型名明显可以从初始表达式一眼看出来"的地方，大胆改用 `auto`。

------

**范围 for**

C++98 必须手写 for 循环遍历容器；C++11 范围 for 是语法糖，背后还是迭代器，但不再需要手写长长的 `begin`/`end` 那一套。

------

**智能指针**

C++98 有 `auto_ptr`，很遗憾这是个灾难性的设计——它用拷贝实现所有权转移，导致函数传参时源指针被暗中掏空。C++11 把 `auto_ptr` 正式标记为弃用并引入 `unique_ptr`（独占、禁止拷贝）和 `shared_ptr`（共享、引用计数），配合移动语义实现安全且清晰的资源管理。

------

**Lambda 表达式**

C++98 只能用函数对象（手写类重载 operator() 或者函数指针），非常繁琐。C++11 起 Lambda 表达式成为编写小型可调用对象的首选——逻辑就近放在使用地点附近，代码可读性跳跃式提升。

------

**移动语义**

C++98 里把一个大的 vector 从函数中返回，会被拷贝一份（除非编译器有 RVO，但不可强依赖）。C++11 引入右值引用和移动语义后，返回大容器变成了指针交换般的 O(1) 开销。

------

**nullptr**

C++98 空指针用 `NULL`（本质上是 `0` 宏），很容易在函数重载时跟整数类型搅在一起产生意想不到的重载决议。C++11 引入 `nullptr_t` 类型的 `nullptr` ——它只能跟指针互操作，不会误匹配 `int` 重载。

什么时候该用 nullptr？永远。绝对不要再写 NULL 或 0 作为空指针字面量。

------

**override 和 final**

C++98 里重新定义基类函数完全靠"手动核对"——签名对不上也不会报错，只会悄无声息地变成一个新函数。C++11 的 `override` 让编译器帮你检查，`final` 阻止继续被重写。

------

**static_assert**

C++98 只有运行时的 `assert`，模板参数不对时希望在编译期就卡断只能靠一些奇技淫巧。C++11 `static_assert` 直接提供编译期断言，早发现问题早报告。

典型的 `static_assert` 用法：要求模板参数必须是某种大小 `static_assert(sizeof(T) >= 4, "Type too small");`。这种约束越早执行越好——编译期就钉死，别让用户的调用跑到一半才崩。

------

**C++14/17/20 的突出改进**

- **C++14**：泛型 Lambda（参数写 auto）、`make_unique`、`decltype(auto)`。
- **C++17**：结构化绑定（`auto [a, b] = pair;`）、`if constexpr`、`string_view`、`optional`、`variant`、`filesystem`。
- **C++20**：Concepts（约束模板）、Ranges（类似管道风格的算法链）、协程、`format`（类似 Python 字符串格式化）、三向比较运算符 `<=>`。

C++ 的进化不是"推翻旧版"，而是在每一版基础上添砖加瓦、修补历史缺陷。学习旧版知识不会白费，而新特性也能让写代码既清爽又安全。
