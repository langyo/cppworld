# 这片土地的旧时光 · 详解 C++98 与新版 C++ 的不同

C++ 是一门持续演进了几十年的语言。你可能在旧项目里撞上 C++98 的写法，或听到老派程序员说"新的花里胡哨，旧的就够用"。了解一下它们到底差在哪，以及新的好在哪里，才能拿来就辨得清。

------

## 变量初始化

C++98 的初始化方式有限：

```cpp
int x = 0;                // 赋值式初始化
vector<int> v;
// v = {1, 2, 3};         //  C++98 不支持初始化列表：会报错
```

C++11 引入统一的 `{}` 初始化：

```cpp
int x{0};
vector<int> v{1, 2, 3, 4, 5};
map<string, int> dict{ {"Alice", 25}, {"Bob", 30} };  // 二重嵌套初始化
```

花括号初始化有个硬气的好处——它禁止"窄化转换"。比如 `int x{3.14}` 会直接报编译错误，而 `int x = 3.14` 是悄悄截掉小数部分，等程序跑出异常行为了你才追悔莫及。

------

## auto 关键字

C++98 必须每次都把类型写全，尤其迭代器类型又臭又长：

```cpp
for (vector<int>::const_iterator it = v.begin(); it != v.end(); ++it)
    cout << *it << " ";
```

C++11 起用 `auto` 让编译器替你推导：

```cpp
for (auto it = v.begin(); it != v.end(); ++it)
    cout << *it << " ";
```

`auto` 不光用在迭代器上——任何地方的类型要是明显能从初始化表达式看出来（比如 `auto x = 42;`），就可以用 `auto` 代替。代码短了，类型改了你只调一处而不是处处改。

------

## 范围 for

C++98 要手写全套 for 循环。C++11 引入范围 for，背后仍然是迭代器，但不用你写了：

```cpp
vector<int> v{10, 20, 30};
for (const auto& x : v)      // 常引用，不拷贝，不修改
    cout << x << " ";
```

------

## 智能指针

C++98 有一个 `std::auto_ptr`——这是个灾难。它用拷贝来实现所有权转移，导致传参时源指针被暗中掏空。C++11 把它扔进了垃圾堆，换上了 `unique_ptr`（独占、禁止拷贝、只允许移动）和 `shared_ptr`（共享、引用计数）。

```cpp
// C++11 起的标准写法
auto p1 = make_unique<int>(42);    // 独占，离开作用域自动 delete
auto p2 = make_shared<int>(100);   // 共享，有引用计数
```

这意味着在新代码里你几乎永远不需要手动写 `delete` 了。

------

## Lambda 表达式

C++98 要传一个"判断某个条件"给算法，只能在外面定义一个类重载 `operator()`，或者传一个函数指针——因为定义位置和调用位置离得太远，代码读起来格外费劲。

C++11 的 Lambda 把这一段逻辑直接写在调用位置旁边：

```cpp
vector<int> v{5, 2, 8, 1, 3};
sort(v.begin(), v.end(),
          [](int a, int b) { return a > b; });   // 降序排序——一行搞定
```

------

## 移动语义

C++98 把一个大型的 `vector` 从函数里返回，理论上会原地把全部元素拷贝一遍。编译器如果有 RVO（返回值优化）可能帮你消掉拷贝，但不能保证每次都降得住。

C++11 引入右值引用和移动语义后，返回大容器变成了"交换三个指针"的操作——O(1) 结束，不再有几十万次元素内存拷贝。

```cpp
vector<int> createBigVector()
{
    vector<int> result(1000000);
    // ... 把每个元素填上 ...
    return result;     // C++11 起，移动出去而不拷贝
}
```

------

## nullptr

C++98 用 `NULL` 表示空指针，但 `NULL` 本质上就是 `0`（一个整数宏）。函数重载时经常会误匹配到 `int` 版的重载。

C++11 引入 `nullptr_t` 类型的 `nullptr`，它只能与指针类型互操作，绝不会错配到整数重载上。这之后不要再写 `NULL` 或 `0` 来代表空指针了。

------

## override 和 final

C++98 里想覆盖基类的虚函数，全靠手写时自己保证签名精确匹配。写错一个字母不会被发现——它会静默变成"另外写了一个新的不相关的函数"，而不是覆盖虚函数。

```cpp
class Base
{
public:
    virtual void draw() const {}
};

class Derived : public Base
{
public:
    // void draw() { ... }          // C++98: 漏了 const，无声地变成了新函数
    void draw() const override {}    // C++11: override 让编译器检查是否真在覆盖
};
```

`final` 则能阻止某一个虚函数或整个类被后代继续覆盖。

------

## C++14/17/20 持续加码

- **C++14**：泛型 Lambda（参数写 `auto`）、`make_unique`、`decltype(auto)`。
- **C++17**：结构化绑定（`auto [a, b] = pair;`）、`if constexpr`、`string_view`、`variant`、`filesystem`。
- **C++20**：Concepts（约束模板）、Ranges（管道式的算法链）、协程、`std::format`（类 Python 字符串格式化）、三向比较运算符 `<=>`。

C++ 的演进不是"推倒旧版"，而是每代添砖加瓦、修复历史缺陷。学会旧的知识并不会白费，但如果不跟上新的一代，你写出来的代码就是在白白多费劲——而别人只用一行新语法就做完了。
