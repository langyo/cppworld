# 类型身份证 · RTTI

在编译期，编译器知道你每一个变量的类型。但到了运行期，如果你有一根基类指针，你能不能知道它到底指向了哪个子类的对象？

答：能——靠 **RTTI**（Run-Time Type Information，运行时类型信息）。C++ 提供了两个核心工具来做运行时类型查询：`typeid` 和 `dynamic_cast`。

------

**typeid：查查你到底是谁**

```cpp
#include <typeinfo>
#include <iostream>
using namespace std;

class Animal { virtual ~Animal() = default; };
class Cat : public Animal {};
class Dog : public Animal {};

int main()
{
    Animal* a = new Cat();

    cout << typeid(*a).name() << endl;       // 输出大至 "class Cat"（具体名字实现定义）

    if (typeid(*a) == typeid(Cat))
        cout << "这确实是一只猫" << endl;

    delete a;
}
```

`typeid` 返回一个 `type_info` 对象。配合 `==` 可以直接判断运行时类型是否等于另一个类型。

注意 `typeid(*a)` 的 `*a` 是解引用——如果你写 `typeid(a)`，拿到的就是指针类型 `Animal*` 而非指针指向的对象的真实类型。这点坑在写 RTTI 表式时最容易踩到。

还要区分：如果类型没有虚函数，`typeid` 不会执行运行时查表——它在编译期就根据静态类型给结果了。所以只有多态的类（有 virtual 函数的）才具备真正的 RTTI。

------

**dynamic_cast：安全地朝下转换**

你有基类指针，想看看它究竟是不是某个派生类。使用 `dynamic_cast`：

```cpp
Animal* a = new Cat();

Cat* c = dynamic_cast<Cat*>(a);
if (c)
    cout << "安全转为 Cat 指针成功" << endl;

Dog* d = dynamic_cast<Dog*>(a);
if (!d)
    cout << "不是 Dog，返回 nullptr" << endl;
```

`dynamic_cast` 在运行时检查对象的真实类型，如果不匹配，指针版本返回 `nullptr`，引用版本抛出 `std::bad_cast` 异常。这是从基类向下安全转型的标准手段——比暴力 `static_cast` 或 C 风格强制转换多了一层安全检查。

**注意**：`dynamic_cast` 只对有虚函数的类有效（即至少有一个 virtual 函数）。对于非多态类型，`dynamic_cast` 不可用。

------

**为什么不要滥用 RTTI**

RTTI 是一种在运行时手动判断类别的工具，但如果你的代码里存在大量的 `if (typeid(...) == ...)`，通常说明设计本身需要重构——多态和虚函数本应由编译器自动分派到正确的方法，而不应该是你在运行时自己枚举类型再选择路径。

好的面向对象设计里，RTTI 只在不得不为的时候亮出来（比如跨语言接口、序列化/反序列化框架、日志诊断、或者联动的动态插件系统中需要将一张基类指针重置回具体子类）。日常业务逻辑中，它出现得太频繁就是一个明显的设计警钟。

------

**RTTI 的开销与禁用背景**

RTTI 不是免费的——编译器要额外维护 type_info 表，dynamic_cast 要查继承树。在嵌入式领域或一些极度看重体量和性能的系统中，会用 `-fno-rtti` 等选项完全禁用 RTTI。禁用 RTTI 的代码中 `dynamic_cast` 和 `typeid` 均不可用。平衡做法通常是：除极端系统外的大部分程序保留 RTTI；只关闭异常（-fno-exceptions）的项目要多得多。
